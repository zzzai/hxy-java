# MiniApp Member Domain SLA Routing v1 (2026-03-10)

## 1. 目标
- 明确会员域告警的主责路由、SLA 和升级链路，保证登录、签到、积分、地址、资产账本故障有人接、有人处置、有人闭环。
- 统一工单字段规则，保证会员域事件与发布门禁、误发布回滚可以串起来追责。

## 2. 告警路由矩阵

| 告警类别 | 典型信号 | 主责团队 | 协同团队 | 默认渠道 |
|---|---|---|---|---|
| 账号鉴权 | 登录成功率下滑、`1004001000` 激增 | 账号体系值班 | 发布负责人、数据治理值班 | IM + 电话 + 工单 |
| 签到转化 | `MEMBER_SIGNIN_CONVERSION_RATE` 连续异常 | 会员增长运营 | 会员资产负责人、发布负责人 | IM + 工单 |
| 积分流水一致性 | `MEMBER_POINT_LEDGER_CONSISTENCY_RATE` 下滑、`1004008000` 激增 | 会员资产负责人 | 数据治理值班、增长运营 | IM + 工单 |
| 地址可用性 | 地址可用率下滑、地址写接口 5xx 升高 | 会员基础资料负责人 | 发布负责人、客服值班 | IM + 工单 |
| 资产账本 / 误发布 | `1004009901` 命中、账本一致率下滑、开关越权 | 会员资产负责人 | 促销账本负责人、SRE、发布负责人 | IM + 电话 + 工单 |

## 3. P0 / P1 / P2 响应时限与升级链路

| 等级 | 典型场景 | 首响 SLA | 缓解 SLA | 关闭 SLA | 升级链路 |
|---|---|---|---|---|---|
| P0 | 登录主入口不可用、账本误发布波及主链路、关键五键缺失导致不可追责 | 5 分钟 | 15 分钟 | 4 小时 | 值班 -> 会员域负责人 -> 技术负责人 -> 业务负责人 |
| P1 | 积分流水一致率/地址可用率跌破阈值、`1004009901` 误返回、签到奖励发放异常 | 15 分钟 | 1 小时 | 24 小时 | 值班 -> 会员域负责人 -> 发布负责人 |
| P2 | 签到转化率异常、趋势预警、TopN 错误码集中但未伤及主链路 | 30 分钟 | 4 小时 | 72 小时 | 指标 Owner -> 会员域负责人 |

## 4. 升级触发规则

| 触发条件 | 升级动作 |
|---|---|
| P0 15 分钟未缓解 | 自动升级到技术负责人 + 业务负责人 |
| P1 1 小时未缓解 | 自动升级到发布负责人 |
| 同类会员域告警 24 小时内重复 >= 3 次 | 升级为专项治理，进入周会跟踪 |
| 工单缺任一必填字段 | 自动退回并标记 `INVALID_TICKET` |
| 开关关闭态返回 `1004009901` | 立即升级到 SRE + 发布负责人 |

## 5. 工单必填字段
- 每张工单必须显式携带以下字段，不允许缺列：
  - `runId`
  - `orderId`
  - `payRefundId`
  - `sourceBizNo`
  - `errorCode`

## 6. 可空字段填 `0` 规则
1. 规则
- 字段不适用、上游未产生、当前事件天然无该业务主键时，统一填字符串 `"0"`。
- 不允许填空字符串、`null`、`UNKNOWN`、`-`。
- 有真实值时必须优先真实值，不得用 `"0"` 覆盖。

2. 会员域典型场景
- 登录告警：
  - `runId=0`
  - `orderId=0`
  - `payRefundId=0`
  - `sourceBizNo=0`
  - `errorCode=1004001000` 或实际错误码
- 地址可用率告警：
  - `runId=0`
  - `orderId=0`
  - `payRefundId=0`
  - `sourceBizNo=0`
  - `errorCode=1004004000` 或实际错误码
- 签到转化告警：
  - `runId=0`
  - `orderId=0`
  - `payRefundId=0`
  - `sourceBizNo=0`
  - `errorCode=0`（纯指标波动但无单一错误码时）
- 资产账本误发布：
  - `runId` 优先取灰度任务/巡检任务 ID，无则 `0`
  - `orderId=0`
  - `payRefundId=0`
  - `sourceBizNo` 优先取真实账本来源号，无则 `0`
  - `errorCode=1004009901`

3. 审计要求
- 工单系统、告警平台、发布门禁日志三处的字段值必须一致。
- 若后续补到真实值，只允许追加修正，不允许覆盖原始审计记录。

## 7. 值班闭环
1. `Open`：监控系统自动建单并写入五键。
2. `Ack`：主责值班确认等级、范围、开关状态。
3. `Mitigating`：执行限流、降级、活动下线、开关回滚或发布回滚。
4. `Resolved`：指标回到阈值内且 2 个窗口稳定。
5. `Postmortem`：48 小时内完成复盘，并记录是否需要调整门禁阈值。

## 8. 验收标准
1. 会员域每类告警都能路由到唯一主责团队。
2. P0/P1/P2 的响应、缓解、关闭时限明确可核验。
3. 工单五键固定存在，且可空字段按 `"0"` 规则统一处理。
4. 误发布事件可从告警、工单、发布记录三侧串联追溯。
