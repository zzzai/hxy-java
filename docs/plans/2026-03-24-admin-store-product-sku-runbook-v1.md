# Admin Store Product / SKU Runbook v1 (2026-03-24)

## 1. 目标与范围
- 为门店 SPU 映射与 SKU 价库存运维提供后台标准操作顺序。
- 范围只覆盖后台运维，不覆盖供应链审批、不覆盖前台放量结论。

## 2. 操作入口

| 能力 | 页面入口 | 核心操作 |
|---|---|---|
| `ADM-007` 门店商品 SPU 映射 | `mall/product/store/spu/index` | 单条保存、批量映射、删除、选项选择 |
| `ADM-008` 门店 SKU 运维 | `mall/product/store/sku/index` | 单条保存、批量保存、批量调整、手工调库存、库存流水重试 |

## 3. 审计键最小集
- `storeId`
- `spuId`
- `skuId`
- `bizType`
- `bizNo`
- `stockFlowId`
- `storeIds[]`
- `operator`

## 4. 标准操作顺序
1. 先完成门店 SPU 映射，再做门店 SKU 精细化管理。
2. 批量映射前先确认 `store-options` / `spu-options` 候选项正确。
3. 批量调价调库存前先拉取当前页和详情样本，记录变更前状态。
4. 手工调库存必须带业务号 `bizNo`，并在操作后回看库存流水页。
5. 批量重试库存流水只对 `allowRetry/canBatchRetry` 的记录执行。

## 5. 失败处理
- 映射或调整后若页面回读数量与预期不一致，暂停扩大批次。
- 库存流水批量重试出现 `failedCount > 0` 时，记录 `id + failReason`，转人工逐条处理。
- 对 `skipped` 样本，只写“未满足重试条件”，不写“已恢复”。

## 6. 回滚 / 暂停规则
- 批量保存 / 调整出错时，以保存前快照或导出样本为准进行回滚。
- 手工调库存异常时，暂停进一步库存重试，只保留只读观察。

## 7. 当前结论
- 门店商品 / SKU 域已具备独立后台 runbook。
- 不得把库存运维动作直接写成销售链路已稳定闭环。
