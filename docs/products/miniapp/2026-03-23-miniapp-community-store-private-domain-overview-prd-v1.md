## 1. 当前真值

| 维度 | 当前状态 | 说明 |
|---|---|---|
| 首页增长入口 | 已有基础能力 | 首页、券中心、积分商城、搜索链路已形成增长入口 |
| 会员登录与留资 | 已有基础能力 | 登录、手机号授权、会员资料、标签、等级、资产页已形成基础会员承接 |
| 邀请裂变 | 已实现 runtime，但仍不可放量 | `Referral` 已形成真实页面/API，但仍处于 `Can Develop / Cannot Release` |
| 到店预约与服务 | 已形成交易主链 | `Booking` 创建/取消/add-on 可继续开发，仍不可放量 |
| 用户侧企微/社群沉淀 | 未形成真实 runtime | 当前没有稳定的“加企微/入社群”用户链路 |
| 门店私域运营后台 | 部分存在 | booking review 已形成店长侧 App/企微双通道治理，但这是门店治理链，不是用户私域链 |

## 2. 北极星目标
1. 用户从公域进入小程序后，能在 1 次到店或 1 次下单后进入门店私域。
2. 门店能围绕“预约服务、复购提醒、社群活动、邀请裂变、会员成长”做持续经营，而不是只依赖一次性交易。
3. 总部能看到“哪家门店私域沉淀做得好、哪种触点转化最好、哪类用户最容易复购”。

## 3. 私域主链路
1. 公域获客：门店海报、二维码、附近流量、老带新、内容触达进入小程序。
2. 首次转化：领券、注册、预约、购买、签到、积分、活动参与。
3. 留资建档：手机号授权、会员资料、门店归属、首单意图标签。
4. 公转私：服务前提醒、到店核销、服务后评价、优惠权益、加企微/进社群。
5. 私域复购：任务触达、社群活动、会员等级、内容种草、邀请裂变、拼团/活动。
6. 数据归因：以 `memberId/storeId/channel/touchpoint` 为最小归因单元，持续看转化和复购。

## 4. 当前缺口

| 主题 | 当前缺口 | 影响 |
|---|---|---|
| 用户侧企微承接 | 没有真实“添加店长/门店企微”链路 | 私域沉淀停在会员态，无法转入长期触达 |
| 社群承接 | 没有门店社群入群链路、群任务和群活动页 | 无法形成社区门店运营闭环 |
| 门店归因 | 缺统一的 `首触门店 / 服务门店 / 私域归属门店` 口径 | 后续裂变、复购、运营考核容易串账 |
| 任务化触达 | 标签/等级/资产已有页面，但未形成自动化任务体系 | 无法把会员资产转化为运营动作 |
| 私域 SOP | 缺店长/店员/企微运营标准动作和日常节奏 | 门店执行依赖人工经验，不可复制 |

## 5. 开发优先级

| 优先级 | 专题 | 目标 |
|---|---|---|
| P0 | 留资、门店归因、到店公转私触点 | 建立私域最小闭环 |
| P1 | 用户侧企微/社群承接、会员任务触达 | 把私域沉淀转成持续运营 |
| P1 | 裂变与邀请、活动种草 | 形成获客与复购双轮 |
| P2 | 指标中台、门店 SOP、运营看板 | 提升可复制性和规模化经营能力 |

## 6. 当前结论
- 这 9 份文档用于补齐“社区门店私域主链”的产品真值。
- 当前项目具备部分私域基础能力，但距离真正的“用户入私域、门店可运营、总部可归因”仍有明显缺口。
- 后续开发优先级必须围绕：留资 -> 归因 -> 公转私 -> 复购 -> 裂变 -> 指标。
