# MiniApp Copy & Terminology Spec v1 (2026-03-08)

## 1. Objective
Normalize copy across stitch pages (`_1~_50`, `shadcn_1~4`, `closure_p0`) and backend contracts.

## 2. Terminology Baseline
- Use Chinese domain terms consistently; avoid random English insertions (e.g. replace "Top Picks" with fixed Chinese term).
- Core nouns:
  - 预约单
  - 订单
  - 售后单
  - 退款进度
  - 优惠券
  - 积分

## 3. Mandatory Rules
- Every page must have `<title>`.
- Error copy must be neutral and include code.
- Marketing copy must not imply success before backend confirmation.
- Compliance copy must include refund/fulfillment boundaries where applicable.

## 4. Review Checklist
1. Title present.
2. No mixed-language term drift.
3. Error copy includes action.
4. Empty-state copy is non-accusatory and actionable.
