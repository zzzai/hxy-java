# MiniApp 社区门店私域专题：公域引流与首触点转化 PRD v1（2026-03-23）

## 0. 文档定位
- 目标：定义社区门店从公域流量进入小程序后的首触点设计，确保用户不是“进来就走”，而是能进入会员、交易或私域沉淀链路。

## 1. 当前真值
- 已有增长入口：首页、券中心、积分商城、搜索。
- 已有交易入口：商品详情、预约服务、订单链路。
- 已有裂变基础：邀请有礼/Referral runtime 已实现，但仍不可放量。
- 当前缺口：没有面向“社区门店私域沉淀”的统一首触点设计，用户来源与后续门店归属弱绑定。

## 2. 目标用户路径
1. 用户通过门店海报、二维码、老客转发、社区社群链接进入小程序。
2. 首屏看到与门店相关的利益点：新人券、预约体验、门店招牌服务、社群福利。
3. 用户完成至少一个转化动作：领券、授权手机号、预约、加购、签到、活动参与。
4. 转化后把用户归到后续链路：会员建档、门店归属、公转私触点、复购任务。

## 3. 产品方案

| 模块 | 目标 | 当前状态 | 目标方案 |
|---|---|---|---|
| 门店专属入口页 | 提升门店场景转化 | 未独立实现 | 按 `storeId` 展示门店券包、招牌项目、社群福利 |
| 新客券包 | 降低首次成交门槛 | 已有券中心基础能力 | 支持门店专属券与渠道券并行 |
| 首次预约引导 | 把浏览变下单 | `Booking` 已有主链 | 在门店页突出“首单体验项目/到店预约” |
| 首次加粉引导 | 把交易用户转私域 | 未实现 | 首单完成、评价完成、券领取后展示企微/社群入口 |

## 4. 关键规则
1. 首触点必须带 `channel`、`storeId`、`campaignId` 三类归因字段。
2. 新客利益点优先级：新人券 > 预约体验 > 门店招牌服务 > 社群福利。
3. 不允许在未授权手机号、未绑定门店归属前直接把用户写成“已进入私域”。
4. 若门店未配置私域承接能力，只允许停留在小程序会员链，不伪造企微/社群成功。

## 5. 依赖与缺口
- 依赖现有首页增长、券中心、预约主链。
- 新增需求：
  - 门店专属 landing 页
  - `sourceChannel/storeId/campaignId` 首触点归因
  - 首触点券包与服务位配置

## 6. 验收口径
- [ ] 门店二维码/链接进入后能识别 `storeId`。
- [ ] 首触点至少支持“领券、预约、会员授权”三类主动作。
- [ ] 门店未配置私域承接时，页面不出现虚假的企微/社群入口。
