# MiniApp 社区门店私域专题：离店后的企微/社群沉淀与复购 PRD v1（2026-03-23）

## 0. 文档定位
- 目标：定义用户离店或交易完成后，如何通过企微和社群完成持续经营。

## 1. 当前真值
- 当前项目没有真实的用户侧“加企微/进社群”链路。
- booking review 已有店长 App/企微双通道治理，但这是后台运营通知，不等于用户私域承接。
- 因此本专题当前属于“文档重建、待开发”，不是“已落地能力说明”。

## 2. 目标链路
1. 用户完成首单/首约/服务评价后，进入门店私域承接页。
2. 用户可选择：
   - 添加门店企微
   - 加入门店社群
   - 领取社群专属福利
3. 入企微/入群后，进入后续任务和触达体系：复购提醒、活动报名、邀新激励、内容种草。

## 3. 业务规则
- 一个门店可以有多个社群，但用户私域归属必须以门店为主。
- 企微与社群是两条不同承接方式：
  - 企微更适合一对一服务与高价值转化
  - 社群更适合活动通知、拼团裂变与内容运营
- 门店未配置企微/社群时，只能回退会员链与站内消息，不伪造成功。

## 5. 开发建议
- P1：先做承接页与门店配置模型，不先做复杂自动化。
- P1：统一 `storeId + operatorId + entryTouchpoint` 的归因结构。
- P2：再补群活跃任务、群活动、群裂变工具。

## 6. 验收口径
- [ ] 用户侧企微/社群承接页具备明确的门店归属。
- [ ] 未配置门店企微/社群时不展示假入口。
- [ ] 企微/社群入链动作可回溯到触点来源。
