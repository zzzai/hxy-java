# MiniApp 社区门店私域专题：社群活动、拼团裂变、邀请有礼 PRD v1（2026-03-23）

## 0. 文档定位
- 目标：定义社区门店在私域场景下的裂变增长机制，包括邀请有礼、拼团、社群活动、门店福利日等。

## 1. 当前真值
- `Referral` runtime 已实现，但仍处于 `Can Develop / Cannot Release`。
- 首页增长、券中心、积分商城已有基础活动承接能力。
- 当前缺口：没有把这些能力组织成“社区门店私域裂变产品包”。

## 2. 裂变结构建议

| 机制 | 目标 | 当前状态 | 目标动作 |
|---|---|---|---|
| 邀请有礼 | 老带新 | Referral runtime 已有基础 | 绑定门店归属、邀请人归因、到店后奖励 |
| 社群拼团 | 低成本拉新 | 待开发 | 支持门店团、社群团、爆款体验团 |
| 门店福利日 | 激活周边社区客群 | 待开发 | 固定门店周活动与限时券 |
| 老客分享海报 | 借助熟人传播 | 待开发 | 从订单完成/评价完成页发起 |

## 3. 关键规则
1. 裂变必须带门店归属与渠道归因。
2. 奖励要区分“邀请成功”“首单成功”“到店成功”三种结算条件。
3. 不允许让跨门店裂变把归属关系冲乱；需有默认门店判定规则。
4. 所有裂变奖励与任务都要能回溯到 `memberId/storeId/sourceBizNo`。

## 4. 开发建议
- P1：先以邀请有礼 + 门店裂变海报为最小闭环。
- P1：拼团和社群活动在门店配置中心沉淀模板。
- P2：再补门店专属活动日、群接龙、社群任务等增强能力。

## 5. 验收口径
- [ ] 邀请关系、门店归属、奖励结算三者口径一致。
- [ ] 至少有 1 个门店维度裂变工具可落地。
- [ ] 不能把当前 Referral runtime 误写成“社区私域裂变已全面上线”。
