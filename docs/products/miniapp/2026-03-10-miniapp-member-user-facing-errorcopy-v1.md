# MiniApp 会员域用户可见错误文案 v1（2026-03-10）

## 1. 目标与强约束
- 目标：建立会员域“错误码/降级信号 -> 用户文案 -> 恢复动作 -> 页面行为”单一映射。
- 强约束：
  1. 只允许按 `errorCode/degraded/degradeReason` 分支，禁止按 `message` 分支。
  2. 每条错误都必须给用户明确恢复动作，且至少包含 `重试`、`刷新`、`联系客服` 之一。
  3. 降级场景不得出现成功态文案、成功 icon、成功彩带、完成动效。
  4. 任何“自动注册”“签到成功”“到账成功”都必须以后端成功结果为前提。

## 3. 降级信号规范

| 降级信号 | 用户文案 | 恢复动作 | 页面行为 | 禁止文案/动效 |
|---|---|---|---|---|
| `degraded=true`（会员域通用） | `当前为降级展示，部分信息可能延迟。` | `刷新` -> `联系客服` | 只展示 warning，不切成功 icon | `已完成`、勾选成功动画 |
| `degraded=true` + `TICKET_SYNC_DEGRADED` | `当前服务记录同步中，不影响你继续查看。` | `刷新` -> `联系客服` | 保留已确认数据，弱化 warning | `全部同步成功` |
| `degraded=true` + `MINIAPP_ASSET_LEDGER_MISMATCH` | `资产汇总确认中，请稍后刷新。` | `刷新` -> `联系客服` | 仅资产总账区降级，不隐藏钱包/积分/券分区 | `到账成功`、庆祝动效 |

## 4. 页面落地检查清单
- [ ] 任一会员域错误态都能展示错误码。
- [ ] 任一会员域错误态都包含明确恢复动作。
- [ ] 降级场景不出现成功态文案、成功按钮文案、成功 icon 或成功动画。
- [ ] 客服话术与本映射一致，不再使用 message 自由发挥。
