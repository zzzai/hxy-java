# MiniApp 用户可见错误文案与恢复动作规范 v1（2026-03-09）

## 1. 文档目标
- 建立“错误码 / 页面结构态 / helper 行为 -> 用户可见结果 -> 恢复动作”单一映射，禁止 message 分支。
- 本批 booking 更新只承认当前真实页面、真实 helper、真实 smoke test 已证明的行为，不补尚未落到页面的失败文案。

## 2. 强约束
1. 前端分支条件必须是 `errorCode`、`code===0`、结构态或 helper 结果，不得解析 message。
2. 没有真实页面失败文案时，不得伪造“成功后又失败”的补文案，只能写当前可见结果。
3. `degraded=true` 场景禁止显示“成功”“已完成”等终态文案。
4. booking 当前六页不消费 `degraded` / `degradeReason`，不得在 booking 页面文案里补写假降级态。
5. 成功动效只能在后端成功态确认后展示；helper 未确认成功时，不得跳页、不得刷新、不得伪成功。

## 4. Booking Runtime 当前页面文案与恢复动作

### 4.1 页面结构态

| 页面 | 当前用户可见文案/控件 | 当前可见结果 | 恢复动作 |
|---|---|---|---|
| `/pages/booking/technician-list` | `暂无可用技师` | 技师列表为空时显示空态；失败时当前也只保留空结构，不额外补错文案 | 重新进入当前页 |
| `/pages/booking/technician-detail` | `该日期暂无可用时段`、`请选择时段`、`立即预约` | 时段为空时显示空态；未选时段时提交按钮不可点 | 切换日期重新拉取时段；或返回上一页重选技师 |
| `/pages/booking/order-confirm` | `提交预约` | 成功时跳详情；失败时停留确认页，停止 loading | 返回技师详情页改选时段后重提 |
| `/pages/booking/order-list` | `暂无预约`、`去支付`、`取消` | 查询为空时显示空态；取消成功后刷新；失败不刷新 | 下拉刷新、切换 tab、或重新进入当前页 |
| `/pages/booking/order-detail` | `订单不存在`、`去支付`、`取消预约`、`加钟` | 查单失败时当前可见结果是 `订单不存在`；取消失败不刷新 | 返回预约列表后重新进入；或稍后重试 |
| `/pages/booking/addon` | `订单不存在`、`确认加钟` | 母单不存在时显示空态；提交成功跳详情；失败不跳详情 | 重新选择 `addonType` 后再次提交；或返回预约详情页 |

### 4.2 写动作恢复规则

| 动作 | 当前 helper 行为 | 当前用户可见结果 | 恢复动作 | 严禁行为 |
|---|---|---|---|---|
| create | `submitBookingOrderAndGo` 仅 `code===0` 跳详情 | 失败时停留确认页 | 返回技师详情页改选时段，再次提交 | create 失败跳详情 |
| cancel | `cancelBookingOrderAndRefresh` 仅 `code===0` 刷新 | 失败时停留列表或详情页，不自动刷新 | 手动刷新或稍后重试 | cancel 失败自动刷新 |
| addon | `submitAddonOrderAndGo` 仅 `code===0` 跳详情 | 失败时停留加钟页 | 重新选择 `addonType` 或返回详情页 | addon 失败跳详情 |

### 4.3 Booking 当前稳定 errorCode / 结构态信号与页面结果

| 错误码 / 信号 | 真实页面场景 | 当前用户可见结果 | 恢复动作 | 备注 |
|---|---|---|---|---|
| `TIME_SLOT_NOT_AVAILABLE(1030003001)` | 确认页提交失败 | 当前页无独立失败文案；停留确认页 | 返回技师详情页改选时段 | create 失败不跳详情 |
| `BOOKING_ORDER_NOT_EXISTS(1030004000)` | cancel 提交失败 / add-on 提交失败 | 当前页无独立失败文案；停留当前页 | 返回预约列表或重新进入当前页 | 只适用于写链路；不适用于 `GET /booking/order/get` |
| `BOOKING_ORDER_STATUS_ERROR(1030004001)` | add-on 提交失败 | 当前页无独立失败文案；停留当前页 | 手动刷新、稍后重试、或返回上一页 | 当前只认 add-on 写链分支 |
| `BOOKING_ORDER_CANNOT_CANCEL(1030004005)` | cancel 提交失败 | 当前页无独立失败文案；停留列表或详情页 | 手动刷新、稍后重试、或返回上一页 | cancel 失败不刷新 |
| `BOOKING_ORDER_NOT_OWNER(1030004006)` | 详情查询越权 / cancel 越权 / add-on 越权 | 当前页无独立失败文案；停留当前页 | 返回预约列表或切换正确账号 | 只按 code 分支，不按 message |
| `success(null)` | `GET /booking/order/get` 缺记录；`GET /booking/technician/get` 缺记录 | 详情 / 加钟当前呈现 `订单不存在` 或空结构；技师详情当前只保留现有空结构 | 返回列表后重新进入，或回上一页重选技师 | 当前不是稳定 errorCode，也不是 `degraded` |

补充：
- booking 当前六页没有独立 toast/error banner 文案池。
- 因此 booking 侧“用户可见错误文案”当前必须写实为页面结构态和 helper 结果，不能虚构 toast。
- `SCHEDULE_CONFLICT(1030002001)`、`TIME_SLOT_ALREADY_BOOKED(1030003002)` 当前没有稳定 runtime 抛出证据，不得再写成 booking create 的当前页面分支。

## 5. 降级文案规范（degraded 不伪成功）

| 降级信号 | 用户文案（示例） | 恢复动作 | 禁止文案 |
|---|---|---|---|
| `degraded=true` + `PAY_ORDER_NOT_FOUND` | `支付结果确认中，请稍后刷新。` | `刷新`（最多3次） -> `查看订单` -> `联系客服` | `支付成功`、`已完成支付` |
| `degraded=true` + `TICKET_SYNC_DEGRADED` | `当前服务记录同步中，不影响主要操作。` | `继续当前流程` + `稍后查看进度` | `全部处理完成` |
| `degraded=true`（通用） | `当前为降级展示，部分信息可能延迟。` | `刷新` + `返回上页` + `联系客服` | `操作已完全成功` |

说明：
- 当前 booking 六页不消费以上降级信号。
- 若后续 booking 页面接入 `degraded`，必须先改页面实现，再改本文件。

## 6. RESERVED_DISABLED 错误码文案（若误返回）
- 适用：`1008009901/1008009902/1008009903/1004009901/1008009904/1011009901/1011009902/1013009901/1013009902/1030009901`
- 统一用户文案：`当前功能配置处理中，请稍后再试。`
- 恢复动作：`刷新` -> `联系客服`。
- 内部动作：按 P1 配置异常上报，不让用户承担定位成本。
