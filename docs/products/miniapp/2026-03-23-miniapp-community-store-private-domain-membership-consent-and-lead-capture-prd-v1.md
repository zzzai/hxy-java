# MiniApp 社区门店私域专题：入会授权与留资链路 PRD v1（2026-03-23）

## 0. 文档定位
- 目标：定义用户进入小程序后的留资、授权、会员建档与门店归属规则，为后续私域运营建立合法、可归因的用户底座。

## 1. 当前真值
- 已有：登录/注册、手机号授权、会员资料、等级页、标签页、资产页。
- 已有但不可放量：等级、标签、资产总账页处于 `Can Develop / Cannot Release`。
- 当前缺口：没有明确的“私域留资链路”文档，也没有统一的门店归属优先级规则。

## 2. 主流程
1. 用户进入小程序并授权手机号，建立 `memberId`。
2. 系统记录当前首触门店、首次来源渠道、首次意图。
3. 用户完成首单、首约、首券领取后，补充用户行为标签。
4. 用户进入后续私域链路前，必须先具备合法的会员身份与门店归属。

## 3. 关键字段

| 字段 | 作用 | 当前状态 |
|---|---|---|
| `memberId` | 会员主键 | 已有 |
| `mobile` | 私域触达前置条件 | 已有 |
| `storeId` | 门店归属 | 部分存在，但缺统一规则 |
| `firstChannel` | 首次来源归因 | 待补 |
| `firstTouchpoint` | 首次触发动作归因 | 待补 |
| `memberTag` | 后续运营分层 | 已有标签页/API 基础 |

## 4. 门店归属规则建议
1. 首次扫码进店页，优先绑定扫码 `storeId` 为首触门店。
2. 首次预约或首单门店，可作为服务门店。
3. 私域运营归属门店默认跟随首单/首约门店，若无交易再回退首触门店。
4. 一旦形成明确私域归属，后续变更需走人工校验或明确迁移规则，避免跨店抢客。

## 5. 缺口与开发建议
- 缺统一的用户留资事件结构。
- 缺门店归属优先级与变更规则。
- 缺“授权成功但未成交”的后续承接页面和标签。
- 建议先开发：
  1. 首触门店归因字段
  2. 用户留资事件结构
  3. 未成交用户的门店私域承接页

## 6. 验收口径
- [ ] 首次授权后可核到 `memberId/mobile/storeId/channel`。
- [ ] 首触门店、服务门店、私域归属门店规则一致且可解释。
- [ ] 标签/等级/资产页不被误写成“私域已闭环”，它们只是会员底座能力。
