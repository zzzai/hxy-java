# MiniApp 社区门店私域专题：店长/店员/企微运营 SOP 与分工 PRD v1（2026-03-23）

## 0. 文档定位
- 目标：定义社区门店私域运营中的角色、动作、频次和协同方式，让私域运营不依赖个别门店经验。

## 1. 角色分工建议

| 角色 | 核心职责 | 当前状态 |
|---|---|---|
| 总部运营 | 规则、活动、素材、指标、稽核 | 待形成私域专项 SOP |
| 店长 | 门店私域归因、社群经营、异常处理、复购目标 | 当前只有部分店长治理能力 |
| 店员/前台 | 扫码引导、福利讲解、预约转化、服务后引导 | 待文档化 |
| 企微运营/社群助手 | 承接加企微、发群通知、社群任务执行 | 待开发/待组织 |
| 客服 | 差评修复、售后兜底、敏感用户转人工 | 已有基础客服流程，但未接入私域 SOP |

## 2. 门店私域日常动作
1. 开店前：检查活动、券包、门店企微/社群承接配置。
2. 营业中：引导新客领券、授权、预约、入会员。
3. 服务后：引导评价、加企微/进社群、领复购券。
4. 闭店后：复盘当日新增会员、入私域人数、复购预约、差评修复。

## 3. 关键管理规则
- 私域运营不允许脱离门店归属做广泛群发。
- 差评、售后、低满意用户先修复后转化。
- 任何企微/社群动作都必须可追溯到门店与操作人。
- 店长 KPI 不应只看拉新，还要看沉淀率、复购率、修复率。

## 4. 开发依赖
- 门店配置中心
- 私域承接页与触点埋点
- 会员任务页
- 裂变活动配置
- 指标与归因看板

## 5. 验收口径
- [ ] 明确总部、店长、店员、社群运营、客服五类角色分工。
- [ ] 明确日/周/月私域运营动作节奏。
- [ ] 差评修复与私域转化不混为一条话术。
