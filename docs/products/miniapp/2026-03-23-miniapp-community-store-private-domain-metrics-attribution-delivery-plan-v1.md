# MiniApp 社区门店私域专题：指标体系、归因口径、项目推进计划 v1（2026-03-23）

## 0. 文档定位
- 目标：定义社区门店私域建设的指标口径、归因维度和开发推进顺序。

## 1. 指标体系

| 指标层级 | 核心指标 | 说明 |
|---|---|---|
| 获客 | 首触 UV、授权率、领券率、预约率 | 看公域到会员/交易的转化 |
| 沉淀 | 加企微率、入群率、门店归属完成率 | 看用户是否真正进入门店私域 |
| 经营 | 首单率、复购率、次月回访率、社群活动参与率 | 看私域经营效果 |
| 裂变 | 邀请率、裂变订单率、社群分享转化率 | 看用户带用户能力 |
| 修复 | 差评修复率、售后回访完成率、修复后复购率 | 看服务质量与私域修复能力 |

## 2. 归因口径
1. 归因最小单元：`memberId + storeId + channel + touchpoint + eventTime`。
2. 必须区分：
   - 首触门店
   - 服务门店
   - 私域归属门店
3. 渠道要区分：扫码、分享、社群、企微、自然访问、活动入口。
4. 评价、售后、复购、邀请都要保留 `sourceBizNo/runId` 便于追踪。

## 3. 推进顺序

| 阶段 | 目标 | 主要交付 |
|---|---|---|
| Phase 1 | 建底座 | 门店归因、用户留资、公转私触点、9 份文档包 |
| Phase 2 | 建承接 | 企微/社群承接页、任务体系、门店 SOP |
| Phase 3 | 建增长 | 裂变活动、社群活动、门店专属福利、数据看板 |
| Phase 4 | 建规模 | 自动化运营、门店考核、区域复制能力 |

## 4. 当前项目判断
- 当前私域建设仍在 Phase 1 文档与基础能力整理阶段。
- 项目已有部分基础能力，但还不能说“社区门店私域已闭环”。
- 指标体系的首要价值是避免后续开发没有分母、没有门店归因、没有复盘口径。

## 5. 验收口径
- [ ] 指标体系覆盖获客、沉淀、经营、裂变、修复五层。
- [ ] 归因口径能区分首触门店、服务门店、私域归属门店。
- [ ] 推进计划能直接映射到后续研发专题。
