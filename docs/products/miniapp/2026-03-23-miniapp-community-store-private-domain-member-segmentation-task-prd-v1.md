# MiniApp 社区门店私域专题：会员分层、标签、任务触达 PRD v1（2026-03-23）

## 0. 文档定位
- 目标：把会员标签、等级、资产、交易行为转成可执行的私域运营分层和任务机制。

## 1. 当前真值
- 已有基础能力：等级页、标签页、资产页、签到、积分、钱包、优惠券。
- 当前仍不可放量：等级、标签、统一资产台账处于 `Can Develop / Cannot Release`。
- 当前缺口：没有把这些会员数据真正转成“门店私域任务体系”。

## 2. 分层建议

| 维度 | 示例标签 | 当前基础 |
|---|---|---|
| 交易频次 | 新客 / 首单客 / 复购客 / 沉默客 | 可基于订单链补 |
| 服务偏好 | 推拿 / 足疗 / 理疗 / 次卡 | 可基于商品与预约链补 |
| 门店归属 | 首触门店 / 服务门店 / 私域归属门店 | 待统一规则 |
| 资产能力 | 高积分 / 高券余额 / 次卡剩余 | 资产页已有基础 |
| 满意度 | 高满意 / 中性 / 待修复 | 可结合评价与售后 |

## 3. 任务体系建议
1. 新客 3 日任务：授权、领券、首约。
2. 首单后 7 日任务：评价、复购券、加企微/进社群。
3. 沉默客唤醒任务：门店专属券、恢复包、活动通知。
4. 高满意用户裂变任务：邀请有礼、拼团、社群分享。
5. 低满意用户修复任务：客服回访、门店店长跟进、补偿券。

## 4. 当前依赖
- 会员标签页/API
- 会员等级页/API
- 资产总账与钱包/券/积分
- 订单、预约、评价、售后

## 5. 开发建议
- P1：先做“任务规则文档 + 标签输入口径 + 用户任务页”。
- P1：门店维度任务必须能按 `storeId` 下钻。
- P2：再做自动化编排与群/企微联动。

## 6. 验收口径
- [ ] 至少定义 5 类私域运营分层。
- [ ] 至少形成新客、复购、沉默、裂变、修复五类任务模板。
- [ ] 标签/等级/资产不再只是展示页，而能映射到任务规则。
