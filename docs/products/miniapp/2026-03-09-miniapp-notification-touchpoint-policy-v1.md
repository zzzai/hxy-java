# MiniApp 通知触达策略 v1（2026-03-09）

## 1. 文档目标与范围
- 目标：统一小程序通知触达策略，覆盖触发、频控、静默、退订、失败补发与错误码分流。
- 范围：站内消息 + 小程序订阅消息 + 短信兜底（仅关键场景）。
- 对齐基线：
  - `docs/contracts/2026-03-08-miniapp-errorcode-recovery-matrix-v1.md`

## 2. 触发策略（核心触点）

| 触发场景 | 触发条件 | 通知渠道 | 默认时机 | 目标 |
|---|---|---|---|---|
| 首单转化 | 首单支付成功 | 站内 + 订阅消息 | 即时 | 引导预约与到店 |
| 预约提醒 | 预约开始前 T-24h / T-2h | 订阅消息 + 站内 | 定时 | 降低爽约 |
| 履约完成 | 服务完成 | 站内 | 即时 | 引导评价/售后 |
| 售后状态变更 | 售后状态切换（审核/退款中/成功） | 站内 + 订阅消息 | 即时 | 降低客服咨询 |
| 券/积分召回 | 可用券到期前、积分活动上线 | 站内（高频）+ 订阅（低频） | 定时 | 拉动复购 |
| 异常提醒 | 需要用户操作的业务异常 | 站内优先 | 即时/延迟 | 引导恢复动作 |

## 3. 频控策略
- 用户级频控：
  - 营销通知：每用户每天最多 2 条；每周最多 8 条。
  - 交易/服务通知：不受营销频控，但同事件最多重发 2 次。
- 场景级频控：
  - 同一订单的支付结果通知最多 3 次（含补发）。
  - 同一售后单状态通知仅在状态变化时发送，禁止重复推送。
- 去重键：`eventType + bizId + status + channel`。

## 4. 静默时段与退订策略
- 静默时段：每日 `22:00 - 次日 08:00`。
- 静默规则：
  - 营销触达延迟到次日 08:30 后发送。
  - 交易关键通知（支付结果、预约提醒、退款结果）可越过静默，但需降频到单事件单次。
- 用户退订：
  - 支持按渠道退订（订阅消息/营销短信）。
  - 退订后仅保留“交易必要通知”（订单、预约、售后状态）。
  - 退订状态需实时生效，且 5 分钟内同步到通知网关。

## 5. 失败补发策略
- 发送失败重试：指数退避 `1m -> 5m -> 30m`，最多 3 次。
- 超过 3 次失败：
  - 交易类通知转客服待跟进清单。
  - 营销类通知直接终止，不再补发。
- 补发审计字段：`runId/orderId/afterSaleId/sourceBizNo/errorCode/channel/degraded`。

## 6. 错误码分流通知策略（提醒/不提醒/延迟提醒）

| 错误码/信号 | 用户通知策略 | 渠道 | 策略说明 |
|---|---|---|---|
| `ORDER_NOT_FOUND(1011000011)` | 提醒 | 站内即时 | 引导返回订单列表并重试 |
| `AFTER_SALE_NOT_FOUND(1011000100)` | 提醒 | 站内即时 | 引导返回售后列表刷新 |
| `AFTER_SALE_REFUND_FAIL_BUNDLE_CHILD_FULFILLED(1011000125)` | 提醒（高优先） | 站内 + 客服接入提示 | 需人工审核，给出客服入口 |
| `BOOKING_ORDER_REFUND_IDEMPOTENT_CONFLICT(1030004012)` | 不提醒用户（转人工） | 客服工单通知 | 防止用户误操作，直接由客服接管 |
| `BOOKING_ORDER_REFUND_REPLAY_RUN_ID_NOT_EXISTS(1030004016)` | 不提醒用户（后台告警） | 运维/运营告警 | 属运营审计问题，不直接打扰用户 |
| `PRICE_CALCULATE_POINT_TOTAL_LIMIT_COUNT(1011003004)` | 提醒 | 站内即时 | 解释规则限制并引导调整 |
| `PROMOTION_COUPON_*` | 提醒 | 站内即时 | 领取失败给可恢复建议 |
| `PAY_ORDER_NOT_FOUND`（degradeReason） | 延迟提醒 | 站内延迟 2 分钟 + 手动刷新引导 | 先给用户等待窗口，避免误报失败 |
| `TICKET_SYNC_DEGRADED` | 不提醒用户（仅内部提醒） | 运营/客服告警 | 主链路成功，用户无感，内部补偿 |
| `TECHNICIAN_NOT_EXISTS(1030001000)` / `TECHNICIAN_DISABLED(1030001001)` | 提醒 | 站内即时 | 引导改约/改选技师 |

## 7. Owner 与 SLA
| 能力 | Owner | SLA |
|---|---|---|
| 通知策略配置 | CRM 运营负责人 | 配置变更 1 小时内生效 |
| 通知通道稳定性 | 消息平台负责人 | 日发送成功率 >= 98% |
| 失败补发任务 | 运营值班 + 平台 on-call | 失败事件 30 分钟内进入补发队列 |
| 退订与静默策略 | 用户运营负责人 | 退订同步 <= 5 分钟 |
| 错误码分流口径 | 产品经理（交易域） | 口径变更需 T+1 同步客服话术 |

## 8. 发布验收检查项
- [ ] 频控与静默策略可在预发环境演练通过。
- [ ] 错误码分流策略与客服 SOP 一致，无冲突口径。
- [ ] 失败补发链路有可观测日志并可追踪重试次数。
- [ ] 用户退订后不再收到营销消息，但保留交易必要通知。
