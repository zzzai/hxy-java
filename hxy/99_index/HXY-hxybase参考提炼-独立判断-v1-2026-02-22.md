# HXY hxybase 参考提炼（独立判断）V1

- 参考源：`/root/crmeb-java/hxybase`
- 定位：作为“探索资料库”参考，不直接照搬为实施方案。

## 1. 采纳的方向

1. 连锁模型：总部统一治理 + 门店标准履约。
2. 交易形态：服务+实物混合的 O2O 业务结构。
3. 支付原则：总部资金归口、门店不直接操作渠道资金。
4. 数据方向：用户数据与经营数据应分层治理。

## 2. 不直接采纳的部分

1. 以文档驱动的“大而全功能清单”不直接进入开发计划。
2. 对商业扩张/收购策略类内容，不作为技术实施输入。
3. 未验证依赖与假设（组织、人力、渠道条件）不直接落地。

## 3. 独立思考后的落地策略

1. 以当前代码事实为准：RuoYi 主线 + 支付优先收敛。
2. 以可验证工程能力为准：上线拦截、回滚预案、异常补偿优先。
3. 以万店治理为准：K8s 微服务治理作为目标架构，分阶段迁移。
4. 以权责利一致为准：总部与门店权责边界先固化再扩功能。
