# 部署指南

## 环境准备

### 1. 服务器要求

**最低配置**：
- CPU: 2核
- 内存: 4GB
- 磁盘: 50GB
- 操作系统: CentOS 7+ / Ubuntu 18.04+

**推荐配置**：
- CPU: 4核
- 内存: 8GB
- 磁盘: 100GB SSD

### 2. 软件安装

#### 安装JDK 1.8

```bash
# CentOS
yum install java-1.8.0-openjdk-devel

# Ubuntu
apt-get install openjdk-8-jdk

# 验证
java -version
```

#### 安装MySQL 5.7

```bash
# CentOS
wget https://dev.mysql.com/get/mysql57-community-release-el7-11.noarch.rpm
rpm -ivh mysql57-community-release-el7-11.noarch.rpm
yum install mysql-server

# 启动
systemctl start mysqld
systemctl enable mysqld

# 获取临时密码
grep 'temporary password' /var/log/mysqld.log

# 修改密码
mysql -u root -p
ALTER USER 'root'@'localhost' IDENTIFIED BY 'YourPassword123!';
```

#### 安装Redis

```bash
# CentOS
yum install redis

# Ubuntu
apt-get install redis-server

# 启动
systemctl start redis
systemctl enable redis

# 验证
redis-cli ping
```

#### 安装Nginx

```bash
# CentOS
yum install nginx

# Ubuntu
apt-get install nginx

# 启动
systemctl start nginx
systemctl enable nginx
```

## 应用部署

### 1. 配置文件

编辑 `application-prod.yml`：

```yaml
server:
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/crmeb_java?useUnicode=true&characterEncoding=utf-8&useSSL=false
    username: root
    password: YourPassword123!
    driver-class-name: com.mysql.cj.jdbc.Driver
    hikari:
      maximum-pool-size: 20
      minimum-idle: 5
      connection-timeout: 30000
      idle-timeout: 600000
      max-lifetime: 1800000
  
  redis:
    host: localhost
    port: 6379
    database: 0
    password: 
    timeout: 3000
    lettuce:
      pool:
        max-active: 8
        max-wait: -1
        max-idle: 8
        min-idle: 0

logging:
  level:
    root: INFO
    com.zbkj: DEBUG
  file:
    name: /var/log/crmeb/application.log
    max-size: 100MB
    max-history: 30
```

### 2. 打包应用

```bash
cd /root/crmeb-java/crmeb_java/crmeb

# 清理并打包
mvn clean package -DskipTests -Pprod

# 查看生成的jar包
ls -lh crmeb-admin/target/crmeb-admin.jar
```

### 3. 启动应用

#### 方式1：直接运行

```bash
java -jar crmeb-admin/target/crmeb-admin.jar \
  --spring.profiles.active=prod
```

#### 方式2：后台运行

```bash
nohup java -jar crmeb-admin/target/crmeb-admin.jar \
  --spring.profiles.active=prod \
  > /var/log/crmeb/app.log 2>&1 &

# 查看进程
ps aux | grep crmeb-admin

# 查看日志
tail -f /var/log/crmeb/app.log
```

#### 方式3：systemd服务

创建服务文件 `/etc/systemd/system/crmeb.service`：

```ini
[Unit]
Description=CRMEB Booking System
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/root/crmeb-java/crmeb_java/crmeb
ExecStart=/usr/bin/java -jar /root/crmeb-java/crmeb_java/crmeb/crmeb-admin/target/crmeb-admin.jar --spring.profiles.active=prod
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
# 重载配置
systemctl daemon-reload

# 启动服务
systemctl start crmeb

# 开机自启
systemctl enable crmeb

# 查看状态
systemctl status crmeb

# 查看日志
journalctl -u crmeb -f
```

## Nginx配置

### 1. 反向代理配置

创建配置文件 `/etc/nginx/conf.d/crmeb.conf`：

```nginx
upstream crmeb_backend {
    server 127.0.0.1:8080;
}

server {
    listen 80;
    server_name your-domain.com;
    
    # 日志
    access_log /var/log/nginx/crmeb_access.log;
    error_log /var/log/nginx/crmeb_error.log;
    
    # API代理
    location /api/ {
        proxy_pass http://crmeb_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }
    
    # 静态文件
    location /static/ {
        alias /root/crmeb-java/static/;
        expires 30d;
    }
    
    # 前端页面
    location / {
        root /root/crmeb-java/frontend/dist;
        try_files $uri $uri/ /index.html;
    }
}
```

### 2. HTTPS配置（可选）

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;
    
    ssl_certificate /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    
    # 其他配置同上
}

# HTTP重定向到HTTPS
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}
```

### 3. 重启Nginx

```bash
# 测试配置
nginx -t

# 重启
systemctl restart nginx
```

## 备份策略

### 1. 数据库备份

创建备份脚本 `/root/scripts/backup_mysql.sh`：

```bash
#!/bin/bash

BACKUP_DIR="/backup/mysql"
DATE=$(date +%Y%m%d_%H%M%S)
MYSQL_USER="root"
MYSQL_PASS="YourPassword123!"
DATABASE="crmeb_java"

mkdir -p $BACKUP_DIR

mysqldump -u$MYSQL_USER -p$MYSQL_PASS $DATABASE | gzip > $BACKUP_DIR/${DATABASE}_${DATE}.sql.gz

# 删除7天前的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +7 -delete

echo "Backup completed: ${DATABASE}_${DATE}.sql.gz"
```

添加定时任务：

```bash
crontab -e

# 每天凌晨2点备份
0 2 * * * /root/scripts/backup_mysql.sh
```

### 2. Redis备份

```bash
#!/bin/bash

BACKUP_DIR="/backup/redis"
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

cp /var/lib/redis/dump.rdb $BACKUP_DIR/dump_${DATE}.rdb

# 删除7天前的备份
find $BACKUP_DIR -name "dump_*.rdb" -mtime +7 -delete
```

## 安全加固

### 1. 防火墙配置

```bash
# 开放端口
firewall-cmd --permanent --add-port=80/tcp
firewall-cmd --permanent --add-port=443/tcp
firewall-cmd --reload

# 限制MySQL只允许本地访问
firewall-cmd --permanent --add-rich-rule='rule family="ipv4" source address="127.0.0.1" port protocol="tcp" port="3306" accept'
```

### 2. 数据库安全

```sql
-- 删除匿名用户
DELETE FROM mysql.user WHERE User='';

-- 禁止root远程登录
DELETE FROM mysql.user WHERE User='root' AND Host NOT IN ('localhost', '127.0.0.1', '::1');

-- 刷新权限
FLUSH PRIVILEGES;
```

- 修改默认端口
- 启用HTTPS
- 配置访问白名单
