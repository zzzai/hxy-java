## 步骤1：执行数据库脚本

### 方式A：命令行执行（推荐）

```bash
# 1. 登录MySQL
mysql -u root -p

# 2. 选择数据库
USE crmeb_java;

# 3. 执行脚本
source /root/crmeb-java/hxy/database/database_migration_v1.0.sql;

# 4. 验证表是否创建
SHOW TABLES LIKE 'eb_%';

# 5. 查看技师表结构
DESC eb_technician;

# 6. 查看排班表结构
DESC eb_technician_schedule;

# 7. 退出
EXIT;
```

### 方式B：使用脚本执行

```bash
# 给脚本执行权限
chmod +x /root/crmeb-java/hxy/tools/scripts/init_database.sh

# 编辑脚本，填写MySQL密码
vi /root/crmeb-java/hxy/tools/scripts/init_database.sh
# 修改: DB_PASS="your_password"

# 执行脚本
/root/crmeb-java/hxy/tools/scripts/init_database.sh
```

### 验证结果

```sql
-- 检查新增表（应该有8个）
SELECT COUNT(*) FROM information_schema.tables 
WHERE table_schema = 'crmeb_java' 
AND table_name IN (
    'eb_technician',
    'eb_technician_schedule',
    'eb_booking_order',
    'eb_member_card',
    'eb_user_member_card',
    'eb_member_card_usage',
    'eb_stock_flow',
    'eb_offpeak_rule'
);

-- 检查库存字段扩展
DESC eb_store_product_sku;
-- 应该看到: available_stock, locked_stock, sold_stock
```

## 步骤2：配置Redis

### 方式A：检查Redis状态

```bash
# 1. 检查Redis是否安装
redis-server --version

# 2. 检查Redis是否运行
systemctl status redis

# 3. 如果未运行，启动Redis
systemctl start redis
systemctl enable redis

# 4. 测试连接
redis-cli ping
# 应该返回: PONG
```

### 方式B：配置application.yml

编辑文件：`/root/crmeb-java/crmeb_java/crmeb/crmeb-admin/src/main/resources/application.yml`

添加或修改Redis配置：

```yaml
spring:
  redis:
    host: localhost
    port: 6379
    database: 0
    password:   # 如果有密码，填写在这里
    timeout: 3000
    lettuce:
      pool:
        max-active: 8
        max-wait: -1
        max-idle: 8
        min-idle: 0
```

### 验证Redis

```bash
# 测试写入
redis-cli SET test_key "test_value"

# 测试读取
redis-cli GET test_key

# 删除测试数据
redis-cli DEL test_key

# 查看Redis信息
redis-cli INFO server
```

## 步骤3：启动应用

### 方式A：Maven启动（开发环境）

```bash
# 进入项目目录
cd /root/crmeb-java/crmeb_java/crmeb

# 清理并编译
mvn clean install -DskipTests

# 启动应用
mvn spring-boot:run

# 或者指定模块启动
cd crmeb-admin
mvn spring-boot:run
```

### 方式B：Jar包启动（生产环境）

```bash
# 打包
cd /root/crmeb-java/crmeb_java/crmeb
mvn clean package -DskipTests

# 启动
java -jar crmeb-admin/target/crmeb-admin.jar

# 后台启动
nohup java -jar crmeb-admin/target/crmeb-admin.jar > /var/log/crmeb/app.log 2>&1 &
```

### 验证启动

```bash
# 1. 检查进程
ps aux | grep crmeb

# 2. 检查端口
netstat -tlnp | grep 8080

# 3. 测试健康检查
curl http://localhost:8080/actuator/health

# 4. 访问Swagger文档
curl http://localhost:8080/swagger-ui.html
```

## 步骤4：运行集成测试

### 方式A：单元测试

```bash
cd /root/crmeb-java/crmeb_java/crmeb

# 运行所有测试
mvn test

# 运行指定测试
mvn test -Dtest=DistributedLockUtilTest
```

### 方式B：API测试

```bash

# 2. 测试排班列表
curl http://localhost:8080/api/admin/schedule/store?storeId=1&startDate=2026-02-14&endDate=2026-02-20

# 3. 测试健康检查
curl http://localhost:8080/actuator/health

# 4. 测试指标
curl http://localhost:8080/actuator/metrics
```

### 方式C：压力测试

```bash
# 安装ab工具
yum install httpd-tools

# 执行压力测试（100并发，1000请求）
ab -n 1000 -c 100 http://localhost:8080/api/admin/technician/list?storeId=1

# 查看结果
# - Requests per second: 每秒请求数
# - Time per request: 平均响应时间
# - Failed requests: 失败请求数
```

## 步骤5：验证完整流程

### 完整预约流程测试

```bash
# 1. 创建技师
curl -X POST http://localhost:8080/api/admin/technician/add \
  -H "Content-Type: application/json" \
  -d '{
    "storeId": 1,
    "name": "张技师",
    "level": 3,
    "serviceYears": 5.0,
    "status": 1
  }'

# 2. 生成排班
curl -X POST http://localhost:8080/api/admin/schedule/generate \
  -H "Content-Type: application/json" \
  -d '{
    "storeId": 1,
    "technicianId": 1,
    "serviceSkuId": 1,
    "workDate": "2026-02-14",
    "startTime": "09:00",
    "endTime": "21:00",
    "slotDuration": 60,
    "basePrice": 128.0
  }'

# 3. 预约时间槽
curl -X POST http://localhost:8080/api/admin/booking/book \
  -d "userId=1&scheduleId=1&slotId=20260214_0900"

# 4. 创建订单
curl -X POST http://localhost:8080/api/admin/booking-order/create \
  -d "userId=1&storeId=1&technicianId=1&scheduleId=1&slotId=20260214_0900&serviceSkuId=1"

# 5. 支付订单
curl -X POST http://localhost:8080/api/admin/booking-order/pay \
  -d "orderNo=BO17394048001234&payType=1"

# 6. 核销订单
curl -X POST http://localhost:8080/api/admin/booking-order/verify \
  -d "verifyCode=123456&verifiedBy=1"
```

## 常见问题排查

### 问题1：数据库连接失败

```bash
# 检查MySQL状态
systemctl status mysqld

# 检查端口
netstat -tlnp | grep 3306

# 测试连接
mysql -h localhost -u root -p

# 查看错误日志
tail -f /var/log/mysqld.log
```

### 问题2：Redis连接失败

```bash
# 检查Redis状态
systemctl status redis

# 检查端口
netstat -tlnp | grep 6379

# 测试连接
redis-cli ping

# 查看日志
tail -f /var/log/redis/redis.log
```

### 问题3：应用启动失败

```bash
# 查看日志
tail -f /var/log/crmeb/app.log

# 检查端口占用
netstat -tlnp | grep 8080

# 检查Java进程
ps aux | grep java

# 查看JVM内存
jmap -heap <pid>
```

### 问题4：API返回错误

```bash
# 查看应用日志
tail -f logs/crmeb.log

# 查看错误日志
tail -f logs/error.log

# 检查数据库连接
mysql -u root -p -e "SHOW PROCESSLIST;"

# 检查Redis连接
redis-cli INFO clients
```
