# 支付系统上线拦截规则一键命令模板

- 日期：2026-02-18
- 目标：在当前环境快速复用“值守刷新 + 上线拦截规则 + 实单演练”命令
- 适用环境：`/root/crmeb-java/crmeb_java/crmeb`，Docker MySQL 端口 `33306`

## 0. 统一环境变量

```bash
cd /root/crmeb-java/crmeb_java/crmeb
export DB_PORT=33306
export MYSQL_DEFAULTS_FILE="$HOME/.my.cnf"
export ALLOW_SHARED_SUBMCHID=1
export BIZ_DATE="$(date -d 'yesterday' +%F)"
```

## 1. 日常值守刷新（无实单也可跑）

```bash
./shell/payment_ops_status.sh \
  --date "$BIZ_DATE" \
  --refresh \
  --refresh-window-hours 72 \
  --refund-window-hours 72 \
  --refund-timeout-minutes 30 \
  --require-booking-repair-pass 0 \
  --require-decision-chain-pass 1 \
  --no-alert
```

## 2. 无实单阶段：上线拦截规则（目标 GO_FOR_ORDER_DRILL）

```bash
./shell/payment_cutover_gate.sh \
  --date "$BIZ_DATE" \
  --require-apply-ready 0 \
  --require-booking-repair-pass 0 \
  --require-mapping-smoke-green 0 \
  --require-mapping-green 0 \
  --require-mock-green 0 \
  --require-refund-green 0 \
  --allow-shared-submchid 1 \
  --refund-window-hours 72 \
  --refund-timeout-minutes 30 \
  --no-alert
```

## 3. 有实单阶段：上线拦截规则（目标 GO）

将 `<ORDER_NO>` 替换为真实商户单号（例如 `wxNo89809177138520395483906`）：

```bash
./shell/payment_cutover_gate.sh \
  --date "$BIZ_DATE" \
  --order-no <ORDER_NO> \
  --require-apply-ready 0 \
  --require-booking-repair-pass 0 \
  --require-mapping-smoke-green 0 \
  --require-mapping-green 0 \
  --require-mock-green 0 \
  --require-refund-green 0 \
  --allow-shared-submchid 1 \
  --refund-window-hours 72 \
  --refund-timeout-minutes 30 \
  --no-alert
```

## 4. 严格发布门槛（发布前切换）

```bash
./shell/payment_cutover_gate.sh \
  --date "$BIZ_DATE" \
  --order-no <ORDER_NO> \
  --require-apply-ready 1 \
  --require-booking-repair-pass 1 \
  --require-mapping-smoke-green 1 \
  --require-mapping-green 1 \
  --require-mock-green 1 \
  --require-refund-green 1 \
  --mapping-strict-missing 1 \
  --allow-shared-submchid 0 \
  --preflight-strict 1 \
  --refund-window-hours 72 \
  --refund-timeout-minutes 30 \
  --no-alert
```

## 5. 实单联调最小链路

```bash
./shell/payment_order_locator.sh --order-no <ORDER_NO>
./shell/payment_fullchain_drill.sh --order-no <ORDER_NO>
./shell/payment_refund_convergence_check.sh \
  --order-no <ORDER_NO> \
  --window-hours 72 \
  --refund-timeout-minutes 30 \
  --no-alert
```

## 6. 结果判定

- `payment_cutover_gate`：
  - `gate_decision=GO`：可上线
  - `gate_decision=GO_WITH_WARN`：可带告警推进
  - `gate_decision=NO_GO`：阻断，先清零 BLOCK
- `payment_ops_status`：
  - `overall=GREEN`：值守正常
  - `overall=YELLOW/RED`：需要先修复再切换
