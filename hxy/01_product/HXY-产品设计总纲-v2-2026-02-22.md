# HXY 产品设计总纲 V2（2026-02-22）

## 1. 产品定位

1. 荷小悦不是单一商城，而是“连锁门店服务履约 + 小程序交易 + 总部运营治理”的 O2O 系统。
2. 核心价值：将线下体验、线上交易、售后与资金闭环统一到一套产品体系。

## 2. 用户与角色

1. C 端用户：浏览、下单、支付、退款申请、进度查询。
2. 门店角色：商品与服务配置、接单履约、上传售后证据。
3. 总部运营：策略配置、订单与售后治理、指标看板。
4. 财务与风控：退款策略、风险拦截、对账差异处理。

## 3. 核心业务链路

1. 交易链路：下单 -> 支付 -> 回调确认 -> 履约 -> 完结/售后。
2. 退款链路：申请 -> 规则分流（自动/人工）-> 渠道处理 -> 终态收敛。
3. 账务链路：交易账 -> 渠道账 -> 差异识别 -> 工单闭环。

## 4. 功能分层

1. P0（必须先完成）
   - 支付与退款闭环可收敛。
   - 异常场景可补偿（回调丢失、乱序、重复、超时）。
   - 上线拦截规则生效。
2. P1（P0 稳定后）
   - 运营后台效率提升（工单、告警、审计检索）。
   - 商品与会员能力二开。
3. P2
   - AI 辅助运营、门店精细化经营、数据增长策略。

## 5. 产品规则（关键）

1. 正常退款尽量秒退，疑似风险单走总部审核。
2. 门店仅负责履约，退款执行权在总部。
3. 支付终态与交易终态必须一致，禁止“前端结果即最终状态”。

## 6. KPI 指标

1. 支付成功率、回调收敛时延 P95。
2. 退款终态收敛率、平均收敛时长。
3. 对账差异率、差异工单闭环时长。
4. 发布后 P0 故障次数。
