1. 补齐总部公司与门店运行中的全角色覆盖（含客服、供应链、加盟管理、审计合规、运维）。
2. 补齐结算审批链与SLA（发起、复核、审批、打款、冲正、超时升级）。
3. 补齐售后工单体系（退款、履约争议、提成争议统一入口）。
4. 补齐实物供应链流程（补货、调拨、盘点、损耗、报废）。

## 1. 产品定位

### 1.1 品牌定位
荷小悦不是“纯预约按摩店”，也不是“纯商城”，而是社区场景下的疗愈生活服务品牌：

1. 到店即时疗愈：解决当下疲劳、亚健康、情绪压力。
2. 居家延续护理：通过实物商品和方案延续疗愈效果。
3. 连锁标准化复制：总部定义标准，门店灵活运营，合规可审计。

### 1.2 产品定位
构建“服务项目（技师时间）+ 实物商品（库存）+ 套餐组合”的混合 O2O 多门店平台，形成一体化交易、履约、结算、增长闭环。

### 1.3 北极星目标
1. 单店模型跑通：门店单日稳定完成服务履约+实物销售+退款闭环。
2. 复制效率：新店 7 天内完成开店、上架、支付、履约、对账上线。
3. 连锁可控：总部可统一管理价格边界、服务SOP、资金结算与风险。

### 1.4 设计原则（补充）
1. 模板与实例解耦：总部管标准主数据，门店管运营实例，不越权。
2. 规则外置：权限边界、价格区间、是否需审批由规则配置驱动，不硬编码。
3. 事件解耦：总部变更通过事件下发，门店异步消费，失败可重试可补偿。
4. 快照不可变：订单项冻结价格/规则/版本快照，历史单不受后续变更影响。
5. 服务与实物分离计算：服务走排班容量，实物走库存容量，套餐按子项拆解。
6. 数据强隔离：门店级数据必须带 store_id 且默认行级隔离。
7. 兜底可回退：门店未配置时可回落总部指导配置，保障业务连续性。

## 2. 用户与角色

### 2.1 外部用户
1. C端用户：预约服务、购买实物、购买套餐、退款、复购。

### 2.2 内部角色
1. 总部运营：类目、SPU/SKU、活动、价盘、审核与风控。
2. 门店店长：门店商品映射、排班、上下架、活动执行。
3. 技师：接单、服务履约、加项推荐、服务完成确认。
4. 财务：分账对账、提成结算、退款稽核。

### 2.3 组织角色覆盖（运行态）
1. 总部品牌商品运营：管理品牌标准、类目模板、SPU/SKU、套餐编排。
2. 总部价格与活动运营：维护总部价盘、门店标签策略、活动冲突处理。
3. 总部加盟与门店管理：门店准入、合约策略、生命周期治理。
4. 区域督导（线下职能）：负责门店训练、巡检、突发事件协同处置，不持有线上配置主权。
5. 门店店长：负责门店经营指标、人员排班、异常处理、复盘。
6. 门店前台/收银：到店核销、改约、现场售后受理、票据交接。
7. 技师：履约、加项、服务确认、异常上报。
8. 客服：售前咨询、售后工单、争议协调、超时升级。
9. 供应链/仓配：补货、调拨、盘点、损耗报备。
10. 财务结算：分账核对、提成结算、月结冲正、审计对账。
11. 合规审计：规则审计、留痕稽核、高风险复核。
12. IT运维：发布、监控、故障处置、回滚执行。

### 2.4 关键职责边界（总部公司 vs 门店）
1. 总部公司负责标准与边界：商品模板、规则、价格区间、结算口径、合规审计。
2. 门店负责执行与反馈：门店上架、排班、履约、现场服务质量、库存实操。
3. 区域督导仅负责线下经营承接：训练、巡检、异常协同，不新增线上配置层。
4. 客服与财务作为跨组织公共能力：统一口径，避免门店各自为政。

### 2.5 线上权限层级约束
1. 线上系统配置管理仅保留两级：总部、门店。
2. 任何模板、价盘、结算、规则配置只允许总部落库。
3. 门店仅允许在总部边界内执行：上下架、价格区间微调、排班与履约操作。
4. 线下岗位（区域督导、训练师、巡检员）通过总部工单与台账协同，不新增线上角色层级。

### 2.6 AI 辅助与人工门禁边界
1. AI 仅用于建议、草稿、预警、分流，不得直接执行高风险动作。
2. 人工强门禁清单：结算打款、大额退款、合规驳回、闭店生效、高风险工单仲裁。
3. 所有高风险动作必须记录“AI建议+人工审批人+审批依据+审批时间”审计链。
4. AI 服务不可用时，自动回退到规则引擎+人工审批，不得阻断主链路。

## 3. 核心业务模型

### 3.1 商品域模型
1. 服务商品 SPU/SKU：核心容量是“技师可服务时间”。
2. 实物商品 SPU/SKU：核心容量是“库存数量”。
3. 套餐商品 SPU：由服务SKU和实物SKU子项组成，按子项核销与结算。

### 3.2 门店映射模型
1. 总部维护标准商品主数据。
2. 门店映射并配置本店可售 SKU（价格、库存、上下架、排序）。
3. 门店未配置时，可按总部兜底规则执行。

### 3.3 订单模型
1. 纯服务订单。
2. 纯实物订单。
3. 混合订单（服务+实物同单）。
4. 套餐订单（下单即拆子项，按子项履约与退款）。

### 3.4 治理与控制模型
1. 双开关可售：总部启用 + 门店上架同时满足才可售。
2. 合约策略：门店挂载合约类型（直营/标准加盟/高级加盟等），决定可操作边界。
3. 规则引擎：对调价、库存、上下架、活动叠加进行实时校验并返回明确失败原因。
4. 版本化模板：类目模板、套餐子项、价盘规则均需版本号和生效时间，支持灰度切换。

## 4. 业务闭环设计

### 4.1 下单与价格计算
统一价格决策顺序：

1. 总部基准价。
2. 门店价覆盖。
3. 活动价覆盖（秒杀/折扣/时段价）。
4. 会员权益与券/积分抵扣。

每笔订单项记录价格来源快照（用于对账、退款、风控）。

### 4.2 履约分流
1. 服务商品：进入服务履约单，核心状态为待预约/已预约/服务中/已完成/已取消。
2. 实物商品：进入库存履约链路，锁库存、扣库存、逆向回补。
3. 套餐商品：拆分子项后分别进入服务履约和实物履约。

### 4.3 退款与逆向
1. 纯服务：按履约状态控制可退范围。
2. 纯实物：按发货状态、签收状态控制可退范围。
3. 套餐/混合：按子项退款，已核销服务子项不可退，未核销子项按分摊金额退款。

### 4.4 库存分层与抢购策略（实物）
1. 总部中央库存、门店库存、线上可售库存分层管理。
2. 抢购/平台活动优先扣中央库存，门店线上库存用于日常售卖。
3. 线上可售库存与门店实物库存分离，防止线上超卖与线下截留冲突。

### 4.5 闲时优惠与排班解耦（服务）
1. 技师排班仅描述可服务时间容量，不承载优惠规则。
2. 闲时优惠独立建模，查询可预约时段时再合并计算展示价。
3. 排班调整不应改变优惠规则，优惠调整不应破坏排班数据。

### 4.6 订单快照冻结清单
1. 价格快照：总部基准价、门店价、活动价、会员价、成交价、价格来源类型。
2. 规则快照：命中规则ID、合约类型、审批状态。
3. 版本快照：模板版本、SKU版本、套餐子项快照JSON。
4. 履约快照：服务项/实物项标识、核销状态、核销人、核销时间。

### 4.7 售后与客服工单闭环
1. 售后入口统一工单化：退款申请、履约争议、提成争议统一受理。
2. 工单分级：普通/紧急/高风险（金额、投诉等级、舆情风险综合判定）。
3. 工单升级链：门店->总部客服->财务/合规仲裁；线下协同由区域督导执行。
4. 高风险工单仅允许人工仲裁，AI 只做优先级建议与证据聚合。
5. 超时SLA：每级处理超时自动升级并通知责任人。
6. 工单与订单、履约、提成、分账全链关联，可一键追溯。

### 4.8 实物供应链作业闭环
1. 补货：门店按安全库存和销量预测发起补货申请。
2. 调拨：支持门店互调与总部仓下拨。
3. 盘点：按日抽盘、周盘、月盘，差异自动生成稽核工单。
4. 损耗与报废：需责任人、原因、图片凭证和审批记录。
5. 收货确认：门店收货后入库，异常数量进入差异工单。

### 4.9 门店生命周期治理
1. 开店：资质、支付、商品模板、人员角色、风控规则一次性校验。
2. 停业：限制新预约/新发货，保留履约与售后处理能力。
3. 闭店：订单尾单清理、库存清算、资金结算、权限收回。
4. 迁店：门店主数据迁移、库存迁移、预约迁移和用户通知。
5. 加盟退出：合约终止、历史账务归档、审计留痕。

## 5. 技师提成体系（一级能力）

### 5.1 提成适用范围
1. 仅服务类订单项参与技师提成。
2. 套餐仅服务子项参与提成。
3. 实物商品默认不计技师提成。

### 5.2 计提时点
1. 服务履约完成后计提，不在支付成功时计提。

### 5.3 计提基数
1. 单品服务：服务订单项实付金额。
2. 套餐服务子项：按子项分摊后的实付金额。
3. 支持加钟、加项单独计提。

### 5.4 规则优先级
1. 总部规则。
2. 门店规则（在总部边界内）。
3. 技师特例。

下单时快照提成规则版本，确保结算可追溯。

### 5.5 逆向冲正
1. 退款后自动生成提成冲正单。
2. 已发放部分在下期结算抵扣。

### 5.6 提成争议处理
1. 争议单必须关联订单项快照、履约记录、规则版本。
2. 争议处理结果沉淀为规则优化输入，不允许口径不留痕。

## 6. 支付与结算（连锁合规）

### 6.1 支付模式
采用服务商+特约商户模式：

1. 统一收款。
2. 订单级分账。
3. 日清月结与对账闭环。

### 6.2 结算对象
1. 总部。
2. 门店。
3. 技师（提成结算）。

### 6.3 对账体系
1. 交易账。
2. 履约账。
3. 提成账。
4. 分账账。

四账可交叉核对，异常自动工单化。

### 6.4 结算审批链与SLA
1. 发起：财务生成门店结算单/技师结算单。
2. 复核：财务复核专员进行口径核对（必要时线下抄送区域督导协查）。
3. 审批：总部财务负责人审批大额/异常结算。
4. 打款：审批通过后执行打款并回写流水号，禁止 AI 自动放款。
5. 冲正：退款、拒付、争议结论触发冲正单并进入下期抵扣。
6. SLA：T+1 预结、T+3 复核、月结日最终校准，超时自动升级。

## 11. 数据与审计要求

1. 所有订单项必须保留商品类型、价格来源、规则版本快照。
2. 所有库存变更必须有幂等流水（业务类型+业务单号+门店+SKU 唯一）。
3. 所有提成记录必须可追溯至订单项与规则版本。
4. 所有关键操作必须有审计日志（操作者、前后值、时间、原因）。
5. 规则执行日志必须可追溯（命中规则、失败原因、是否审批通过）。
6. 领域事件需落库（至少含事件ID、主题、重试次数、最终状态）用于补偿。

## 12. 核心指标（KPI）

### 12.1 经营指标
1. 到店服务完成率。
2. 单店日均订单数。
3. 客单价。
4. 复购率（30天）。
5. 服务带货率（服务后实物购买率）。

### 12.2 运营效率指标
1. 平均预约转化时长。
2. 履约准时率。
3. 退款率。
4. 门店缺货率。

### 12.3 财务与风险指标
1. 分账差异率。
2. 对账及时率。
3. 提成争议率。
4. 退款冲正及时率。

### 12.4 角色化运营指标
1. 总部运营：上新时效、模板复用率、活动ROI。
2. 区域督导（线下职能）：训练覆盖率、巡检闭环率、突发协同时效。
3. 门店店长：预约履约率、库存准确率、客诉率。
4. 技师：服务完成率、复购贡献率、提成争议率。
5. 客服：首响时长、一次解决率、升级率。
6. 供应链：补货满足率、盘点差异率、损耗率。
7. 财务：结算准时率、冲正时效、对账差异率。

## 13. 非功能要求

1. 高峰时段下单链路可用性 >= 99.9%。
2. 价格和库存链路必须幂等。
3. 关键链路支持可回放与审计。
4. 万店规模下，门店价/库存读写支持分层缓存与异步同步。

## 14. 0->50 店阶段范围（本期）

### 14.1 必做
1. 服务/实物/套餐统一下单。
2. 服务履约状态闭环。
3. 门店 SKU 批量铺货、调价、调库存。
4. 技师提成计提与冲正。
5. 支付分账与对账闭环。
6. 双开关可售、合约策略、规则引擎最小可用版上线。
7. 订单快照冻结（价格/规则/版本/套餐子项）上线。

### 14.2 延后
1. 全量 EAV 迁移。
2. 全链路 ES 检索。
3. 高级智能定价策略。
4. 全量事件总线化（先关键交易事件）。

## 15. 风险与应对

1. 风险：服务与实物链路耦合导致回归风险高。
   - 应对：订单履约分域，统一订单聚合。
2. 风险：门店运营自由度过高导致总部失控。
   - 应对：总部边界+审批流+门禁策略。
3. 风险：提成与退款冲突引发财务争议。
   - 应对：规则快照+冲正单+月结校准。

## 16. 验收标准

1. 一笔混合订单可完整走通：下单、支付、履约、退款、对账。
2. 服务订单可触发技师提成，并在退款后自动冲正。
3. 门店可独立运营价格库存，总部可审计并可回滚。
4. 核心报表（交易/履约/提成/分账）口径一致。
5. 套餐订单支持子项分次核销，且核销以订单快照为准。
6. 规则调整支持灰度生效，不影响历史订单核销与退款。
7. 服务链路不扣实物库存，实物链路不锁技师排班，分流约束可测试验证。
8. 客服工单链可覆盖退款/履约/提成争议并按SLA自动升级。
9. 门店生命周期（开店/停业/闭店/迁店）至少完成一套全链路演练。
10. 线上权限层级仅保留“总部+门店”，无新增线上区域配置角色。
11. AI 辅助链路具备人工门禁与审计留痕，关键动作可回退到规则引擎+人工。
