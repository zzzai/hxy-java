# HXY 第二阶段命名迁移启动 v1（2026-02-28）

## 1. 本次已落地

1. Docker Compose 项目名默认改为 `hxy-platform`。
2. 容器名默认改为 `hxy-*`：
   - `hxy-mysql`
   - `hxy-redis`
   - `hxy-server`
   - `hxy-admin`
3. 镜像名默认改为 `hxy-*`：
   - `hxy-server`
   - `hxy-admin`
4. GitHub Workflow 名统一改为 `hxy-*` 前缀。
5. 新增命名门禁 CI：`hxy-naming-guard`（PR/Push 自动校验新增资产命名）。

## 2. Redis 升级

1. Docker 默认 Redis 镜像从 `redis:6-alpine` 升级为 `redis:7-alpine`。
2. `docker.env` 增加 `REDIS_IMAGE=redis:7-alpine`，支持显式覆盖。

## 3. 包名迁移启动策略

1. 当前状态：运行时与运维命名已进入 `hxy-*` 轨道，Java 包名仍为历史前缀。
2. 下一步（按模块批次）：
   - 批次 A：`yudao-module-pay` 与 `yudao-server` 新增代码优先使用 `com.hxy.*`（增量先行）。
   - 批次 B：`yudao-module-mall` 的新增业务子模块迁移到 `com.hxy.*`。
   - 批次 C：遗留包名全量迁移（需专窗 + 全回归）。
3. 门禁规则：禁止新增资产命名出现 `yudao/ruoyi/crmeb`，存量逐步消减。

## 4. 风险与回滚

1. 风险：容器名切换后，依赖旧容器名的脚本/监控可能失效。
2. 回滚：
   - 通过 `script/docker/docker.env` 临时改回旧容器名与镜像名；
   - 重新 `docker compose --env-file script/docker/docker.env -f script/docker/docker-compose.yml up -d`。
