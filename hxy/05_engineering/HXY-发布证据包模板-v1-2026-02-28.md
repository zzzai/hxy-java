# HXY 发布证据包模板 v1

## 1. 发布基础信息

1. 发布编号：`REL-YYYYMMDD-XX`
2. 发布环境：`dev / staging / prod`
3. 责任人：`技术Owner / 运维Owner / 值班人`
4. 发布时间窗：`开始时间 - 结束时间`
5. 关联需求/任务：`链接或编号`

## 2. 变更范围

1. 代码变更：
   - 分支/提交：`git sha`
   - 关键文件：`路径列表`
2. 配置变更：
   - 配置项：`key -> new value（脱敏）`
3. 数据库变更：
   - 脚本路径：`sql/mysql/...`
   - 执行顺序：`1,2,3`
   - 幂等说明：`是/否 + 说明`

## 3. 验证证据（必须）

1. 构建验证
   - 命令：`mvn -pl yudao-server -am -DskipTests package`
   - 结果：`PASS/FAIL`
   - 证据：`日志路径`
2. 核心接口冒烟
   - 商品、订单、优惠券接口结果：`code=0`
   - 证据：`curl 输出/截图路径`
3. 支付链路验证（涉及支付时必填）
   - 链路一致性：`check_wx_openid_chain_consistency.sh`
   - 真单支付退款：`run_wx_partner_real_pay_refund.sh`
   - 对账门禁：`REQUIRE_RECONCILE=1`
   - 证据：`summary/report 路径`

## 4. 门禁结论

1. 门禁脚本命令：`script/dev/check_payment_release_blockers.sh ...`
2. 结论：`PASS / PASS_WITH_WARN / BLOCK`
3. 阻断项（如有）：`问题码 + 处理人 + 截止时间`

## 5. 回滚方案（必须）

1. 回滚触发条件：
   - `关键错误率超阈值`
   - `支付/退款链路中断`
   - `对账阻断`
2. 回滚操作步骤：
   - 回滚代码：`镜像/版本`
   - 回滚配置：`配置项恢复`
   - 回滚数据：`SQL 回退脚本（如适用）`
3. 回滚验证命令：
   - `构建/接口/支付门禁复验命令`

## 6. 风险与应急

1. 已知风险：`列表`
2. 应急联系人：`产品/研发/运维/财务`
3. 处置SLA：`P0/P1/P2 响应时限`

## 7. 审批与签字

1. 技术Owner：`同意/不同意 + 时间`
2. 产品Owner：`同意/不同意 + 时间`
3. 运维Owner：`同意/不同意 + 时间`

## 8. 发布后复盘

1. 实际结果：`成功/失败`
2. 偏差说明：`与计划差异`
3. 后续动作：`优化项 + 负责人 + 截止时间`
