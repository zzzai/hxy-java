# 项目审查报告：ruoyi-vue-pro-master (芋道源码 + 荷小悦定制)

**审查日期：** 2026-02-27

## 项目概况

基于 ruoyi-vue-pro 开源框架的 O2O 服务商城系统，为"荷小悦"按摩/养生业务定制。Spring Boot 2.7.18 + JDK 8 单体架构，MyBatis Plus ORM，MySQL 数据库，Redis 缓存。

### 技术栈

| 组件 | 版本 |
|------|------|
| JDK | 1.8 |
| Spring Boot | 2.7.18 (EOL) |
| Spring Security | 5.8.16 |
| MyBatis Plus | 3.5.15 |
| Druid | 1.2.27 |
| Redisson | 3.52.0 |
| MapStruct | 1.6.3 |
| Hutool | 5.8.42 |

### 模块结构

```
yudao-dependencies/       # BOM 依赖版本管理
yudao-framework/          # 框架层 (15 个 starter)
yudao-server/             # 单体部署入口
yudao-module-system/      # 认证、用户、角色、菜单、租户
yudao-module-infra/       # 代码生成、定时任务、文件存储
yudao-module-member/      # 会员账户
yudao-module-pay/         # 支付渠道 (微信、支付宝)
yudao-module-mall/        # 商城域 (商品、交易、预约、营销、统计)
```

## HIGH — 上线前必须解决

### 3. Mock 登录旁路

**文件：** `application-local.yaml:232`

```yaml
yudao.security.mock-enable: true
```

**文件：** `SecurityProperties.java:40`

默认 secret 为 `"test"`。发送 token `test<userId>` 即可冒充任意用户。如果 local profile 意外激活到非本地环境，后果严重。

**修复方案：** 确保非 local 环境显式设置 `mock-enable: false`，修改默认 secret 为强随机值。

### 4. BCrypt 强度因子 = 4（过低）

**文件：** `SecurityProperties.java:50`

```java
private Integer passwordEncoderLength = 4;
```

BCrypt cost factor 4 是最低值，密码哈希可被快速暴力破解。

**修复方案：** 提升至 10 以上。需同步重新哈希现有用户密码（可在用户下次登录时自动升级）。

### 5. SQL 注入 — IoT TDengine Mapper

**文件：** `yudao-module-iot/.../IotDevicePropertyMapper.xml`、`IotDeviceMessageMapper.xml`

IoT 模块的 TDengine mapper XML 使用 `${}` 拼接表名和字段名，存在 SQL 注入风险。涉及 `productId`、`deviceId`、`field.field`、`field.type`、`reqVO.identifier` 等参数。

虽然 IoT 模块当前被注释掉未编译，但代码仍在仓库中。

**修复方案：** 所有动态标识符（表名、列名）必须通过严格白名单校验后才能使用。

### 6. Booking 模块 NPE 风险

**文件：** `BookingOrderServiceImpl.java:56-62`

```java
TimeSlotDO timeSlot = timeSlotService.getTimeSlot(timeSlotId);
// 无 null 检查，直接使用 timeSlot.getStoreId()
```

`lockTimeSlot` 成功后到 `getTimeSlot` 之间存在竞态条件，如果时间段被删除，`timeSlot` 为 null 导致 NPE。

**文件：** `ScheduleServiceImpl.java:171-175`

`workStartTime`/`workEndTime` 为 null 时同样会 NPE。

**修复方案：**
```java
TimeSlotDO timeSlot = timeSlotService.getTimeSlot(timeSlotId);
if (timeSlot == null) {
    throw exception(TIME_SLOT_NOT_EXISTS);
}
```

### 7. 短信发送无频率限制

**文件：** `SmsCodeServiceImpl.java:65-66`

```java
// TODO 芋艿：提升，每个 IP 每天可发送数量
// TODO 芋艿：提升，每个 IP 每小时可发送数量
```

无速率限制，攻击者可无限触发短信发送，造成费用损失和短信轰炸。

**修复方案：** 基于 IP + 手机号双维度限流，建议每 IP 每小时 10 次、每手机号每天 5 次。

## MEDIUM — 应尽快修复

### 8. 多个 Controller 缺少权限注解

以下 admin Controller 无 `@PreAuthorize` 注解，任何已认证用户均可调用：

| Controller | 风险 |
|------------|------|
| `CrmPermissionController` | 可删除任意 CRM 对象的数据权限 |
| `CrmFollowUpRecordController` | 可创建/删除任意跟进记录 |
| `PayStatisticsController` | 暴露支付/充值财务汇总 |
| `IotStatisticsController` | 暴露 IoT 设备/产品/消息统计 |

### 9. X-Frame-Options 完全禁用

**文件：** `YudaoWebSecurityConfigurerAdapter.java:119`

```java
.headers(c -> c.frameOptions(HeadersConfigurer.FrameOptionsConfig::disable))
```

存在点击劫持风险。应设置为 `SAMEORIGIN` 而非完全禁用。

### 10. XSS 过滤器全局禁用

**文件：** `application.yaml:268`

```yaml
yudao.xss.enable: false
```

如果应用渲染用户提交的 HTML 内容（商品描述、富文本），存在存储型 XSS 风险。

### 11. Druid 监控台无认证

**文件：** `application-local.yaml:20-23`、`application-dev.yaml:19-22`

Druid stat-view-servlet 启用但无用户名密码、无 IP 白名单。任何可达服务器的人均可访问 SQL 统计和慢查询日志。

### 12. CRMEB 兼容层技术债

**总计 4,538 行代码，分布在 9 个 Controller、3 个模块中。**

| 问题 | 详情 |
|------|------|
| Controller 直接注入 Mapper | `CrmebFrontOrderCompatController`、`CrmebFrontPayCompatController` 绕过 Service 层事务和业务规则 |
| `CrmebCompatResult` 重复定义 | 同一个类在 trade/pay/member 三个模块各复制一份 |
| 包结构不一致 | `CrmebAdminStoreOrderCompatController` 在 `controller/compat/crmeb/`，`CrmebAdminOrderCompatController` 在 `controller/admin/compat/crmeb/` |
| `CrmebPreOrderStoreImpl` 放在 Controller 包 | Redis 存储组件不应在 Controller 层 |

**建议：** 制定明确的兼容层退役时间表，前端迁移完成后移除。

| 文件 | 行数 | 建议 |

## LOW — 建议改进

### 14. 架构层面

- **`yudao-module-pay` 依赖 `yudao-module-trade-api`** — 通用支付模块不应编译依赖商城域 API，违反依赖方向原则
- **根 pom 注释掉 7 个模块**（bpm/crm/erp/iot/ai/report/mp）— CI 静默跳过，无法发现编译错误
- **无 Jacoco 覆盖率门禁** — 无构建时覆盖率检查
- **无 maven-enforcer-plugin** — 无最低 Maven 版本、无禁用依赖规则
- **版本号 `2026.01-jdk8-SNAPSHOT`** — SNAPSHOT 版本不可复现构建
- **trade ↔ promotion 循环依赖** — 通过 `yudao-module-trade-api` 解耦，结构合理但增加维护成本

### 15. 依赖问题

|------|------|

- `ScheduleServiceImpl.java:201-204` 循环单条 INSERT 应改为批量插入
