## 1. 项目概述

| 维度 | 内容 |
|------|------|
| 项目名称 | 荷小悦（HXY）O2O 健康养生服务平台 |
| 业务模式 | 线下体验 + 线上交易 + 售后 + 资金闭环 |
| 目标规模 | 0→50 店（2026 Q1-Q2），100 店（2026 Q4），万店蓝图（2027+） |
| 执行模式 | 1 人 + AI 协作 |
| 代码底座 | ruoyi-vue-pro（Spring Boot 2.7.18 / JDK 8 / MyBatis Plus 3.5.15） |
| 知识库 | `/root/crmeb-java/hxy/`（323 份文档，9 个主题目录） |

### 1.1 核心价值

不是单纯电商平台，而是：服务履约 + 商品交易 + 结算分润 + 合规治理 的一体化 O2O 系统。

### 1.2 用户角色

| 角色 | 职责 |
|------|------|
| C 端用户 | 浏览、预约、支付、退款、进度追踪 |
| 门店员工 | 服务/商品配置、订单履约、售后举证 |
| 总部运营 | 策略配置、订单/售后管理、KPI 看板 |
| 财务/风控 | 退款策略、风险拦截、对账 |

### 1.3 总部-门店权责边界

- **总部掌控**：支付执行、退款执行、对账、发布决策
- **门店掌控**：服务履约、售后材料提交、运营配置
- **禁止**：门店不可直接执行退款/打款，不可接触渠道账户密钥
