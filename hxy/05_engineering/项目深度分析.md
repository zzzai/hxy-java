# 荷小悦项目深度分析

> 基于 ruoyi-vue-pro 框架的 O2O 按摩/养生服务商城系统

## 1. 技术架构全景

### 1.1 技术栈

| 层级 | 技术选型 |
|------|---------|
| 后端框架 | Spring Boot 2.7.18 + JDK 8 |
| ORM | MyBatis Plus 3.x + MyBatis Plus Join |
| 数据库 | MySQL 8.x（多数据源 dynamic-datasource） |
| 缓存 | Redis + Spring Cache |
| 消息队列 | RocketMQ / Kafka / RabbitMQ（可选） |
| 定时任务 | Quartz（JDBC JobStore） |
| 工作流 | Flowable |
| 接口文档 | Knife4j + SpringDoc OpenAPI 3 |
| 前端 | UniApp Vue 3 + Pinia + sheep 框架 |
| 安全 | Spring Security + JWT + BCrypt |

### 1.2 模块依赖图

```
yudao-dependencies (BOM)
  └── yudao-framework
        ├── yudao-common

        ├── yudao-spring-boot-starter-redis

        └── ... (17 个 starter)

  └── yudao-module-mall (商城父模块)
        ├── yudao-module-product (SPU/SKU/分类/品牌)

        ├── yudao-module-trade-api (交易API契约/枚举)
        ├── yudao-module-booking (预约/技师/排班/时间槽) ← HXY核心
```

### 1.3 分层模式

```
Controller (接收请求，参数校验)
    ↓ VO/ReqVO
Convert (MapStruct 对象转换)
    ↓ DTO
Service (业务逻辑)
    ↓ DO
DAL: Mapper + DO (数据访问)
    ↓ SQL
Database (MySQL)
```

### 1.4 跨模块通信

- **API 接口模式**：`yudao-module-trade-api` 定义 `TradeOrderApi`、`TradeBrokerageApi` 等接口，由实现模块提供 Bean
- **Spring Bean 注入**：模块间通过 `@Autowired` / `@RequiredArgsConstructor` 注入 API 接口
- **示例**：Booking 模块通过 `ProductSpuApi` 和 `ProductSkuApi` 获取商品信息

### 1.5 配置体系

```
application.yaml          # 全局默认配置（所有环境共享）
  ↓ 覆盖
application-local.yaml    # 本地开发环境
application-dev.yaml      # 开发/测试环境
```

关键配置项：
- `spring.profiles.active: local` — 默认激活 local
- `yudao.security.mock-enable: true` — 仅 local 开启 mock
- `yudao.sms-code.begin-code/end-code: 9999` — 固定验证码（仅测试用）
- `yudao.xss.enable: false` — XSS 过滤器当前关闭

## 2. 核心业务模块详解

### 2.1 支付模块 (yudao-module-pay)

**核心实体：**
- `PayOrderDO` — 支付订单（appId, merchantOrderId, subject, price, status, channelCode, channelOrderNo）
- `PayOrderExtensionDO` — 支付扩展信息（渠道请求/响应数据）
- `PayRefundDO` — 退款单（orderId, merchantRefundId, payPrice, refundPrice, status）
- `PayChannelDO` — 支付渠道配置（appId, code, config JSON）
- `PayAppDO` — 支付应用

**支持的 12 种支付渠道：**
| 渠道 | Client 类 |
|------|----------|
| 微信 App | WxAppPayClient |
| 微信公众号 | WxPubPayClient |
| 微信 H5 | WxWapPayClient |
| 微信条码 | WxBarPayClient |
| 微信 Native | WxNativePayClient |
| 微信小程序 | WxLitePayClient |
| 支付宝 App | AlipayAppPayClient |
| 支付宝 PC | AlipayPcPayClient |
| 支付宝 H5 | AlipayWapPayClient |
| 支付宝条码 | AlipayBarPayClient |
| 支付宝扫码 | AlipayQrPayClient |
| 钱包支付 | WalletPayClient |

**回调流程：** 先应答 200 → 异步处理 → PayNotifyTask 重试机制

### 2.2 交易订单 (yudao-module-trade)

**核心实体：**
- `TradeOrderDO` — 主订单（userId, orderNo, type, status, price, payOrderId, deliveryType）
- `TradeOrderItemDO` — 订单项（orderId, spuId, skuId, count, price, afterSaleStatus）
- `TradeServiceOrderDO` — 服务订单（orderId, orderItemId, status, bookingNo, technicianId）

**订单状态机：**
```
UNPAID → UNDELIVERED → DELIVERED → COMPLETED
  ↓                                    ↓
CANCELED                          (可售后)
```

**价格计算管道（10 个 Calculator 按 Order 排序）：**
TradeProductSkuOrderHandler → TradeCouponOrderHandler → TradePointOrderHandler → TradeMemberPointOrderHandler → TradeBargainOrderHandler → TradeCombinationOrderHandler → TradeSeckillOrderHandler → TradeBrokerageOrderHandler → TradeServiceOrderHandler → TradeStatusSyncToWxaOrderHandler

**TradeOrderHandler 生命周期钩子：**
- `beforeOrderCreate` — 创建前校验
- `afterOrderCreate` — 创建后处理（如扣库存、创建服务订单）
- `afterPayOrder` — 支付后处理
- `afterCancelOrder` — 取消后处理

### 2.3 分销佣金 (Brokerage)

**三表模型：**
- `BrokerageUserDO` — 分销用户（userId, bindUserId, brokerageEnabled, brokeragePrice, frozenPrice）
- `BrokerageRecordDO` — 佣金记录（userId, bizType, price, totalPrice, status, frozenDays, unfreezeTime）
- `BrokerageWithdrawDO` — 提现申请（userId, price, feePrice, totalPrice, type, status, bankName, bankAccount）

**二级分佣计算：**
1. 订单完成 → TradeBrokerageOrderHandler 触发
2. 查找推荐人（一级）和推荐人的推荐人（二级）
3. 按 TradeConfigDO 中配置的比例计算佣金
4. 创建 BrokerageRecordDO（状态=冻结）
5. 冻结期满 → 解冻 → 可提现

### 2.4 预约系统 (yudao-module-booking)

**四层模型：**
```
TechnicianDO (技师)
  └── TechnicianScheduleDO (排班，按日)
        └── TimeSlotDO (时间槽，按小时)
              └── BookingOrderDO (预约订单)
```

**BookingOrderDO 状态机：**
```
PENDING_PAYMENT(0) → PAID(1) → IN_SERVICE(2) → COMPLETED(3)
       ↓                ↓
  CANCELLED(4)     REFUNDED(5)
```

**CAS 锁定机制（TimeSlotService）：**
1. `lockTimeSlot(slotId, userId)` — CAS 更新 status=LOCKED, lockUserId, lockExpireTime
2. 15 分钟未支付 → 自动释放锁
3. `confirmBooking(slotId, orderId)` — 确认预约，status=BOOKED
4. `cancelBooking(slotId)` — 取消，status=AVAILABLE

**闲时定价规则（OffpeakRuleDO）：**
- 按门店+星期+时间段匹配
- 支持固定价格和折扣率两种模式
- 时间槽生成时预计算闲时标记

**超时自动取消 Job：**
- `autoCancelTimeoutOrders()` — 扫描 PENDING_PAYMENT 状态，超过 15 分钟自动取消

### 2.5 服务订单履约 (TradeServiceOrder)

**TradeServiceOrderDO 状态机：**
```
WAIT_BOOKING(10) → BOOKED(20) → SERVING(30) → FINISHED(40)
       ↓               ↓
  CANCELED(50)    CANCELED(50)
```

- 支付回调自动创建 TradeServiceOrder（TradeServiceOrderHandler.afterPayOrder）
- 关联 bookingNo 字段与 BookingOrder 打通
- 补偿重试 Job 处理异常状态

### 2.6 售后退款

**AfterSaleRefundRule 分层策略：**
- `AfterSaleRefundRuleConfigDO` — 退款规则配置（金额阈值、黑名单、自动/人工）
- `AfterSaleRefundDecisionService` — 规则引擎，判断走自动退款还是人工审核

**ReviewTicket 工单升级：**
- `AfterSaleReviewTicketDO` — 审核工单（afterSaleId, priority P0/P1/P2, status, slaDeadline）
- P0 = 1小时 SLA，P1 = 4小时，P2 = 24小时
- 超时自动升级优先级

### 2.7 会员体系 (yudao-module-member)

**MemberUserDO 核心字段：**
- userId, nickname, avatar, mobile, status
- levelId, experience, point（等级/经验/积分）
- groupId, tagIds（分组/标签）

**已有功能：** 等级体系、积分体系、签到、分组、标签
**未实现：** 卡项体系（仅有 ProductType.CARD 枚举定义，无实际实现）

### 2.8 商品模块 (yudao-module-product)

**SPU/SKU 模型：**
- `ProductSpuDO` — 标准产品单元（name, categoryId, brandId, picUrl, price, type, status）
- `ProductSkuDO` — 库存单元（spuId, properties, price, stock, picUrl）

**4 种商品类型（ProductTypeEnum）：**
1. 实物商品 (NORMAL)
2. 服务商品 (SERVICE) — 对应预约服务
3. 卡项商品 (CARD) — 枚举已定义，无业务实现
4. 虚拟商品 (VIRTUAL)

**服务加项表：** `hxy_service_addon`（spuId, addonName, addonPrice, duration）

### 2.9 营销促销 (yudao-module-promotion)

- 优惠券模板 + 用户优惠券
- 满减活动
- 秒杀活动 + 秒杀时段配置
- 拼团活动 + 拼团记录
- 砍价活动 + 砍价记录
- 积分商城
- DIY 模板 + 页面可视化编辑

## 3. 前端现状 (yudao-mall-uniapp)

### 3.1 技术栈
- UniApp + Vue 3 Composition API
- Pinia 状态管理
- sheep 框架（组件库 + 工具集）
- 支持微信小程序 / H5 / App

### 3.2 页面结构
```
pages/
├── index/          # 首页（4 个 Tab）
├── goods/          # 商品详情、列表
├── order/          # 订单列表、详情、售后
├── user/           # 个人中心、设置、地址
├── pay/            # 支付、结果
├── commission/     # 分销佣金
├── coupon/         # 优惠券
├── activity/       # 营销活动
├── chat/           # 客服聊天
├── public/         # 登录、注册
└── components/     # 页面级组件
```

### 3.3 API 层
```
sheep/api/
├── member/         # 会员相关
├── trade/          # 交易订单
├── product/        # 商品
├── pay/            # 支付
├── promotion/      # 营销
├── system/         # 系统
└── infra/          # 基础设施
```

### 3.4 缺失页面
- **预约相关：** 技师列表、技师详情、时段选择、预约确认、预约订单详情/列表 — 全部缺失
- **卡项相关：** 卡项列表、卡项详情、我的卡项 — 全部缺失
- **加钟/升级：** 无页面
