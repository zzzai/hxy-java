# HXY 工程与发布规范 V1（2026-02-22）

## 1. 分支与变更策略

1. 主分支：`main`。
2. 功能分支：`feature/<domain>-<topic>`。
3. 修复分支：`hotfix/<issue-id>`。
4. 变更必须附带：风险说明、验收命令、回滚步骤。

## 2. 发布门槛

1. P0 验收未通过禁止发布。
2. 上线拦截规则必须 PASS（或经过明确豁免）。
3. 回滚预案必须可执行且已演练。

## 3. 运行与监控

1. 支付关键链路必须有实时指标与告警。
2. 任务补偿必须有积压监控与告警。
3. 对账差异必须进入工单并闭环。

## 4. 文档要求

1. 所有关键决策文档化。
2. 所有验收结果落地到 `hxy`。
3. 历史资料只归档，不散落。
