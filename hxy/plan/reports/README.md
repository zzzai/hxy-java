# reports 目录说明

阶段性报告已按“当前入口 + 历史沉淀”拆分。

## 目录结构

```text
reports/
├── current/   # 当前可直接使用的状态与部署入口
└── history/   # 历史阶段报告与工作总结
```

## current

1. `current/项目迁移包-状态快照.md`
2. `current/项目完整部署报告.md`

## history

1. `history/阶段一-后端文件创建完成.md`
2. `history/会员卡片组件开发完成报告.md`
3. `history/会员卡片集成完成报告.md`
4. `history/今日工作总结-2026-02-08.md`

## 使用建议

1. 默认先看 `current/`，用于快速恢复当前上下文。
2. 需要追溯实现细节与过程时再看 `history/`。
