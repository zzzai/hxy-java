## 🔍 故障排查

### 问题：访问 8088 跳转到 HTTPS 或丢失端口

**原因**: 系统配置中的 `site_url` 不正确

**解决方案**:
```bash
# 1. 更新数据库配置
docker exec -i hexiaoyue_mysql mysql -u root -proot crmeb -e \
  "UPDATE eb_system_config SET value='\"http://115.190.245.14:8088\"' WHERE menu_name='site_url';"

# 2. 清除缓存
docker exec hexiaoyue_redis redis-cli FLUSHALL
docker exec hexiaoyue_php rm -rf /var/www/html/runtime/cache/*

# 3. 刷新浏览器
```

---

### 问题：无法访问公网地址

**检查清单**:
1. ✅ FRP 服务是否运行
2. ✅ 防火墙是否开放 8088 端口
3. ✅ Docker 容器是否正常运行
4. ✅ Nginx 是否监听 80 端口

**检查命令**:
```bash
# 检查 FRP 进程
ps aux | grep frpc

# 检查 Docker 容器
docker ps | grep hexiaoyue

# 检查端口监听
netstat -tlnp | grep 8080

# 测试本地访问
curl -I http://localhost:8080

# 检查 FRP 日志
journalctl -u frpc -n 50
```

## 📝 重要提示

### ⚠️ 配置注意事项

1. **系统配置使用公网地址**
   - `site_url` 必须配置为 `http://115.190.245.14:8088`
   - 这样系统生成的链接才能正确访问

2. **API 接口调用**
   - 小程序/前端应使用公网地址：`http://115.190.245.14:8088/api/...`
   - 服务器内部可使用 `localhost:8080`

3. **缓存清除**
   - 修改配置后必须清除 Redis 和文件缓存
   - 否则旧配置会继续生效

4. **FRP 稳定性**
   - 确保 FRP 客户端持续运行
   - 建议使用 systemd 管理 FRP 进程

---

## 🔐 安全建议

1. **FRP Token**
   - 使用强密码作为 FRP token
   - 定期更换 token

2. **防火墙配置**
   - 仅开放必要的端口（8088）
   - 限制访问来源 IP（如果可能）

3. **HTTPS 配置**（未来）
   - 建议配置 SSL 证书
   - 使用 HTTPS 加密传输
