# 荷小悦小程序 MVP 版本规划（V1.0）

> **基于**: CRMEB 二次开发  
> **定位**: 千店连锁足疗预约小程序  
> **目标**: 跑通核心预约闭环，验证商业模式  
> **开发周期**: 6-8周

---

## 一、产品定位

### 1.1 核心价值主张

**用户视角**: "附近的足疗店，在线预约，到店即享"  
**门店视角**: "统一品牌，独立运营，线上获客"  
**总部视角**: "标准化服务，数据化管理，规模化扩张"

### 1.2 MVP 目标

- ✅ 跑通预约闭环：选店 → 选服务 → 选时间 → 支付 → 到店核销
- ✅ 验证千店模式：总部统一管理，门店独立定价
- ✅ 建立用户心智：足疗预约 = 荷小悦
- ✅ 积累核心数据：用户画像、预约习惯、门店效率

### 1.3 不做什么（MVP 边界）

- ❌ 不做游戏化（虚拟农场、挑战赛）→ V2.0
- ❌ 不做 AI 诊断（舌诊、面诊）→ V2.0
- ❌ 不做社交裂变（礼品卡、邀请红包）→ V2.0
- ❌ 不做会员等级（简化为普通用户/会员两档）→ V2.0
- ❌ 不做内容社区（养生文章、UGC）→ V2.0

## 四、千店模式设计

### 4.1 数据架构

```
总部层
├── 标准服务库 (eb_service)
├── 服务分类 (eb_service_category)
└── 品牌配置

门店层
├── 门店信息 (eb_system_store)
├── 门店服务 (eb_store_service) - 独立定价
├── 技师管理 (eb_store_technician)
├── 技师排班 (eb_store_technician_schedule)
└── 门店订单

用户层
├── 用户信息 (eb_user)
├── 用户订单 (eb_store_order)
└── 用户优惠券
```

### 4.2 权限设计

| 角色 | 权限范围 | 典型操作 |
|------|---------|---------|
| **总部管理员** | 全局 | 创建标准服务、管理所有门店、查看全局数据 |
| **门店店长** | 单店 | 服务定价、技师管理、排班设置、订单管理 |
| **技师** | 个人 | 查看自己的排班、接单、核销 |
| **用户** | 个人 | 预约、支付、评价 |

### 4.3 定价策略

```
标准服务（总部）
├── 服务名称：经典足疗
├── 服务时长：60分钟
├── 指导价：¥88
└── 服务标准：统一

门店定价（门店）
├── 世纪大道店：¥88（CBD，房租高）
├── 社区店：¥68（社区，房租低）
└── 会员价：各店自定（通常9折）
```

### 5.3 三大盲点保护（必须实现）

#### 盲点1: 并发预约锁
```php
// 原子更新 + 影响行数判断
$affected = Db::name('store_technician_schedule')
    ->where('id', $scheduleId)
    ->where('is_available', 1)
    ->update(['is_available' => 0, 'order_id' => $orderId]);

if ($affected === 0) {
    throw new ValidateException('手慢了，该时段已被预约');
}
```

#### 盲点2: 技师端通知
```php
// 预约成功后发送技师通知
SubscribeTemplateService::send([
    'uid' => $technician->uid,
    'template_id' => 'booking_notify',
    'data' => [
        'thing1' => $order->service_name,
        'time2' => $order->reserve_date . ' ' . $order->reserve_time,
        'thing3' => $order->user->nickname
    ]
]);
```

#### 盲点3: 订单状态映射
```php
// 服务状态映射到 CRMEB 状态
const CRMEB_STATUS_MAP = [
    0 => OrderModel::STATUS_UNSHIPPED,    // 待服务 → 待发货
    1 => OrderModel::STATUS_DELIVERY,     // 服务中 → 已发货
    2 => OrderModel::STATUS_RECEIVED,     // 已完成 → 已收货
];
```

## 八、风险与应对

### 8.1 技术风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| 并发预约冲突 | 用户体验差 | 高 | ✅ 已实现原子锁机制 |
| CRMEB 兼容性 | 功能异常 | 中 | ✅ 使用状态映射，不破坏原有逻辑 |
| 第三方接口不稳定 | 支付/定位失败 | 低 | 增加重试机制和降级方案 |

### 8.2 业务风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| 门店接受度低 | 推广困难 | 中 | 提供培训、激励政策 |
| 用户习惯未养成 | 使用率低 | 中 | 首单优惠、推广活动 |
| 技师操作不熟练 | 核销效率低 | 高 | 简化操作流程、提供培训 |
