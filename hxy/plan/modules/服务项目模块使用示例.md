## 🎯 核心设计理念

### 1. 双表模式解决的问题

| 问题 | 传统方案 | 双表方案 |
|------|---------|---------|
| **千店千价** | 每个门店复制一份商品 | 主表存标准信息，关联表存价格 |
| **时间库存** | 使用 stock 字段（不适用） | duration 字段 + 排班表 |
| **技师匹配** | 无法实现 | tech_level_ids 字段筛选 |
| **总部管控** | 难以统一标准 | 主表统一管理，门店灵活定价 |

### 2. 数据流转逻辑

```
总部创建标准服务 (eb_service)
         ↓
门店选择上架并定价 (eb_store_service)
         ↓
用户浏览门店服务列表
         ↓
选择服务 → 选择时间 → 选择技师
         ↓
创建预约订单 (eb_store_order)
         ↓
锁定技师时间槽 (eb_store_technician_schedule)
```

## 💻 代码使用示例

### 示例1: 获取门店服务列表（前端接口）

```php
<?php
namespace app\controller\api;

use app\model\store\service\StoreService;
use think\Response;

class ServiceController
{
    /**
     * 获取门店服务列表
     * @param int $storeId 门店ID
     * @return Response
     */
    public function getStoreServices($storeId)
    {
        // 查询该门店所有可售服务（含服务详情）
        $services = StoreService::with(['service'])
            ->where('store_id', $storeId)
            ->valid()  // 门店上架 且 服务主表启用
            ->sort()   // 按排序字段排序
            ->select()
            ->map(function ($item) {
                return [
                    'id' => $item->id,
                    'service_id' => $item->service_id,
                    'name' => $item->service->name,
                    'image' => $item->service->image,
                    'intro' => $item->service->intro,
                    'duration' => $item->service->duration,
                    'price' => $item->price,
                    'member_price' => $item->member_price,
                    'member_discount' => $item->member_discount,  // 会员折扣
                    'member_save' => $item->member_save,          // 会员节省
                    'sales' => $item->sales,
                    'can_choose_tech' => $item->can_choose_tech,
                ];
            });

        return json(['code' => 200, 'data' => $services]);
    }
}
```

### 示例3: 获取可服务的技师列表

```php
/**
 * 获取可服务的技师列表
 * @param int $storeServiceId 门店服务ID
 * @param string $date 预约日期
 * @param string $timeSlot 预约时段
 * @return Response
 */
public function getAvailableTechnicians($storeServiceId, $date, $timeSlot)
{
    $storeService = StoreService::find($storeServiceId);
    
    if (!$storeService) {
        return json(['code' => 404, 'msg' => '服务不存在']);
    }

    // 获取符合等级要求的技师
    $technicians = $storeService->getAvailableTechnicians();

    // 进一步筛选：该时段有空的技师
    $availableTechs = [];
    foreach ($technicians as $tech) {
        // 检查该技师在指定时段是否有空
        $schedule = \app\model\store\StoreTechnicianSchedule::where([
            'technician_id' => $tech->id,
            'work_date' => $date,
            'time_slot' => $timeSlot,
            'is_available' => 1
        ])->find();

        if ($schedule) {
            $availableTechs[] = [
                'id' => $tech->id,
                'name' => $tech->name,
                'avatar' => $tech->avatar,
                'level' => $tech->level,
                'specialty' => $tech->specialty,
                'rating' => $tech->rating,
                'order_count' => $tech->order_count,
            ];
        }
    }

    return json(['code' => 200, 'data' => $availableTechs]);
}
```

### 示例4: 创建预约订单（核心逻辑）

```php
/**
 * 创建预约订单
 * @param array $data 订单数据
 * @return Response
 */
public function createBooking($data)
{
    // 开启事务
    Db::startTrans();
    try {
        // 1. 获取门店服务信息
        $storeService = StoreService::with(['service'])->find($data['store_service_id']);
        
        if (!$storeService || !$storeService->is_show) {
            throw new \Exception('服务不存在或已下架');
        }

        // 2. 检查技师是否可以提供该服务
        if ($data['technician_id'] > 0) {
            $technician = \app\model\store\StoreTechnician::find($data['technician_id']);
            
            if (!$storeService->canServeByTechnician($technician->level)) {
                throw new \Exception('该技师无法提供此服务');
            }
        }

        // 3. 锁定技师时间槽（原子操作，防止并发）⚠️ 关键盲点
        $scheduleId = $this->findScheduleId($data['technician_id'], $data['reserve_date'], $data['reserve_time']);
        
        $affected = Db::name('store_technician_schedule')
            ->where('id', $scheduleId)
            ->where('is_available', 1)  // 必须是可用状态
            ->update([
                'is_available' => 0,
                'order_id' => 0,  // 先占位，订单创建成功后再更新
                'update_time' => time()
            ]);

        if ($affected === 0) {
            throw new \Exception('手慢了，该时段已被预约');
        }

        // 4. 创建订单
        $order = \app\model\store\StoreOrder::create([
            'uid' => $data['uid'],
            'order_type' => 2,  // 2=服务预约
            'store_id' => $storeService->store_id,
            'store_service_id' => $storeService->id,
            'technician_id' => $data['technician_id'],
            'reserve_date' => $data['reserve_date'],
            'reserve_time_slot' => $data['reserve_time'],
            'reserve_start_time' => strtotime($data['reserve_date'] . ' ' . $data['reserve_time']),
            'service_duration' => $storeService->service->duration,
            'total_price' => $storeService->price,
            'pay_price' => $this->calculatePayPrice($storeService, $data['uid']),
            'status' => 0,  // 待支付
            'service_status' => 0,  // 待服务（使用状态映射）⚠️ 关键盲点
        ]);

        // 5. 更新排班表的订单ID
        Db::name('store_technician_schedule')
            ->where('id', $scheduleId)
            ->update(['order_id' => $order->id]);

        // 6. 发送技师通知（订阅消息）⚠️ 关键盲点
        $this->notifyTechnician($data['technician_id'], $order);

        Db::commit();

        return json(['code' => 200, 'data' => ['order_id' => $order->id]]);

    } catch (\Exception $e) {
        Db::rollback();
        return json(['code' => 500, 'msg' => $e->getMessage()]);
    }
}

/**
 * 计算实付金额（考虑会员折扣）
 */
private function calculatePayPrice($storeService, $uid)
{
    $user = \app\model\user\User::find($uid);
    
    // 如果是会员，使用会员价
    if ($user->member_level > 0) {
        return $storeService->member_price;
    }
    
    return $storeService->price;
}

/**
 * 发送技师通知
 */
private function notifyTechnician($technicianId, $order)
{
    $technician = \app\model\store\StoreTechnician::find($technicianId);
    
    if ($technician && $technician->uid) {
        // 使用 CRMEB 的订阅消息服务
        \app\services\message\SubscribeTemplateService::send([
            'uid' => $technician->uid,
            'template_id' => 'booking_notify',
            'data' => [
                'thing1' => $order->service_name,
                'time2' => $order->reserve_date . ' ' . $order->reserve_time_slot,
                'thing3' => $order->user->nickname
            ]
        ]);
    }
}
```

### 示例5: 后台管理 - 门店服务上架

```php
/**
 * 门店服务上架（后台管理）
 * @param int $storeId 门店ID
 * @param int $serviceId 服务ID
 * @param array $data 上架数据
 * @return Response
 */
public function addStoreService($storeId, $serviceId, $data)
{
    // 检查服务是否已存在
    $exists = StoreService::where([
        'store_id' => $storeId,
        'service_id' => $serviceId
    ])->find();

    if ($exists) {
        return json(['code' => 400, 'msg' => '该服务已上架']);
    }

    // 创建门店服务
    $storeService = StoreService::create([
        'store_id' => $storeId,
        'service_id' => $serviceId,
        'price' => $data['price'],
        'member_price' => $data['member_price'],
        'is_show' => $data['is_show'] ?? 1,
        'sort' => $data['sort'] ?? 0,
        'can_choose_tech' => $data['can_choose_tech'] ?? 1,
        'tech_level_ids' => $data['tech_level_ids'] ?? [],
    ]);

    return json(['code' => 200, 'msg' => '上架成功', 'data' => $storeService]);
}
```

## 🔍 查询优化建议

### 1. 使用预加载避免 N+1 问题

```php
// ❌ 错误：会产生 N+1 查询
$services = StoreService::where('store_id', 1)->select();
foreach ($services as $service) {
    echo $service->service->name;  // 每次循环都查询一次
}

// ✅ 正确：使用预加载
$services = StoreService::with(['service'])->where('store_id', 1)->select();
foreach ($services as $service) {
    echo $service->service->name;  // 只查询一次
}
```

### 2. 使用查询范围简化代码

```php
// ❌ 冗长的查询
$services = StoreService::where('store_id', 1)
    ->where('is_show', 1)
    ->whereHas('service', function($query) {
        $query->where('status', 1);
    })
    ->order('sort', 'desc')
    ->select();

// ✅ 使用查询范围
$services = StoreService::where('store_id', 1)
    ->valid()  // 封装了上架和启用的条件
    ->sort()   // 封装了排序逻辑
    ->select();
```

### 3. 索引优化

```sql
-- 确保以下索引存在
ALTER TABLE `eb_store_service` 
ADD INDEX `idx_store_show` (`store_id`, `is_show`),
ADD INDEX `idx_service_show` (`service_id`, `is_show`);

-- 查询时会使用索引
SELECT * FROM eb_store_service 
WHERE store_id = 1 AND is_show = 1;
```

## ⚠️ 三大盲点的实现

### 盲点1: 并发预约锁机制

```php
// ✅ 正确：原子更新 + 影响行数判断
$affected = Db::name('store_technician_schedule')
    ->where('id', $scheduleId)
    ->where('is_available', 1)  // 作为更新条件
    ->update(['is_available' => 0, 'order_id' => $orderId]);

if ($affected === 0) {
    throw new ValidateException('手慢了，该时段已被预约');
}

// ❌ 错误：先查询再更新（会产生并发问题）
$schedule = Db::name('store_technician_schedule')->find($scheduleId);
if ($schedule['is_available'] == 1) {
    // 这里可能被其他请求抢占

        ->where('id', $scheduleId)
        ->update(['is_available' => 0]);
}
```

### 盲点3: 订单状态映射

```php
// 服务订单的状态映射
class BookingOrderService 
{
    // 我们的服务状态
    const SERVICE_STATUS_WAITING = 0;    // 待服务
    const SERVICE_STATUS_SERVING = 1;    // 服务中
    const SERVICE_STATUS_COMPLETED = 2;  // 已完成
    
    // CRMEB 状态映射
    const CRMEB_STATUS_MAP = [
        self::SERVICE_STATUS_WAITING => OrderModel::STATUS_UNSHIPPED,    // 待发货
        self::SERVICE_STATUS_SERVING => OrderModel::STATUS_DELIVERY,     // 已发货
        self::SERVICE_STATUS_COMPLETED => OrderModel::STATUS_RECEIVED,   // 已收货
    ];
    
    /**
     * 更新服务状态（同时更新 CRMEB 状态）
     */
    public function updateServiceStatus($orderId, $serviceStatus)
    {
        $order = OrderModel::find($orderId);
        
        // 更新我们的服务状态字段
        $order->service_status = $serviceStatus;
        
        // 映射到 CRMEB 状态（不破坏原有逻辑）
        $order->status = self::CRMEB_STATUS_MAP[$serviceStatus];
        
        $order->save();
    }
}
```

```php
public function getHotServices($storeId)
```
