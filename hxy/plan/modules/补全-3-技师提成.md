# 补全模块3：技师提成计算规则

**优先级**：P0（必须）  
**影响范围**：技师激励、门店运营、财务结算  
**开发工作量**：1天

---

## 一、问题描述

原方案提到"技师排班"，但**未说明技师提成计算规则**，导致：

### 业务痛点

1. **技师收入不透明**
   - 完成服务后，技师不知道能拿多少提成
   - 影响技师积极性和服务质量

2. **套餐提成分配不清**
   - 套餐包含多个子项，提成怎么分配？
   - 多个技师协作完成套餐，提成怎么拆分？

3. **闲时优惠影响提成**
   - 闲时价格低，技师提成是否同步降低？
   - 如何平衡门店利润和技师收入？

4. **财务对账困难**
   - 没有提成计算规则，财务无法自动结算
   - 月底手工计算，效率低且易出错

---

## 二、解决方案

### 方案核心：多层级提成规则 + 自动计算 + 实时展示

## 四、提成计算规则

### 4.1 基础提成计算公式

```php
// 1. 按比例提成
$baseCommission = $orderPrice * $commissionRate;

// 2. 固定金额提成
$baseCommission = $commissionAmount;

// 3. 加上技师等级加成
$finalCommission = $baseCommission * (1 + $levelBonusRate);

// 4. 保障最低提成
if ($finalCommission < $minCommission) {
    $finalCommission = $minCommission;
}
```

### 4.2 不同场景的提成规则

| 场景 | 提成计算规则 | 示例 |
|------|------------|------|
| **单品服务** | 订单实付价 × 提成比例 × (1 + 等级加成) | ¥198 × 30% × 1.05 = ¥62.37 |
| **闲时服务** | 订单实付价 × 闲时提成比例 × (1 + 等级加成) | ¥98 × 20% × 1.05 = ¥20.58 |
| **套餐服务** | 按子项价格占比分配套餐总提成 | 见下文详细说明 |
| **多技师协作** | 按服务时长占比分配提成 | 见下文详细说明 |
| **会员折扣** | 按实付价计算（不是原价） | 原价¥198，会员价¥178，按¥178计算 |
| **优惠券抵扣** | 按实付价计算（扣除优惠券） | 原价¥198，券后¥168，按¥168计算 |

---

### 4.3 套餐提成分配规则

#### 场景：养生套餐A = 60分钟足疗 + 养生茶 + 艾草贴

**套餐总价**：¥298  
**子项指导价**：
- 60分钟足疗：¥198（服务类，技师A）
- 养生茶：¥50（实物类，无提成）
- 艾草贴：¥50（实物类，无提成）

**提成计算**：
```php
// 1. 计算服务类子项总价值
$serviceTotalValue = 198;  // 只有足疗是服务类

// 2. 计算套餐服务类提成基数
$serviceCommissionBase = 298 * 0.30;  // 套餐总价 × 提成比例 = ¥89.4

// 3. 技师A的提成
$technicianACommission = $serviceCommissionBase * (198 / 198) * 1.05;  // ¥93.87
```

**规则总结**：
- 实物类子项不参与提成分配
- 服务类子项按价值占比分配套餐总提成
- 单个技师完成全部服务，拿全部提成

---

### 4.4 多技师协作提成分配

#### 场景：120分钟全身SPA = 60分钟足疗（技师A）+ 60分钟推拿（技师B）

**订单总价**：¥398  
**提成比例**：30%  
**提成总额**：¥398 × 30% = ¥119.4

**分配规则**：
```php
// 按服务时长占比分配
$technicianATime = 60;  // 技师A服务60分钟
$technicianBTime = 60;  // 技师B服务60分钟
$totalTime = 120;

// 技师A提成
$technicianACommission = 119.4 * (60 / 120) * 1.05 = ¥62.37

// 技师B提成
$technicianBCommission = 119.4 * (60 / 120) * 1.00 = ¥59.70
```

## 五、代码实现示例

### 5.1 下单时计算提成

```php
// app/services/order/OrderService.php

public function calculateCommission($orderItem)
{
    // 1. 获取门店SKU信息
    $storeSku = Db::name('store_service_sku')
        ->where('id', $orderItem['store_sku_id'])
        ->find();
    
    // 2. 获取技师信息
    $technician = Db::name('technician')
        ->where('id', $orderItem['technician_id'])
        ->find();
    
    // 3. 判断是否闲时
    $isOffpeak = $this->isOffpeakTime($orderItem['service_time']);
    $commissionRate = $isOffpeak 
        ? $storeSku['offpeak_commission_rate'] 
        : $storeSku['commission_rate'];
    
    // 4. 计算基础提成
    if ($storeSku['commission_type'] == 1) {
        // 按比例
        $baseCommission = $orderItem['price'] * $commissionRate;
    } else {
        // 固定金额
        $baseCommission = $storeSku['commission_amount'];
    }
    
    // 5. 加上等级加成
    $finalCommission = $baseCommission * (1 + $technician['level_bonus_rate']);
    
    // 6. 保障最低提成
    if ($finalCommission < $storeSku['min_commission']) {
        $finalCommission = $storeSku['min_commission'];
    }
    
    // 7. 写入订单子项
    Db::name('order_item')->where('id', $orderItem['id'])->update([
        'commission_amount' => $finalCommission,
        'commission_status' => 0
    ]);
    
    // 8. 写入提成流水
    Db::name('technician_commission')->insert([
        'technician_id' => $technician['id'],
        'store_id' => $storeSku['store_id'],
        'order_id' => $orderItem['order_id'],
        'order_item_id' => $orderItem['id'],
        'sku_id' => $storeSku['sku_id'],
        'sku_name' => $storeSku['sku_name'],
        'order_price' => $orderItem['price'],
        'commission_type' => $storeSku['commission_type'],
        'commission_rate' => $commissionRate,
        'level_bonus_rate' => $technician['level_bonus_rate'],
        'commission_amount' => $finalCommission,
        'status' => 0,
        'create_time' => time()
    ]);
    
    return $finalCommission;
}
```

### 5.2 技师提成查询接口

```php
// app/api/controller/technician/CommissionController.php

public function myCommission(Request $request)
{
    $technicianId = $request->technicianId;  // 从token解析
    $month = $request->param('month', date('Y-m'));  // 默认当月
    
    // 1. 查询当月提成流水
    $list = Db::name('technician_commission')
        ->where('technician_id', $technicianId)
        ->where('create_time', '>=', strtotime($month . '-01'))
        ->where('create_time', '<', strtotime($month . '-01 +1 month'))
        ->order('create_time', 'desc')
        ->select();
    
    // 2. 统计汇总
    $total = Db::name('technician_commission')
        ->where('technician_id', $technicianId)
        ->where('create_time', '>=', strtotime($month . '-01'))
        ->where('create_time', '<', strtotime($month . '-01 +1 month'))
        ->sum('commission_amount');
    
    $settled = Db::name('technician_commission')
        ->where('technician_id', $technicianId)
        ->where('create_time', '>=', strtotime($month . '-01'))
        ->where('create_time', '<', strtotime($month . '-01 +1 month'))
        ->where('status', '>=', 1)
        ->sum('commission_amount');
    
    $paid = Db::name('technician_commission')
        ->where('technician_id', $technicianId)
        ->where('create_time', '>=', strtotime($month . '-01'))
        ->where('create_time', '<', strtotime($month . '-01 +1 month'))
        ->where('status', 2)
        ->sum('commission_amount');
    
    return json([
        'code' => 200,
        'data' => [
            'list' => $list,
            'summary' => [
                'total' => $total,      // 总提成
                'settled' => $settled,  // 已结算
                'paid' => $paid,        // 已发放
                'pending' => $total - $settled  // 待结算
            ]
        ]
    ]);
}
```

## 七、财务结算流程

### 7.1 月度结算流程

```
每月1日自动触发
    ↓
查询上月所有待结算提成
    ↓
按技师汇总提成金额
    ↓
生成结算单
    ↓
财务审核
    ↓ 通过
更新提成状态为"已结算"
    ↓
发放工资
    ↓
更新提成状态为"已发放"
```
