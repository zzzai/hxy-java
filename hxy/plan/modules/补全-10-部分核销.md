# 补全模块10：订单子项部分核销

**优先级**：P1（重要）  
**影响范围**：次卡消费、套餐核销、财务对账  
**开发工作量**：1天

---

## 一、问题描述

原方案提到"原子化核销"，但**未说明部分核销场景**，导致：

### 业务痛点

1. **次卡无法分次消费**
   - 用户购买"10次足疗卡"
   - 每次到店只能核销1次，但系统只支持全部核销

2. **套餐核销不灵活**
   - 套餐包含3个子项，用户分3次消费
   - 系统无法记录每次核销了哪些子项

3. **核销记录不清晰**
   - 不知道谁核销的、什么时间核销的
   - 无法追溯核销历史

4. **财务对账困难**
   - 次卡分次核销，收入如何确认？
   - 技师提成如何计算？

---

## 二、解决方案

### 方案核心：次卡数量管理 + 核销记录表 + 分次确认收入

## 三、核销模型设计

### 3.1 核销类型

| 类型 | 说明 | 核销方式 | 示例 |
|------|------|----------|------|
| **单次服务** | 一次性消费完成 | 全部核销 | 60分钟足疗 |
| **次卡服务** | 分多次消费 | 部分核销 | 10次足疗卡，每次核销1次 |
| **套餐服务** | 包含多个子项 | 子项逐个核销 | 养生套餐A（足疗+茶+艾草贴） |
| **实物商品** | 一次性核销 | 全部核销 | 精油10ml |

---

### 3.2 核销状态流转

```
未核销（0）
    ↓ 部分核销
部分核销（1）
    ↓ 全部核销完
已核销（2）
    
或

未核销（0）
    ↓ 一次性核销
已核销（2）
```

## 五、核销逻辑实现

### 5.1 部分核销

```php
// app/services/verification/VerificationService.php

public function verifyOrderItem($orderItemId, $quantity = 1, $technicianId = 0, $remark = '')
{
    // 1. 获取订单子项
    $orderItem = Db::name('order_item')->where('id', $orderItemId)->find();
    
    if (!$orderItem) {
        throw new Exception('订单子项不存在');
    }
    
    // 2. 检查核销状态
    if ($orderItem['verification_status'] == 2) {
        throw new Exception('该订单已全部核销');
    }
    
    // 3. 检查剩余数量
    if ($orderItem['remaining_quantity'] < $quantity) {
        throw new Exception('剩余数量不足');
    }
    
    // 4. 检查有效期
    if ($orderItem['expire_time'] > 0 && $orderItem['expire_time'] < time()) {
        throw new Exception('订单已过期');
    }
    
    Db::startTrans();
    try {
        // 5. 更新订单子项
        $usedQuantity = $orderItem['used_quantity'] + $quantity;
        $remainingQuantity = $orderItem['remaining_quantity'] - $quantity;
        $verificationStatus = ($remainingQuantity == 0) ? 2 : 1;
        
        Db::name('order_item')->where('id', $orderItemId)->update([
            'used_quantity' => $usedQuantity,
            'remaining_quantity' => $remainingQuantity,
            'verification_status' => $verificationStatus,
            'first_verification_time' => $orderItem['first_verification_time'] ?: time(),
            'last_verification_time' => time()
        ]);
        
        // 6. 生成核销码
        $verificationCode = $this->generateVerificationCode($orderItemId);
        
        // 7. 记录核销记录
        Db::name('verification_record')->insert([
            'order_id' => $orderItem['order_id'],
            'order_item_id' => $orderItemId,
            'store_id' => request()->storeId ?? 0,
            'technician_id' => $technicianId,
            'verification_quantity' => $quantity,
            'before_quantity' => $orderItem['remaining_quantity'],
            'after_quantity' => $remainingQuantity,
            'verification_code' => $verificationCode,
            'verification_type' => 1,
            'operator_id' => request()->uid ?? 0,
            'operator_name' => request()->userName ?? '',
            'remark' => $remark,
            'create_time' => time()
        ]);
        
        // 8. 如果是服务类，计算技师提成
        if ($technicianId > 0) {
            $this->calculateCommission($orderItem, $quantity, $technicianId);
        }
        
        // 9. 如果全部核销完，更新订单状态
        if ($verificationStatus == 2) {
            $this->checkOrderComplete($orderItem['order_id']);
        }
        
        Db::commit();
        
        return [
            'verification_code' => $verificationCode,
            'remaining_quantity' => $remainingQuantity,
            'status' => $verificationStatus == 2 ? '已核销完' : '部分核销'
        ];
        
    } catch (Exception $e) {
        Db::rollback();
        throw $e;
    }
}

private function generateVerificationCode($orderItemId)
{
    return 'V' . date('YmdHis') . str_pad($orderItemId, 6, '0', STR_PAD_LEFT);
}

private function checkOrderComplete($orderId)
{
    // 检查订单所有子项是否都已核销
    $unverified = Db::name('order_item')
        ->where('order_id', $orderId)
        ->where('verification_status', '<', 2)
        ->count();
    
    if ($unverified == 0) {
        // 所有子项都已核销，更新订单状态
        Db::name('order')->where('id', $orderId)->update([
            'status' => 3,  // 已完成
            'complete_time' => time()
        ]);
    }
}
```

### 5.2 批量核销（套餐场景）

```php
public function verifyComboItems($orderId, $itemIds, $technicianId = 0)
{
    // 1. 获取订单信息
    $order = Db::name('order')->where('id', $orderId)->find();
    
    if (!$order) {
        throw new Exception('订单不存在');
    }
    
    // 2. 获取订单子项
    $items = Db::name('order_item')
        ->where('order_id', $orderId)
        ->where('id', 'in', $itemIds)
        ->select();
    
    Db::startTrans();
    try {
        $results = [];
        
        foreach ($items as $item) {
            // 逐个核销子项
            $result = $this->verifyOrderItem($item['id'], 1, $technicianId);
            $results[] = [
                'item_id' => $item['id'],
                'item_name' => $item['item_name'],
                'result' => $result
            ];
        }
        
        Db::commit();
        
        return $results;
        
    } catch (Exception $e) {
        Db::rollback();
        throw $e;
    }
}
```

## 七、收入确认规则

### 7.1 次卡收入确认

```php
// 次卡收入按核销次数分摊

public function confirmRevenue($orderItemId, $verificationQuantity)
{
    $orderItem = Db::name('order_item')->where('id', $orderItemId)->find();
    
    // 1. 计算单次收入
    $totalRevenue = $orderItem['price'];  // 次卡总价
    $totalQuantity = $orderItem['total_quantity'];  // 总次数
    $unitRevenue = $totalRevenue / $totalQuantity;  // 单次收入
    
    // 2. 本次确认收入
    $thisRevenue = $unitRevenue * $verificationQuantity;
    
    // 3. 记录收入
    Db::name('revenue_record')->insert([
        'order_id' => $orderItem['order_id'],
        'order_item_id' => $orderItemId,
        'revenue_type' => 2,  // 次卡收入
        'revenue_amount' => $thisRevenue,
        'verification_quantity' => $verificationQuantity,
        'create_time' => time()
    ]);
    
    return $thisRevenue;
}
```

protected function execute(Input $input, Output $output)

        ->where('verification_status', '<', 2)  // 未核销完
