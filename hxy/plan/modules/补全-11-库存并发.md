# 补全模块11：库存扣减原子化设计

**优先级**：P0（必须）  
**影响范围**：订单系统、库存管理、并发控制  
**开发工作量**：1天

---

## 一、问题描述

原方案提到"策略模式"处理库存扣减，但**未说明并发场景处理**，存在严重风险：

### 风险场景

1. **库存超卖**
   - 库存剩余1件，10个用户同时下单
   - 导致实际售出10件，库存变成-9

2. **服务时间冲突**
   - 技师A在14:00-15:00只能服务1人
   - 多个用户同时预约该时间槽
   - 导致技师时间冲突

3. **套餐子项库存不一致**
   - 套餐包含3个子项，扣减到第2个时库存不足
   - 前2个已扣减，第3个失败，数据不一致

4. **锁定库存未释放**
   - 用户下单锁定库存，但30分钟未支付
   - 库存一直被占用，影响其他用户购买

---

## 二、解决方案

### 方案核心：预约即扣减 + 分布式锁 + 三段式库存 + 自动释放

**注意**：服务类商品与实物商品的库存逻辑不同！

---

## 三、库存模型设计

### 3.1 实物商品库存（三段式）

| 字段名 | 说明 | 变化时机 |
|--------|------|----------|
| `total_stock` | 总库存 | 采购入库时增加 |
| `available_stock` | 可售库存 | 下单时减少，取消时增加 |
| `locked_stock` | 锁定库存（已下单未支付） | 下单时增加，支付/取消时减少 |
| `sold_stock` | 已售库存 | 支付成功时增加 |

**关系公式**：
```
total_stock = available_stock + locked_stock + sold_stock
```

**状态流转**：
```
采购入库
    ↓
total_stock += 100
available_stock += 100
    ↓
用户下单
    ↓
available_stock -= 1
locked_stock += 1
    ↓
支付成功
    ↓
locked_stock -= 1
sold_stock += 1
    ↓
或：取消订单
    ↓
locked_stock -= 1
available_stock += 1
```

---

### 3.2 服务类商品库存（时间槽状态）

**关键差异**：服务类不是"库存数量"，而是"时间槽状态"

| 状态 | 说明 | 变化时机 |
|------|------|----------|
| `1=可预约` | 时间槽空闲 | 初始状态 |
| `2=已锁定` | 已被预约 | **预约时立即锁定** |
| `3=已完成` | 服务已完成 | 核销后 |
| `4=技师请假` | 时间槽不可用 | 技师请假 |

**状态流转**：
```
生成排班
    ↓
status = 1（可预约）
    ↓
用户预约（无论是否支付）
    ↓
status = 2（已锁定）
    ↓
到店核销
    ↓
status = 3（已完成）
    ↓
或：用户取消（24小时前）
    ↓
status = 1（释放时间槽）
```

**重要**：服务类预约**不需要"锁定-支付-扣减"流程**，预约即锁定！

## 五、并发控制实现

### 5.1 实物商品下单（分布式锁 + 行锁）

```php
// app/services/order/StockService.php

public function lockStock($storeId, $skuId, $quantity, $orderId)
{
    // 1. Redis分布式锁（防止高并发）
    $lockKey = "stock_lock:{$storeId}:{$skuId}";
    $lock = Redis::set($lockKey, 1, ['NX', 'EX' => 10]);
    
    if (!$lock) {
        throw new Exception('库存正在被操作，请稍后重试');
    }
    
    try {
        // 2. 数据库行锁（防止数据不一致）
        Db::startTrans();
        
        $stock = Db::name('store_product_sku')
            ->where('store_id', $storeId)
            ->where('sku_id', $skuId)
            ->lock(true)  // 行锁
            ->find();
        
        // 3. 检查可售库存
        if ($stock['available_stock'] < $quantity) {
            throw new Exception('库存不足');
        }
        
        // 4. 扣减可售库存，增加锁定库存
        $beforeStock = $stock['available_stock'];
        
        Db::name('store_product_sku')
            ->where('id', $stock['id'])
            ->update([
                'available_stock' => Db::raw('available_stock - ' . $quantity),
                'locked_stock' => Db::raw('locked_stock + ' . $quantity),
                'update_time' => time()
            ]);
        
        // 5. 记录库存流水
        Db::name('stock_flow')->insert([
            'store_id' => $storeId,
            'sku_id' => $skuId,
            'sku_type' => 2,
            'order_id' => $orderId,
            'change_type' => 2,  // 下单锁定
            'change_quantity' => -$quantity,
            'before_stock' => $beforeStock,
            'after_stock' => $beforeStock - $quantity,
            'operator_type' => 1,  // 用户操作
            'remark' => '用户下单锁定库存',
            'create_time' => time()
        ]);
        
        Db::commit();
        Redis::del($lockKey);
        
        return true;
        
    } catch (Exception $e) {
        Db::rollback();
        Redis::del($lockKey);
        throw $e;
    }
}
```

```php
// app/services/order/ScheduleService.php

public function lockSchedule($scheduleId, $orderId)
{
    // 1. Redis分布式锁
    $lockKey = "schedule_lock:{$scheduleId}";
    $lock = Redis::set($lockKey, 1, ['NX', 'EX' => 10]);
    
    if (!$lock) {
        throw new Exception('该时间槽正在被预约，请稍后重试');
    }
    
    try {
        Db::startTrans();
        
        // 2. 数据库行锁
        $schedule = Db::name('technician_schedule')
            ->where('id', $scheduleId)
            ->lock(true)
            ->find();
        
        // 3. 检查状态
        if ($schedule['status'] != 1) {
            throw new Exception('该时间槽已被预约');
        }
        
        // 4. 锁定时间槽（预约即锁定，不等支付）
        Db::name('technician_schedule')
            ->where('id', $scheduleId)
            ->update([
                'status' => 2,
                'order_id' => $orderId,
                'lock_time' => time(),
                'update_time' => time()
            ]);
        
        Db::commit();
        Redis::del($lockKey);
        
        return true;
        
    } catch (Exception $e) {
        Db::rollback();
        Redis::del($lockKey);
        throw $e;
    }
}
```

## 六、套餐库存原子化扣减

### 6.1 套餐下单流程

```php
// app/services/order/ComboService.php

public function createComboOrder($comboId, $storeId, $userId)
{
    // 1. 获取套餐子项
    $items = Db::name('service_combo_relation')
        ->where('combo_service_id', $comboId)
        ->where('is_mandatory', 1)  // 只处理必选子项
        ->select();
    
    // 2. 预检查所有子项库存/时间槽
    foreach ($items as $item) {
        if ($item['item_type'] == 1) {
            // 服务类：检查时间槽
            $this->checkScheduleAvailable($item['item_id'], $storeId);
        } else {
            // 实物类：检查库存
            $this->checkStockAvailable($item['item_id'], $storeId, $item['item_num']);
        }
    }
    
    // 3. 创建订单
    $orderId = $this->createOrder($comboId, $storeId, $userId);
    
    // 4. 原子化扣减所有子项（事务）
    Db::startTrans();
    try {
        foreach ($items as $item) {
            if ($item['item_type'] == 1) {
                // 服务类：锁定时间槽
                $scheduleId = $this->selectSchedule($item['item_id'], $storeId);
                $this->lockSchedule($scheduleId, $orderId);
            } else {
                // 实物类：锁定库存
                $this->lockStock($storeId, $item['item_id'], $item['item_num'], $orderId);
            }
            
            // 记录订单子项
            Db::name('order_item')->insert([
                'order_id' => $orderId,
                'item_type' => $item['item_type'],
                'item_id' => $item['item_id'],
                'item_num' => $item['item_num'],
                'status' => 0,
                'create_time' => time()
            ]);
        }
        
        Db::commit();
        return $orderId;
        
    } catch (Exception $e) {
        Db::rollback();
        throw new Exception('套餐下单失败：' . $e->getMessage());
    }
}
```

### 7.1 释放超时未支付的实物库存

```php
// app/command/ReleaseStock.php

protected function execute(Input $input, Output $output)
{
    // 每5分钟执行一次
    
    // 1. 查询超时未支付的订单
    $timeout = time() - 1800;  // 30分钟超时
    
    $orders = Db::name('order')
        ->where('pay_status', 0)  // 未支付
        ->where('create_time', '<', $timeout)
        ->where('is_del', 0)
        ->select();
    
    foreach ($orders as $order) {
        // 2. 获取订单子项
        $items = Db::name('order_item')
            ->where('order_id', $order['id'])
            ->where('item_type', 2)  // 只处理实物类
            ->select();
        
        // 3. 释放库存
        foreach ($items as $item) {
            try {
                $stockService = new StockService();
                $stockService->releaseStock(
                    $order['store_id'],
                    $item['item_id'],
                    $item['item_num'],
                    $order['id']
                );
            } catch (Exception $e) {
                $output->writeln('释放库存失败：' . $e->getMessage());
            }
        }
        
        // 4. 取消订单
        Db::name('order')->where('id', $order['id'])->update([
            'order_status' => -1,  // 已取消
            'cancel_reason' => '超时未支付自动取消',
            'cancel_time' => time()
        ]);
    }
    
    $output->writeln('超时库存释放完成');
}
```

```php
// app/services/stock/AlertService.php

public function checkLowStock()
{
    // 查询库存低于预警线的SKU
    $lowStocks = Db::name('store_product_sku')
        ->alias('sps')
        ->join('service_sku ss', 'sps.sku_id = ss.id')
        ->where('sps.available_stock', '<=', Db::raw('sps.alert_stock'))
        ->where('sps.available_stock', '>', 0)
        ->field('sps.*, ss.sku as sku_name')
        ->select();
    
    foreach ($lowStocks as $stock) {
        // 发送预警通知给门店
        $this->sendAlert($stock['store_id'], [
            'type' => 'low_stock',
            'sku_name' => $stock['sku_name'],
            'current_stock' => $stock['available_stock'],
            'alert_stock' => $stock['alert_stock']
        ]);
    }
}
```

## 十一、总结

### 核心价值

1. **防止超卖**：分布式锁+行锁双重保障
2. **服务类特殊处理**：预约即锁定，不等支付
3. **套餐原子化**：所有子项同时成功或失败
4. **自动释放**：超时未支付自动释放库存
5. **数据可追溯**：所有库存变动记录流水

### 关键差异

| 维度 | 实物商品 | 服务类商品 |
|------|----------|------------|
| 库存模型 | 三段式库存 | 时间槽状态 |
| 扣减时机 | 支付成功后 | 预约时立即 |
| 超时处理 | 自动释放库存 | 按取消规则处理 |
| 并发控制 | 分布式锁+行锁 | 分布式锁+行锁 |
