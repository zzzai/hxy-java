# 补全模块1：门店定价边界控制

**优先级**：P0（必须）  
**影响范围**：门店运营、品牌管控、价格体系  
**开发工作量**：1天

---

## 一、问题描述

原方案提到"门店可自主定价"，但**未明确定价边界**，存在以下风险：

### 风险场景

1. **恶意低价抢客**
   - 总部指导价 ¥198，门店A定价 ¥98
   - 导致其他门店客流流失，破坏价格体系

2. **过高定价损害品牌**
   - 总部指导价 ¥198，门店B定价 ¥398
   - 用户投诉"同品牌价格差异大"，损害品牌形象

3. **闲时优惠无底线**
   - 门店设置闲时价 ¥1，批量占位后转卖
   - 影响真实需求用户预约

---

## 二、解决方案

### 方案核心：总部设定浮动范围 + 门店在范围内自主定价

## 四、业务逻辑规则

### 4.1 门店定价流程

```
门店设置价格
    ↓
判断：是否在允许范围内？
    ↓ 是
直接生效（price_status=1）
    ↓ 否
提交审核（price_status=2）
    ↓
总部审核
    ↓ 通过
生效（price_status=1）
    ↓ 驳回
恢复指导价（price_status=0）
```

### 4.2 价格范围计算公式

```php
// 门店定价范围
$minPrice = $guidePrice * $priceMinRate;  // 最低价
$maxPrice = $guidePrice * $priceMaxRate;  // 最高价

// 闲时定价范围
$offpeakMinPrice = $guidePrice * $offpeakMinRate;  // 闲时最低价
$offpeakMaxPrice = $minPrice;  // 闲时最高价 = 正常最低价
```

### 4.3 定价校验规则

| 场景 | 校验规则 | 处理方式 |
|------|----------|----------|
| 门店设置正常价格 | `$minPrice ≤ 门店价 ≤ $maxPrice` | 直接生效 |
| 门店设置正常价格（超范围） | `门店价 < $minPrice` 或 `门店价 > $maxPrice` | 提交审核 |
| 门店设置闲时价格 | `$offpeakMinPrice ≤ 闲时价 ≤ $offpeakMaxPrice` | 直接生效 |
| 门店设置闲时价格（超范围） | `闲时价 < $offpeakMinPrice` | 禁止设置，提示"低于最低限价" |
| 总部禁止自定义 | `allow_custom_price = 0` | 强制使用指导价，门店无法修改 |

## 五、代码实现示例（ThinkPHP 6.x）

### 5.1 门店设置价格接口

```php
// app/api/controller/store/ServiceController.php

public function setPrice(Request $request)
{
    $storeId = $request->storeId;  // 从token解析
    $skuId = $request->param('sku_id');
    $price = $request->param('price');
    $isOffpeak = $request->param('is_offpeak', 0);  // 是否闲时价
    
    // 1. 获取总部SKU信息
    $sku = Db::name('service_sku')->where('id', $skuId)->find();
    if (!$sku) {
        return json(['code' => 404, 'msg' => 'SKU不存在']);
    }
    
    // 2. 检查是否允许自定义价格
    if ($sku['allow_custom_price'] == 0) {
        return json(['code' => 403, 'msg' => '总部禁止自定义价格']);
    }
    
    // 3. 计算价格范围
    $guidePrice = $sku['guide_price'];
    if ($isOffpeak) {
        // 闲时价格范围
        $minPrice = $guidePrice * $sku['offpeak_min_rate'];
        $maxPrice = $guidePrice * $sku['price_min_rate'];
    } else {
        // 正常价格范围
        $minPrice = $guidePrice * $sku['price_min_rate'];
        $maxPrice = $guidePrice * $sku['price_max_rate'];
    }
    
    // 4. 校验价格
    if ($price < $minPrice || $price > $maxPrice) {
        // 超出范围，提交审核
        $priceStatus = 2;
        $msg = "价格超出范围（¥{$minPrice}-¥{$maxPrice}），已提交审核";
    } else {
        // 在范围内，直接生效
        $priceStatus = 1;
        $msg = "设置成功";
    }
    
    // 5. 更新门店SKU
    Db::name('store_service_sku')->where([
        'store_id' => $storeId,
        'sku_id' => $skuId
    ])->update([
        'price' => $price,
        'price_status' => $priceStatus,
        'is_custom_price' => 1,
        'update_time' => time()
    ]);
    
    return json(['code' => 200, 'msg' => $msg]);
}
```

### 5.2 总部审核价格接口

```php
// app/admin/controller/ServiceController.php

public function auditPrice(Request $request)
{
    $id = $request->param('id');  // 门店SKU ID
    $status = $request->param('status');  // 1=通过 2=驳回
    $remark = $request->param('remark', '');
    
    $storeSku = Db::name('store_service_sku')->where('id', $id)->find();
    if (!$storeSku || $storeSku['price_status'] != 2) {
        return json(['code' => 400, 'msg' => '无待审核记录']);
    }
    
    if ($status == 1) {
        // 审核通过
        Db::name('store_service_sku')->where('id', $id)->update([
            'price_status' => 1,
            'price_audit_status' => 1,
            'price_audit_time' => time(),
            'price_audit_remark' => $remark
        ]);
        $msg = '审核通过';
    } else {
        // 审核驳回，恢复指导价
        $sku = Db::name('service_sku')->where('id', $storeSku['sku_id'])->find();
        Db::name('store_service_sku')->where('id', $id)->update([
            'price' => $sku['guide_price'],
            'price_status' => 0,
            'price_audit_status' => 2,
            'price_audit_time' => time(),
            'price_audit_remark' => $remark
        ]);
        $msg = '审核驳回，已恢复指导价';
    }
    
    return json(['code' => 200, 'msg' => $msg]);
}
```

## 八、业务规则补充

### 8.1 特殊场景处理

| 场景 | 处理规则 |
|------|----------|
| 总部调整指导价 | 自动重新计算所有门店的价格范围，超出范围的自动提交审核 |
| 总部调整浮动比例 | 同上 |
| 总部禁止自定义 | 强制所有门店恢复指导价，已设置的自定义价格失效 |
| 门店多次超范围被驳回 | 3次驳回后，禁止该门店30天内再次申请超范围定价 |

### 8.2 价格展示优先级

```
前端展示价格 = 
  IF (price_status = 1 AND is_custom_price = 1) THEN 门店自定义价格
  ELSE IF (price_status = 0) THEN 总部指导价
  ELSE IF (price_status = 2) THEN 总部指导价（审核中）
```

-- 1. 总部SKU表新增字段
ALTER TABLE `eb_service_sku`

ADD COLUMN `price_max_rate` decimal(5,2) DEFAULT 1.50 COMMENT '最高定价比例' AFTER `price_min_rate`,

## 十一、总结

### 核心价值

1. **保护品牌**：防止门店恶意定价损害品牌形象
2. **灵活运营**：门店在合理范围内自主定价
3. **风险可控**：超范围定价需总部审核
4. **数据透明**：所有定价操作可追溯
