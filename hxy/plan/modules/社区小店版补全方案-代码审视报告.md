**示例实现**:
```php
namespace app\services\pricing;

class PriceCalculatorService
{
    /**
     * 根据定价策略计算最终价格
     * 
     * @param float $basePrice 基础价格
     * @param array $strategy 定价策略
     * @param array $context 上下文（会员等级、优惠券、时间段等）
     * @return float
     */
    public function calculate(float $basePrice, array $strategy, array $context): float
    {
        $pipeline = $strategy['price_calc_pipeline']; // 如 'base->time->member->coupon'
        $steps = explode('->', $pipeline);
        
        $currentPrice = $basePrice;
        
        foreach ($steps as $step) {
            switch ($step) {
                case 'time':
                    $currentPrice = $this->applyTimeDiscount($currentPrice, $context);
                    break;
                case 'member':
                    $currentPrice = $this->applyMemberDiscount($currentPrice, $context, $strategy);
                    break;
                case 'coupon':
                    $currentPrice = $this->applyCoupon($currentPrice, $context, $strategy);
                    break;
            }
        }
        
        return $currentPrice;
    }
    
    // ... 具体实现
}
```

## 💡 核心算法亮点解析

### 1. "Penny Perfect Split" 算法

**问题**: 传统的按比例分摊会产生"尾差"（除不尽的情况）

**示例**:
```
订单实付: 135.01元
子项A原价: 100元
子项B原价: 100元

传统算法:
- 子项A分摊: 135.01 * (100/200) = 67.505 → 67.51元
- 子项B分摊: 135.01 * (100/200) = 67.505 → 67.51元
- 总和: 67.51 + 67.51 = 135.02元 ❌ 多了0.01元！

Penny Perfect Split:
- 子项A分摊: 135.01 * (100/200) = 67.51元（四舍五入）
- 子项B分摊: 135.01 - 67.51 = 67.50元（兜底）
- 总和: 67.51 + 67.50 = 135.01元 ✅ 精确！
```

**核心代码**:
```php
if ($index == $count) {
    // 最后一个子项：兜底尾差
    $splitPrice = bcsub($totalPayPrice, $currentSumSplit, 2);
} else {
    $splitPrice = bcmul($totalPayPrice, $weight, 2);
    $currentSumSplit = bcadd($currentSumSplit, $splitPrice, 2);
}
```

**优势**:
- ✅ 确保财务合规：`Sum(Split) === Total Paid`（精确到分）
- ✅ 避免"多退少补"的尴尬情况
- ✅ 支持任意数量的子项

### 2. BOM 自动扣减算法

**问题**: 服务完成后，物料消耗无法自动扣减库存

**解决方案**:
1. 配置服务的 BOM 清单（必耗品 + 可选品）
2. 技师完成服务时，自动扣减库存
3. 记录库存流水（`flow_type=2` 服务消耗）

**核心代码**:
```php
// 判断是否需要扣减
$shouldDeduct = $bom['is_mandatory'] == 1 || in_array($bom['product_sku_id'], $optionalSkus);

if ($shouldDeduct) {
    // 扣减库存
    $newStock = bcsub($storeSku['stock'], $bom['quantity'], 2);
    
    // 记录流水（flow_type=2 服务消耗）
    $this->stockFlowDao->save([
        'flow_type' => 2,
        'num' => -$bom['quantity'],
        'remark' => "技师[{$technicianId}]服务消耗"
    ]);
}
```

**优势**:
- ✅ 自动化：技师点一下"完成"，库存自动扣
- ✅ 可追溯：完整的库存流水记录
- ✅ 灵活性：支持必耗品和可选品
- ✅ 精确性：支持小数扣减（如 10.5ml 精油）
