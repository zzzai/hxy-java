## 一、问题描述

原方案提到"可选子项"，但**未说明库存不足的处理**，导致：

### 业务痛点

1. **用户选择后发现缺货**
   - 用户选了可选子项，下单时发现库存不足
   - 体验差，可能放弃购买

2. **没有库存提示**
   - 不知道可选子项库存状态
   - 无法提前做出选择

3. **没有替代方案**
   - 缺货后没有推荐替代商品
   - 用户流失

---

## 二、解决方案

### 方案核心：实时库存展示 + 缺货提示 + 替代推荐 + 缺货登记

## 五、代码实现

### 5.1 获取可选子项库存状态

```php
// app/services/combo/ComboItemService.php

public function getComboItemsWithStock($comboId, $storeId)
{
    // 1. 获取套餐子项
    $items = Db::name('service_combo_relation')
        ->where('combo_service_id', $comboId)
        ->select();
    
    $result = [];
    
    foreach ($items as $item) {
        // 2. 获取库存信息
        $stock = Db::name('store_product_sku')
            ->where('store_id', $storeId)
            ->where('sku_id', $item['item_id'])
            ->find();
        
        // 3. 判断库存状态
        $stockStatus = $this->getStockStatus($stock['available_stock']);
        
        // 4. 如果缺货，查找替代商品
        $alternatives = [];
        if ($stockStatus['status'] == 'out_of_stock') {
            $alternatives = $this->findAlternatives($item['item_id'], $storeId);
        }
        
        $result[] = [
            'item_id' => $item['item_id'],
            'item_name' => $item['item_name'],
            'add_price' => $item['add_price'],
            'is_mandatory' => $item['is_mandatory'],
            'stock_status' => $stockStatus,
            'alternatives' => $alternatives
        ];
    }
    
    return $result;
}

private function getStockStatus($availableStock)
{
    if ($availableStock > 10) {
        return [
            'status' => 'sufficient',
            'label' => '库存充足',
            'color' => 'green',
            'icon' => '🟢'
        ];
    } elseif ($availableStock >= 3) {
        return [
            'status' => 'low',
            'label' => '仅剩' . $availableStock . '件',
            'color' => 'orange',
            'icon' => '🟠'
        ];
    } else {
        return [
            'status' => 'out_of_stock',
            'label' => '已售罄',
            'color' => 'gray',
            'icon' => '⚪'
        ];
    }
}

private function findAlternatives($skuId, $storeId)
{
    // 查找同类商品
    $sku = Db::name('service_sku')->where('id', $skuId)->find();
    
    return Db::name('store_product_sku')
        ->alias('sps')
        ->join('service_sku ss', 'sps.sku_id = ss.id')
        ->where('sps.store_id', $storeId)
        ->where('ss.service_id', $sku['service_id'])  // 同SPU
        ->where('ss.id', '<>', $skuId)  // 排除自己
        ->where('sps.available_stock', '>', 0)  // 有库存
        ->field('ss.id, ss.sku as name, sps.price')
        ->limit(3)
        ->select();
}
```

### 5.2 下单前库存校验

```php
public function validateComboStock($comboId, $storeId, $selectedItems)
{
    $errors = [];
    
    foreach ($selectedItems as $itemId) {
        $stock = Db::name('store_product_sku')
            ->where('store_id', $storeId)
            ->where('sku_id', $itemId)
            ->find();
        
        if ($stock['available_stock'] < 1) {
            $sku = Db::name('service_sku')->where('id', $itemId)->find();
            $errors[] = $sku['sku'] . '库存不足';
        }
    }
    
    if (!empty($errors)) {
        throw new Exception(implode('；', $errors));
    }
    
    return true;
}
```

### 5.3 缺货登记

```php
// app/services/stock/StockNotifyService.php

public function registerStockNotify($userId, $skuId, $storeId)
{
    // 检查是否已登记
    $exists = Db::name('stock_notify')
        ->where('user_id', $userId)
        ->where('sku_id', $skuId)
        ->where('store_id', $storeId)
        ->where('status', 0)  // 待通知
        ->find();
    
    if ($exists) {
        return ['message' => '您已登记过该商品到货提醒'];
    }
    
    // 创建登记记录
    Db::name('stock_notify')->insert([
        'user_id' => $userId,
        'sku_id' => $skuId,
        'store_id' => $storeId,
        'status' => 0,
        'create_time' => time()
    ]);
    
    return ['message' => '登记成功，到货后将第一时间通知您'];
}

// 到货通知
public function notifyStockArrival($skuId, $storeId)
{
    // 查询登记用户
    $notifies = Db::name('stock_notify')
        ->where('sku_id', $skuId)
        ->where('store_id', $storeId)
        ->where('status', 0)
        ->select();
    
    foreach ($notifies as $notify) {
        // 发送通知
        $this->sendNotification($notify['user_id'], [
            'type' => 'stock_arrival',
            'sku_id' => $skuId,
            'store_id' => $storeId,
            'message' => '您关注的商品已到货'
        ]);
        
        // 更新状态
        Db::name('stock_notify')
            ->where('id', $notify['id'])
            ->update(['status' => 1, 'notify_time' => time()]);
    }
}
```

## 六、数据表结构

### 6.1 缺货登记表（`eb_stock_notify`）

```sql
CREATE TABLE `eb_stock_notify` (
  `id` int(11) unsigned NOT NULL AUTO_INCREMENT,
  `user_id` int(11) NOT NULL DEFAULT 0 COMMENT '用户ID',
  `sku_id` int(11) NOT NULL DEFAULT 0 COMMENT 'SKU ID',
  `store_id` int(11) NOT NULL DEFAULT 0 COMMENT '门店ID',
  `status` tinyint(1) NOT NULL DEFAULT 0 COMMENT '状态：0=待通知 1=已通知',
  `notify_time` int(11) NOT NULL DEFAULT 0 COMMENT '通知时间',
  `create_time` int(11) NOT NULL DEFAULT 0 COMMENT '创建时间',
  PRIMARY KEY (`id`),
  KEY `idx_user` (`user_id`, `status`),
  KEY `idx_sku_store` (`sku_id`, `store_id`, `status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='缺货登记表';
```
