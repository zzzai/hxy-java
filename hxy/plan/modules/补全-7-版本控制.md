# 补全模块7：版本控制机制细化

**优先级**：P1（重要）  
**影响范围**：数据一致性、历史兼容、平滑升级  
**开发工作量**：1天

---

## 一、问题描述

原方案提到"版本控制"，但**实现逻辑不够清晰**，导致：

### 业务痛点

1. **版本切换影响历史业务**
   - 总部修改SKU价格，历史订单价格也变了
   - 技师排班关联的SKU被修改，导致数据不一致

2. **版本过渡期处理不清**
   - 新版本立即生效还是次日生效？
   - 过渡期内，新订单用新版本，老订单用旧版本？

3. **版本回滚困难**
   - 新版本有问题，想回滚到旧版本
   - 没有回滚机制，只能手工修改

4. **版本影响评估缺失**
   - 不知道修改会影响哪些门店
   - 不知道有多少订单、排班需要迁移

---

## 二、解决方案

### 方案核心：版本快照 + 生效时间 + 影响评估 + 回滚机制

## 三、版本控制设计

### 3.1 版本状态流转

```
草稿（draft）
    ↓ 提交审核
待生效（pending）
    ↓ 到达生效时间
已生效（active）
    ↓ 新版本生效
已过期（expired）
    
或

已生效（active）
    ↓ 发现问题
回滚中（rollback）
    ↓ 回滚完成
已生效（active，回到上一版本）
```

### 3.2 版本号规则

```
格式：主版本.次版本.修订号
示例：1.0.0 → 1.0.1 → 1.1.0 → 2.0.0

规则：
- 主版本：重大变更（如SKU规格变更）
- 次版本：功能变更（如价格调整）
- 修订号：小修改（如图片更新）
```

## 五、版本控制逻辑

### 5.1 创建新版本

```php
// app/services/version/VersionService.php

public function createNewVersion($skuId, $updateData, $effectiveTime = 0, $reason = '')
{
    // 1. 获取当前版本
    $currentSku = Db::name('service_sku')->where('id', $skuId)->find();
    
    // 2. 生成新版本号
    $newVersion = $this->generateNextVersion($currentSku['version'], $updateData);
    
    // 3. 保存当前版本到历史表
    $this->saveVersionHistory($currentSku, 'modify', $reason);
    
    // 4. 计算影响范围
    $affected = $this->calculateAffectedRange($skuId, $effectiveTime);
    
    // 5. 创建新版本（草稿状态）
    $versionData = array_merge($updateData, [
        'version' => $newVersion,
        'version_status' => 1,  // 草稿
        'effective_time' => $effectiveTime ?: time(),
        'parent_version_id' => $currentSku['id'],
        'change_log' => json_encode([
            'old_version' => $currentSku['version'],
            'new_version' => $newVersion,
            'changes' => $this->getChangedFields($currentSku, $updateData),
            'reason' => $reason,
            'operator' => request()->uid ?? 0,
            'time' => time()
        ]),
        'affected_stores' => json_encode($affected['stores']),
        'affected_orders' => $affected['orders'],
        'affected_schedules' => $affected['schedules']
    ]);
    
    // 6. 更新SKU
    Db::name('service_sku')->where('id', $skuId)->update($versionData);
    
    return [
        'version' => $newVersion,
        'affected' => $affected,
        'effective_time' => $effectiveTime
    ];
}

private function generateNextVersion($currentVersion, $updateData)
{
    list($major, $minor, $patch) = explode('.', $currentVersion);
    
    // 判断变更级别
    if ($this->isMajorChange($updateData)) {
        // 主版本+1
        return ($major + 1) . '.0.0';
    } elseif ($this->isMinorChange($updateData)) {
        // 次版本+1
        return $major . '.' . ($minor + 1) . '.0';
    } else {
        // 修订号+1
        return $major . '.' . $minor . '.' . ($patch + 1);
    }
}

private function isMajorChange($updateData)
{
    // 规格、时长变更 = 主版本
    $majorFields = ['sku', 'duration', 'spec_type'];
    foreach ($majorFields as $field) {
        if (isset($updateData[$field])) {
            return true;
        }
    }
    return false;
}

private function isMinorChange($updateData)
{
    // 价格、图片变更 = 次版本
    $minorFields = ['guide_price', 'image', 'add_price'];
    foreach ($minorFields as $field) {
        if (isset($updateData[$field])) {
            return true;
        }
    }
    return false;
}

private function calculateAffectedRange($skuId, $effectiveTime)
{
    // 1. 影响的门店
    $stores = Db::name('store_service_sku')
        ->where('sku_id', $skuId)
        ->column('store_id');
    
    // 2. 影响的订单（未完成的）
    $orders = Db::name('order_item')
        ->alias('oi')
        ->join('order o', 'oi.order_id = o.id')
        ->where('oi.item_id', $skuId)
        ->where('o.status', '<', 3)  // 未完成
        ->count();
    
    // 3. 影响的排班（未来的）
    $schedules = Db::name('technician_schedule')
        ->where('service_sku_id', $skuId)
        ->where('work_date', '>=', date('Y-m-d', $effectiveTime))
        ->where('status', 1)  // 可预约
        ->count();
    
    return [
        'stores' => $stores,
        'orders' => $orders,
        'schedules' => $schedules
    ];
}
```

### 5.2 版本审核与生效

```php
public function approveVersion($skuId)
{
    $sku = Db::name('service_sku')->where('id', $skuId)->find();
    
    if ($sku['version_status'] != 1) {
        throw new Exception('只有草稿状态的版本可以审核');
    }
    
    // 1. 更新状态为待生效
    Db::name('service_sku')->where('id', $skuId)->update([
        'version_status' => 2,
        'audit_time' => time()
    ]);
    
    // 2. 如果生效时间已到，立即生效
    if ($sku['effective_time'] <= time()) {
        $this->activateVersion($skuId);
    }
}

public function activateVersion($skuId)
{
    $sku = Db::name('service_sku')->where('id', $skuId)->find();
    
    Db::startTrans();
    try {
        // 1. 将旧版本标记为过期
        if ($sku['parent_version_id'] > 0) {
            Db::name('service_sku')
                ->where('id', $sku['parent_version_id'])
                ->update([
                    'version_status' => 4,  // 已过期
                    'expire_time' => time()
                ]);
        }
        
        // 2. 新版本标记为已生效
        Db::name('service_sku')->where('id', $skuId)->update([
            'version_status' => 3,
            'effective_time' => time()
        ]);
        
        // 3. 同步到门店SKU（仅同步基础信息，不覆盖门店定价）
        $this->syncToStores($sku);
        
        Db::commit();
        
    } catch (Exception $e) {
        Db::rollback();
        throw $e;
    }
}

private function syncToStores($sku)
{
    // 只同步基础信息，不覆盖门店的价格、库存等运营数据
    $syncFields = [
        'sku' => $sku['sku'],
        'image' => $sku['image'],
        'duration' => $sku['duration'],
        'guide_price' => $sku['guide_price'],  // 指导价更新
        'version' => $sku['version'],
        'update_time' => time()
    ];
    
    Db::name('store_service_sku')
        ->where('sku_id', $sku['id'])
        ->update($syncFields);
}
```

### 5.3 版本回滚

```php
public function rollbackVersion($skuId)
{
    $currentSku = Db::name('service_sku')->where('id', $skuId)->find();
    
    if (!$currentSku['parent_version_id']) {
        throw new Exception('没有可回滚的版本');
    }
    
    // 1. 获取父版本数据
    $parentVersion = Db::name('sku_version_history')
        ->where('sku_id', $skuId)
        ->where('version', '<', $currentSku['version'])
        ->order('create_time', 'desc')
        ->find();
    
    if (!$parentVersion) {
        throw new Exception('父版本数据不存在');
    }
    
    // 2. 恢复父版本数据
    $rollbackData = json_decode($parentVersion['version_data'], true);
    $rollbackData['version_status'] = 3;  // 已生效
    $rollbackData['effective_time'] = time();
    
    Db::startTrans();
    try {
        // 3. 保存当前版本到历史
        $this->saveVersionHistory($currentSku, 'rollback', '回滚到版本' . $parentVersion['version']);
        
        // 4. 更新为父版本
        Db::name('service_sku')->where('id', $skuId)->update($rollbackData);
        
        // 5. 同步到门店
        $this->syncToStores($rollbackData);
        
        Db::commit();
        
    } catch (Exception $e) {
        Db::rollback();
        throw $e;
    }
}
```

## 七、定时任务

### 7.1 自动激活待生效版本

```php
// app/command/ActivateVersion.php

protected function execute(Input $input, Output $output)
{
    // 每小时执行一次
    
    // 查询待生效且生效时间已到的版本
    $skus = Db::name('service_sku')
        ->where('version_status', 2)  // 待生效
        ->where('effective_time', '<=', time())
        ->select();
    
    foreach ($skus as $sku) {
        try {
            $versionService = new VersionService();
            $versionService->activateVersion($sku['id']);
            $output->writeln('版本已生效：' . $sku['sku'] . ' v' . $sku['version']);
        } catch (Exception $e) {
            $output->writeln('版本生效失败：' . $e->getMessage());
        }
    }
}
```
