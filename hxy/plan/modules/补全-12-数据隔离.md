# 补全模块12：数据隔离安全机制

**优先级**：P1（重要）  
**影响范围**：数据安全、权限控制、多门店隔离  
**开发工作量**：1天

---

## 一、问题描述

原方案提到"四层隔离"，但**未说明具体实现**，导致：

### 业务痛点

1. **门店数据泄露风险**
   - A店员工可能查询到B店的数据
   - SQL注入可能绕过权限控制

2. **代码层面缺少拦截**
   - 每个查询都要手工加 `where store_id = ?`
   - 容易遗漏，导致数据泄露

3. **权限控制不细化**
   - 门店店长、普通员工权限一样
   - 总部超管权限过大

4. **跨店查询困难**
   - 总部需要查询所有门店数据
   - 但门店只能查自己的数据

---

## 二、解决方案

### 方案核心：ThinkPHP全局范围 + 中间件 + 权限分级 + 审计日志

---

## 三、四层隔离设计

### 3.1 隔离层级

| 层级 | 隔离方式 | 实现位置 | 作用 |
|------|----------|----------|------|
| **代码层** | ThinkPHP全局查询范围 | Model基类 | 自动添加 store_id 条件 |
| **中间件层** | 请求拦截 | Middleware | 验证 token 中的 store_id |
| **数据库层** | 索引优化 | MySQL | 强制使用 store_id 索引 |
| **日志层** | 操作审计 | 日志表 | 记录所有跨店查询尝试 |

## 四、代码层隔离实现

### 4.1 Model基类全局范围

```php
// app/model/BaseModel.php

namespace app\model;

use think\Model;
use think\model\concern\SoftDelete;

class BaseModel extends Model
{
    use SoftDelete;
    
    protected $deleteTime = 'delete_time';
    
    /**
     * 全局查询范围：自动添加门店ID过滤
     */
    protected function base($query)
    {
        // 1. 获取当前用户的门店ID
        $storeId = $this->getCurrentStoreId();
        
        // 2. 如果是门店用户，自动添加门店过滤
        if ($storeId > 0) {
            $query->where($this->table . '.store_id', $storeId);
        }
        
        // 3. 如果是总部用户，不添加过滤（可查询所有门店）
    }
    
    /**
     * 获取当前用户的门店ID
     */
    private function getCurrentStoreId()
    {
        // 从请求中获取（中间件已解析token并注入）
        $request = request();
        
        // 1. 总部用户（role=admin）
        if ($request->userRole == 'admin') {
            return 0;  // 0表示不限制门店
        }
        
        // 2. 门店用户
        if ($request->userRole == 'store') {
            return $request->storeId ?? 0;
        }
        
        // 3. 普通用户（C端用户）
        return 0;  // 不限制门店
    }
    
    /**
     * 跨店查询（需要明确调用）
     */
    public function withoutStoreScope()
    {
        return $this->withoutGlobalScope(['base']);
    }
}
```

### 4.2 具体Model使用

```php
// app/model/StoreServiceSku.php

namespace app\model;

class StoreServiceSku extends BaseModel
{
    protected $name = 'store_service_sku';
    
    // 自动继承全局范围，无需额外代码

// 2. 跨店查询（总部使用）
```

## 五、中间件层隔离

### 5.1 门店权限中间件

```php
// app/middleware/StoreAuthMiddleware.php

namespace app\middleware;

use Closure;
use think\Request;
use think\Response;
use think\facade\Db;

class StoreAuthMiddleware
{
    /**
     * 处理请求
     */
    public function handle(Request $request, Closure $next): Response
    {
        // 1. 获取token
        $token = $request->header('Authorization');
        
        if (!$token) {
            return json(['code' => 401, 'msg' => '未登录']);
        }
        
        // 2. 解析token
        $tokenData = $this->parseToken($token);
        
        if (!$tokenData) {
            return json(['code' => 401, 'msg' => 'token无效']);
        }
        
        // 3. 注入用户信息到请求
        $request->uid = $tokenData['uid'];
        $request->userRole = $tokenData['role'];
        $request->storeId = $tokenData['store_id'] ?? 0;
        $request->userName = $tokenData['name'] ?? '';
        
        // 4. 检查门店权限
        if ($tokenData['role'] == 'store') {
            // 门店用户只能访问自己门店的数据
            $requestStoreId = $request->param('store_id', 0);
            
            if ($requestStoreId > 0 && $requestStoreId != $tokenData['store_id']) {
                // 记录越权尝试
                $this->logUnauthorizedAccess($tokenData, $requestStoreId);
                
                return json(['code' => 403, 'msg' => '无权访问其他门店数据']);
            }
        }
        
        return $next($request);
    }
    
    /**
     * 解析token
     */
    private function parseToken($token)
    {
        // 使用JWT或其他方式解析token
        // 这里简化处理
        $tokenData = Db::name('user_token')
            ->where('token', $token)
            ->where('expire_time', '>', time())
            ->find();
        
        if (!$tokenData) {
            return false;
        }
        
        return [
            'uid' => $tokenData['uid'],
            'role' => $tokenData['role'],
            'store_id' => $tokenData['store_id'],
            'name' => $tokenData['name']
        ];
    }
    
    /**
     * 记录越权访问尝试
     */
    private function logUnauthorizedAccess($tokenData, $requestStoreId)
    {
        Db::name('security_log')->insert([
            'log_type' => 'unauthorized_access',
            'user_id' => $tokenData['uid'],
            'user_role' => $tokenData['role'],
            'user_store_id' => $tokenData['store_id'],
            'request_store_id' => $requestStoreId,
            'request_url' => request()->url(),
            'request_ip' => request()->ip(),
            'create_time' => time()
        ]);
    }
}
```

## 六、权限分级设计

### 6.1 角色权限定义

| 角色 | 权限范围 | 数据范围 | 典型操作 |
|------|----------|----------|----------|
| **总部超管** | 全部权限 | 所有门店 | 查询所有门店数据、修改总部配置 |
| **总部运营** | 部分权限 | 所有门店（只读） | 查询所有门店数据、生成报表 |
| **门店店长** | 门店管理 | 本门店 | 管理本店商品、订单、员工 |
| **门店员工** | 基础操作 | 本门店 | 核销订单、查询库存 |
| **技师** | 服务操作 | 本门店 | 查看排班、核销服务 |

### 6.2 权限控制表

```php
// app/services/auth/PermissionService.php

class PermissionService
{
    /**
     * 权限配置
     */
    private $permissions = [
        'admin' => [
            'store' => ['*'],  // 所有门店
            'actions' => ['*']  // 所有操作
        ],
        'store_manager' => [
            'store' => ['self'],  // 仅本店
            'actions' => [
                'store.view',
                'store.edit',
                'product.manage',
                'order.manage',
                'staff.manage'
            ]
        ],
        'store_staff' => [
            'store' => ['self'],
            'actions' => [
                'order.view',
                'order.verify',
                'product.view'
            ]
        ],
        'technician' => [
            'store' => ['self'],
            'actions' => [
                'schedule.view',
                'order.verify',
                'commission.view'
            ]
        ]
    ];
    
    /**
     * 检查权限
     */
    public function checkPermission($role, $action, $storeId = 0)
    {
        // 1. 获取角色权限
        $rolePermissions = $this->permissions[$role] ?? [];
        
        // 2. 检查操作权限
        if (!in_array('*', $rolePermissions['actions']) 
            && !in_array($action, $rolePermissions['actions'])) {
            return false;
        }
        
        // 3. 检查门店权限
        if ($storeId > 0) {
            $allowedStores = $rolePermissions['store'] ?? [];
            
            if (!in_array('*', $allowedStores) && !in_array('self', $allowedStores)) {
                return false;
            }
            
            // 如果是self，检查是否是本店
            if (in_array('self', $allowedStores)) {
                $userStoreId = request()->storeId ?? 0;
                if ($storeId != $userStoreId) {
                    return false;
                }
            }
        }
        
        return true;
    }
}
```

## 七、数据库层优化

### 7.1 强制索引

```sql

-- 订单表

ADD KEY `idx_store_id` (`store_id`);

ALTER TABLE `eb_order_item`
```

```sql
  KEY `idx_type` (`log_type`, `create_time`),

) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='安全日志表';
```

### 8.2 日志记录

```php
// app/services/log/SecurityLogService.php

class SecurityLogService
{
    /**
     * 记录安全日志
     */
    public function log($type, $data = [])
    {
        $request = request();
        
        Db::name('security_log')->insert([
            'log_type' => $type,
            'user_id' => $request->uid ?? 0,
            'user_role' => $request->userRole ?? '',
            'user_store_id' => $request->storeId ?? 0,
            'request_store_id' => $data['request_store_id'] ?? 0,
            'request_url' => $request->url(),
            'request_ip' => $request->ip(),
            'request_data' => json_encode($data),
            'response_code' => $data['response_code'] ?? 0,
            'create_time' => time()
        ]);
    }
}
```

## 九、特殊场景处理

### 9.1 总部查询所有门店数据

```php
// 总部生成报表，需要查询所有门店

public function generateReport()
{
    // 检查权限
    if (request()->userRole != 'admin') {
        throw new Exception('无权限');
    }
    
    // 使用 withoutStoreScope 跨店查询
    $allOrders = Order::withoutStoreScope()
        ->where('create_time', '>=', strtotime('-30 days'))
        ->select();
    
    // 记录日志
    SecurityLogService::log('cross_store_query', [
        'action' => 'generate_report',
        'store_count' => count(array_unique(array_column($allOrders, 'store_id')))
    ]);
    
    return $allOrders;
}
```

### 9.2 用户跨店下单

```php
// 用户在A店下单，但选择B店的商品

public function createOrder($userId, $storeId, $skuId)
{
    // 用户可以跨店下单，不受门店限制
    // 但要记录日志
    
    $userLastStore = $this->getUserLastStore($userId);
    
    if ($userLastStore != $storeId) {
        SecurityLogService::log('cross_store_order', [
            'user_id' => $userId,
            'last_store' => $userLastStore,
            'current_store' => $storeId
        ]);
    }
    
    // 正常创建订单
    return $this->doCreateOrder($userId, $storeId, $skuId);
}
```

## 十、前端配合

### 10.1 请求拦截器

```javascript
// 前端请求拦截器

axios.interceptors.request.use(config => {
    // 1. 添加token
    const token = localStorage.getItem('token');
    if (token) {
        config.headers['Authorization'] = `Bearer ${token}`;
    }
    
    // 2. 添加门店ID（如果是门店端）
    const storeId = localStorage.getItem('storeId');
    if (storeId && config.params) {
        config.params.store_id = storeId;
    }
    
    return config;
});

// 响应拦截器
axios.interceptors.response.use(
    response => response,
    error => {
        if (error.response?.status === 403) {
            // 无权限，跳转到错误页
            router.push('/error/403');
        }
        return Promise.reject(error);
    }
);
```
