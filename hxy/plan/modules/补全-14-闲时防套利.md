# 补全模块14：闲时优惠防套利机制

**优先级**：P2（建议）  
**影响范围**：营销活动、风控保护、用户体验  
**开发工作量**：1天

---

## 一、问题描述

原方案提到"闲时优惠"，但**未说明防滥用机制**，导致：

### 业务痛点

1. **恶意占位**
   - 用户批量预约闲时时间槽
   - 实际不到店消费，占用资源

2. **黄牛转卖**
   - 用户低价预约后转卖给他人
   - 影响真实需求用户预约

3. **频繁取消**
   - 用户预约后频繁取消
   - 浪费门店资源

4. **账号刷单**
   - 注册多个账号批量预约
   - 薅羊毛行为

---

## 二、解决方案

### 方案核心：使用限制 + 取消惩罚 + 实名认证 + 行为监控

**注意**：服务类商品天然防转卖（实名制+到店核销），重点防恶意占位。

---

## 三、防套利规则设计

### 3.1 闲时优惠使用限制

| 限制类型 | 规则 | 说明 |
|---------|------|------|
| **每日限制** | 每人每天最多预约2次闲时优惠 | 防止单日批量预约 |
| **每周限制** | 每人每周最多预约5次闲时优惠 | 防止长期占用 |
| **每月限制** | 每人每月最多预约15次闲时优惠 | 总量控制 |
| **单次预约限制** | 每次只能预约1个时间槽 | 防止批量预约 |
| **提前预约限制** | 最多提前3天预约闲时优惠 | 防止囤积 |

---

### 3.2 取消惩罚机制

| 取消时间 | 惩罚措施 | 说明 |
|---------|---------|------|
| **24小时前** | 无惩罚，全额退款 | 正常取消 |
| **24小时内** | 扣除50%，禁止预约闲时7天 | 轻度惩罚 |
| **2小时内** | 扣除100%，禁止预约闲时30天 | 重度惩罚 |
| **未到店** | 扣除100%，禁止预约闲时90天 | 最严厉惩罚 |
| **月累计取消3次** | 禁止预约闲时30天 | 累计惩罚 |

---

### 3.3 实名认证要求

| 认证级别 | 要求 | 闲时优惠权限 |
|---------|------|-------------|
| **未认证** | 无 | 不可使用闲时优惠 |
| **手机认证** | 绑定手机号 | 可使用，每日限1次 |
| **实名认证** | 身份证认证 | 可使用，每日限2次 |

## 五、防套利逻辑实现

### 5.1 预约前检查

```php
// app/services/offpeak/OffpeakService.php

public function checkOffpeakLimit($userId, $serviceTime)
{
    // 1. 检查实名认证
    $user = Db::name('user')->where('id', $userId)->find();
    
    if (!$user['mobile']) {
        throw new Exception('请先绑定手机号');
    }
    
    if (!$user['id_card']) {
        throw new Exception('闲时优惠需要实名认证');
    }
    
    // 2. 检查是否被限制
    $limit = Db::name('user_offpeak_limit')
        ->where('user_id', $userId)
        ->where('limit_end_time', '>', time())
        ->find();
    
    if ($limit) {
        throw new Exception('您已被限制预约闲时优惠，限制至：' . date('Y-m-d H:i', $limit['limit_end_time']));
    }
    
    // 3. 检查每日限制
    $todayCount = Db::name('user_offpeak_record')
        ->where('user_id', $userId)
        ->where('create_time', '>=', strtotime('today'))
        ->where('status', '<>', 2)  // 排除已取消
        ->count();
    
    $dailyLimit = $user['id_card'] ? 2 : 1;  // 实名认证2次，否则1次
    
    if ($todayCount >= $dailyLimit) {
        throw new Exception('今日闲时优惠预约次数已达上限');
    }
    
    // 4. 检查每周限制
    $weekCount = Db::name('user_offpeak_record')
        ->where('user_id', $userId)
        ->where('create_time', '>=', strtotime('this week'))
        ->where('status', '<>', 2)
        ->count();
    
    if ($weekCount >= 5) {
        throw new Exception('本周闲时优惠预约次数已达上限');
    }
    
    // 5. 检查每月限制
    $monthCount = Db::name('user_offpeak_record')
        ->where('user_id', $userId)
        ->where('create_time', '>=', strtotime('first day of this month'))
        ->where('status', '<>', 2)
        ->count();
    
    if ($monthCount >= 15) {
        throw new Exception('本月闲时优惠预约次数已达上限');
    }
    
    // 6. 检查提前预约限制
    $maxAdvanceDays = 3;
    $maxServiceTime = strtotime('+' . $maxAdvanceDays . ' days');
    
    if ($serviceTime > $maxServiceTime) {
        throw new Exception('闲时优惠最多提前' . $maxAdvanceDays . '天预约');
    }
    
    return true;
}
```

### 5.3 取消预约处理

```php
public function cancelOffpeakBooking($userId, $orderId)
{
    // 1. 获取预约记录
    $record = Db::name('user_offpeak_record')
        ->where('user_id', $userId)
        ->where('order_id', $orderId)
        ->where('status', 0)
        ->find();
    
    if (!$record) {
        throw new Exception('预约记录不存在');
    }
    
    // 2. 计算取消时间差
    $now = time();
    $serviceTime = $record['service_time'];
    $timeDiff = $serviceTime - $now;
    
    // 3. 判断惩罚措施
    $penaltyAmount = 0;
    $banDays = 0;
    $refundRate = 1.0;
    
    if ($timeDiff >= 86400) {
        // 24小时前取消，无惩罚
        $penaltyAmount = 0;
        $banDays = 0;
        $refundRate = 1.0;
    } elseif ($timeDiff >= 7200) {
        // 24小时内但2小时前，扣50%，禁7天
        $penaltyAmount = $record['order_amount'] * 0.5;
        $banDays = 7;
        $refundRate = 0.5;
    } else {
        // 2小时内，扣100%，禁30天
        $penaltyAmount = $record['order_amount'];
        $banDays = 30;
        $refundRate = 0;
    }
    
    // 4. 更新记录
    Db::name('user_offpeak_record')
        ->where('id', $record['id'])
        ->update([
            'status' => 2,  // 已取消
            'cancel_time' => $now,
            'penalty_amount' => $penaltyAmount
        ]);
    
    // 5. 添加限制
    if ($banDays > 0) {
        $this->addOffpeakLimit($userId, $banDays, '取消闲时预约');
    }
    
    // 6. 检查月累计取消次数
    $monthCancelCount = Db::name('user_offpeak_record')
        ->where('user_id', $userId)
        ->where('status', 2)
        ->where('cancel_time', '>=', strtotime('first day of this month'))
        ->count();
    
    if ($monthCancelCount >= 3) {
        $this->addOffpeakLimit($userId, 30, '月累计取消3次');
    }
    
    return [
        'refund_rate' => $refundRate,
        'penalty_amount' => $penaltyAmount,
        'ban_days' => $banDays
    ];
}

private function addOffpeakLimit($userId, $days, $reason)
{
    Db::name('user_offpeak_limit')->insert([
        'user_id' => $userId,
        'limit_type' => 1,
        'limit_reason' => $reason,
        'limit_start_time' => time(),
        'limit_end_time' => strtotime('+' . $days . ' days'),
        'create_time' => time()
    ]);
}
```

public function handleNoShow($userId, $orderId)
{
    // 1. 更新记录
    Db::name('user_offpeak_record')
        ->where('user_id', $userId)
        ->where('order_id', $orderId)
        ->update([
            'status' => 3,  // 未到店
            'penalty_amount' => Db::raw('order_amount')  // 扣除全额
        ]);
    
    // 2. 禁止预约90天
    $this->addOffpeakLimit($userId, 90, '预约后未到店');
    
    // 3. 记录黑名单
    $this->addToBlacklist($userId, 'offpeak_no_show');
}

## 六、行为监控

### 6.1 异常行为检测

```php
// app/services/risk/RiskMonitorService.php

public function detectAbnormalBehavior($userId)
{
    $risks = [];
    
    // 1. 检测批量预约
    $recentBookings = Db::name('user_offpeak_record')
        ->where('user_id', $userId)
        ->where('create_time', '>=', time() - 3600)  // 1小时内
        ->count();
    
    if ($recentBookings >= 3) {
        $risks[] = '1小时内预约' . $recentBookings . '次，疑似批量预约';
    }
    
    // 2. 检测频繁取消
    $cancelRate = $this->getCancelRate($userId);
    
    if ($cancelRate > 0.5) {
        $risks[] = '取消率' . ($cancelRate * 100) . '%，疑似恶意占位';
    }
    
    // 3. 检测多账号关联
    $user = Db::name('user')->where('id', $userId)->find();
    $sameDeviceUsers = Db::name('user')
        ->where('device_id', $user['device_id'])
        ->where('id', '<>', $userId)
        ->count();
    
    if ($sameDeviceUsers >= 3) {
        $risks[] = '同设备' . ($sameDeviceUsers + 1) . '个账号，疑似刷单';
    }
    
    // 4. 如果有风险，记录日志
    if (!empty($risks)) {
        Db::name('risk_log')->insert([
            'user_id' => $userId,
            'risk_type' => 'offpeak_abuse',
            'risk_detail' => json_encode($risks),
            'create_time' => time()
        ]);
    }
    
    return $risks;
}

private function getCancelRate($userId)
{
    $total = Db::name('user_offpeak_record')
        ->where('user_id', $userId)
        ->count();
    
    if ($total == 0) {
        return 0;
    }
    
    $canceled = Db::name('user_offpeak_record')
        ->where('user_id', $userId)
        ->where('status', 2)
        ->count();
    
    return $canceled / $total;
}
```

## 九、总结

1. **防止恶意占位**：使用次数限制
2. **防止频繁取消**：取消惩罚机制
3. **防止批量刷单**：实名认证+行为监控
4. **保护真实用户**：确保闲时优惠给真正需要的用户

### 关键规则
- ✅ 实名认证才能使用
- ✅ 每日/周/月次数限制
- ✅ 取消惩罚（时间越近惩罚越重）
- ✅ 未到店重罚（禁90天）
- ✅ 行为监控（异常检测）
