## 🎯 方案概述

### 设计目标

本方案针对社区小店的**4大盲区**进行补全：

| 维度 | 痛点 | 解决方案 | 核心表/字段 |
|------|------|----------|------------|
| **财务维度** | 组合订单退款时无法精确计算子项退款金额 | "Penny Perfect Split" 分摊算法 | `eb_store_order_cart_info.real_split_price` |
| **履约维度** | 服务完成后物料消耗无法自动扣减库存 | BOM 自动扣减机制 | `eb_service_bom` + `eb_stock_flow` |
| **营销维度** | 闲时优惠、会员价、优惠券叠加规则混乱 | 定价策略流水线 | `eb_pricing_strategy` |
| **技师维度** | 技师技能与服务匹配关系缺失 | 技师技能矩阵 | `eb_technician_skill` |

### 架构特点

✅ **极简设计**: 不引入重型 ERP/WMS，仅用最少字段解决核心问题  
✅ **财务合规**: 分摊算法确保 `Sum(Split) === Total Paid`（精确到分）  
✅ **事务安全**: 所有库存操作使用数据库事务保证原子性  
✅ **可追溯性**: 完整的库存流水记录（支持按流水类型查询）

## 🧮 核心算法实现

### 1. 财务分摊算法 (Penny Perfect Split)

**文件**: `app/services/order/OrderPriceService.php`

#### 算法原理

```
假设订单：
- 子项A: 原价 100元，数量 1
- 子项B: 原价 50元，数量 2
- 原价总和: 100 + 50*2 = 200元
- 用户实付: 135元（使用了优惠券）

传统错误做法（会产生尾差）:
- 子项A分摊: 135 * (100/200) = 67.50元
- 子项B分摊: 135 * (100/200) = 67.50元
- 总和: 67.50 + 67.50 = 135.00元 ✅

但如果实付是 135.01元：
- 子项A分摊: 135.01 * (100/200) = 67.505 → 四舍五入 67.51元
- 子项B分摊: 135.01 * (100/200) = 67.505 → 四舍五入 67.51元
- 总和: 67.51 + 67.51 = 135.02元 ❌ 多了0.01元！

正确做法（尾差兜底）:
- 子项A分摊: 135.01 * (100/200) = 67.51元（四舍五入）
- 子项B分摊: 135.01 - 67.51 = 67.50元（兜底）
- 总和: 67.51 + 67.50 = 135.01元 ✅
```

#### 核心代码

```php
// 最后一个子项：兜底尾差
if ($index == $count) {
    $splitPrice = bcsub($totalPayPrice, $currentSumSplit, 2);
} else {
    $splitPrice = bcmul($totalPayPrice, $weight, 2);
    $currentSumSplit = bcadd($currentSumSplit, $splitPrice, 2);
}
```

#### 调用示例

```php
use app\services\order\OrderPriceService;

// 订单支付成功后调用
$orderPriceService = app()->make(OrderPriceService::class);
$orderPriceService->calculateSplitPrice($orderId, $totalPayPrice);
```

### 2. BOM 自动扣减算法

**文件**: `app/services/product/ProductStockService.php`

#### 算法流程

```
1. 技师点击"完成服务"
   ↓
2. 查询服务的BOM清单
   ↓
3. 区分必耗品和可选品
   ↓
4. 检查库存是否充足
   ↓
5. 扣减库存（支持小数）
   ↓
6. 记录流水（flow_type=2）
```

#### 核心代码

```php
// 判断是否需要扣减
$shouldDeduct = $bom['is_mandatory'] == 1 || in_array($bom['product_sku_id'], $optionalSkus);

if ($shouldDeduct) {
    // 扣减库存
    $newStock = bcsub($storeSku['stock'], $bom['quantity'], 2);
    
    // 记录流水（flow_type=2 服务消耗）
    $this->stockFlowDao->save([
        'flow_type' => 2,
        'num' => -$bom['quantity'],
        'remark' => "技师[{$technicianId}]服务消耗"
    ]);
}
```

#### 调用示例

```php
use app\services\product\ProductStockService;

// 技师完成服务时调用
$stockService = app()->make(ProductStockService::class);
$result = $stockService->deductServiceBom(
    $storeId,
    $serviceId,
    $orderId,
    $technicianId,
    ['optional_sku_1', 'optional_sku_2'] // 用户选中的可选物料
);
```

## 🎓 最佳实践

### 1. 财务分摊的调用时机

✅ **正确**: 订单支付成功后立即调用  
❌ **错误**: 退款时临时计算

```php
// 正确做法
public function paySuccess($orderId, $totalPayPrice)
{
    // 1. 更新订单状态
    Order::where('id', $orderId)->update(['paid' => 1]);
    
    // 2. 立即计算分摊金额
    $orderPriceService->calculateSplitPrice($orderId, $totalPayPrice);
    
    // 3. 其他业务逻辑...
}
```

### 2. BOM 扣减的事务处理

✅ **正确**: 使用数据库事务确保原子性  
❌ **错误**: 逐个扣减，中途失败导致数据不一致

```php
// 正确做法（已在 ProductStockService 中实现）
Db::transaction(function () use ($bomList) {
    foreach ($bomList as $bom) {
        // 扣减库存
        // 记录流水
    }
});
```

### 3. 库存流水的查询优化

✅ **正确**: 使用 `flow_type` 索引查询  
❌ **错误**: 全表扫描

```php
// 正确做法
$serviceFlows = StockFlow::where('store_id', $storeId)
    ->where('flow_type', 2) // 使用索引
    ->whereBetweenTime('create_time', $startDate, $endDate)
    ->select();
```
