## 二、安全问题（高优先级）

### 2.1 配置文件明文密码

以下文件含数据库/Redis/Swagger 明文密码，且部分已被本仓或子模块仓库跟踪：

| 文件 | 风险 |
|------|------|
| `crmeb_java/crmeb/crmeb-admin/src/main/resources/application-dev.yml` | `password: crmeb123` |
| `crmeb_java/crmeb/crmeb-front/.../application-dev.yml` | 同上 |
| `application-beta.yml` / `application-trip.yml` / `application-prod.yml` | `password: 111111`、Redis `111111`、Swagger `crmeb.com` |
| `hxy/local_archive/**/application-dev.yml` | 已在本仓跟踪，含相同密码 |

**建议**（任选一种并贯彻）：

1. **环境变量**：`application.yml` 仅写占位，密码用 `spring.datasource.password: ${DB_PASSWORD}`，部署时注入环境变量。
2. **本地不提交**：在根目录及 `crmeb_java/crmeb/.gitignore` 中增加：
   - `**/application-dev.yml`
   - `**/application-beta.yml`
   - `**/application-trip.yml`
   - `**/application-prod.yml`  
   并提交「示例」文件如 `application-dev.yml.sample`（无真实密码）。
3. 若已推送到远程：在 GitHub 删除对应文件历史或轮换所有已暴露密码（DB、Redis、Swagger 等）。

### 2.2 Token 与鉴权

- Admin/Front 使用 **Redis 存 token**（`TokenComponent` / `FrontTokenComponent`），非 JWT 签名，无密钥配置问题。
- 请求头常量：`Constants.HEADER_AUTHORIZATION_KEY = "Authori-zation"` 为**拼写错误**，应为 `Authorization`。若前端/网关按标准 `Authorization` 传，会导致鉴权失败，需统一改为 `Authorization` 并回归接口与 Swagger。

## 三、依赖与版本

| 依赖 | 当前版本 | 建议 |
|------|----------|------|
| **fastjson** | 1.2.83 | 1.x 有历史 CVE；建议升级到 **fastjson2**（如 2.0.43+）或逐步改用 Jackson，避免反序列化不可信 JSON。 |
| **Spring Boot** | 2.2.6 | 已 EOL；建议规划升级到 2.7.x 或 3.x（含 JDK 与依赖兼容）。 |
| **Druid** | 1.1.20 | 建议升级到 1.2.x 获取安全与稳定性修复。 |
| **JJWT** | 0.9.1 | 若后续启用 JWT 签名，需使用 0.12.x 并配合密钥配置。 |

## 五、其他
