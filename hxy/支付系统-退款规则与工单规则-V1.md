# 支付系统-退款规则与工单规则-V1

更新时间：2026-02-18

## 1. 目标

- 正常退款：尽量秒退。
- 异常退款：进入总部工单池处理。
- 门店职责：仅履约，不承担退款打款动作。

## 2. 退款规则（当前实现）

用户在前端发起退款申请后，系统先写入申请状态，再执行路由判定：

1. 命中自动秒退规则：
   - 直接由总部支付中台发起退款（不要求门店提工单）。
   - 订单日志写入 `refund_auto_submit`。
2. 未命中自动秒退规则：
   - 进入总部退款工单池（`refund_status=1`）。
   - 订单日志写入 `refund_ticket`。
3. 自动秒退提交失败：
   - 自动降级为总部人工工单。
   - 订单日志写入 `refund_ticket`（含失败原因）。

## 3. 自动秒退判定条件

全部满足才自动秒退：

1. 自动秒退开关开启（默认开启）。
2. 订单金额合法且大于 0。
3. 支付方式在自动范围内（微信/余额/支付宝）。
4. 订单状态仍为已支付待履约（`status=0`）。
5. 微信单要求有 `out_trade_no`。
6. 支付金额不超过自动秒退阈值。
7. 申请时间未超过自动秒退时效窗口。
8. 退款原因未命中高风险关键词。

## 4. 工单规则

以下场景自动进入总部工单池：

1. 自动秒退开关关闭。
2. 订单已进入履约过程（非 `status=0`）。
3. 金额超阈值、超时效。
4. 命中风险关键词（如洗钱/套现/盗刷/诈骗/非法）。
5. 自动退款提交失败（渠道错误、签名错误、受理关系异常等）。

工单池承载方式：

- 当前版本以 `refund_status=1` + `eb_store_order_status.change_type=refund_ticket` 实现。
- 总部运营/财务在后台订单售后列表统一处理。
- 新增总部工单池接口：`GET /api/admin/store/order/refund/ticket/list`（只返回 `refund_ticket` 单据）。

## 5. 可配置参数（系统配置表）

1. `payment_refund_auto_enable`：自动秒退开关，`1/0`，默认 `1`。
2. `payment_refund_auto_max_minutes`：自动秒退时效（分钟），默认 `120`。
3. `payment_refund_auto_max_amount`：自动秒退金额上限（元），默认 `200.00`。
4. `payment_refund_manual_keywords`：命中后转人工关键词（逗号分隔）。

## 6. 建议运营基线

1. 首月建议：
   - 金额阈值：`100~200`
   - 时效窗口：`60~120 分钟`
2. 观察指标：
   - 自动秒退命中率
   - 自动秒退失败率
   - 人工工单平均处理时长
3. 每周复盘一次关键词与阈值，逐步扩大自动秒退覆盖面。

## 7. 总部工单池接口（已落地）

1. 列表接口：
   - `GET /api/admin/store/order/refund/ticket/list`
2. 权限：
   - 复用订单列表权限 `admin:order:list`
3. 行为：
   - 服务端自动注入 `refundTicketOnly=true`
   - 查询条件增加 `exists eb_store_order_status.change_type='refund_ticket'`
4. 用法建议：
   - 总部售后工作台优先使用该接口，不再让门店直接执行退款动作。

## 8. 日报新增指标（已落地）

`shell/payment_daily_report.sh` 已新增当日退款自动化统计：

1. `refund_auto_submit`：自动秒退提交命中次数。
2. `refund_ticket_total`：进入总部工单池总次数。
3. `refund_ticket_submit_failed`：自动秒退提交失败后转工单次数。
4. `refund_ticket_rule_routed`：规则直转工单次数（`total - submit_failed`）。

说明：

- 统计源：`eb_store_order_status`（按 `create_time` 当日窗口）。
- 无数据库连接或凭据不可用时，指标显示 `-`，并标记 `refund_metrics_source=db_unavailable`。
