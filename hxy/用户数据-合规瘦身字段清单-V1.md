# 用户数据-合规瘦身字段清单 V1

> 状态：执行版（两周落地）  
> 日期：2026-02-16  
> 适用范围：荷小悦连锁门店（线下服务 + 小程序）

## 1. 执行目标

1. 以合法合规为底线，先上线最小可用数据集，不做过度采集。
2. 每个字段必须绑定业务目的和运营动作，不能“先采再说”。
3. 对高敏感和高争议字段实施默认关闭、按场景授权、可撤回。

## 2. 合规红线（上线门槛）

1. 健康信息、生物识别信息属于敏感个人信息，必须单独同意后采集。
2. 公共场所人脸识别不得用于运营画像、营销分析、会员签到等非公共安全目的。
3. 用户可查看、导出、删除、撤回授权，且需要有可执行入口与工单闭环。
4. 无法说明目的、无法绑定动作、无法设置保留期的字段，一律不上线。

## 4. 明确禁采（V1直接禁止）

1. 身份证号、精确家庭住址、与服务无关的联系人信息。
2. 通过爬取或抓取外部社交平台补充用户画像。
3. 在公共区域使用人脸识别做营销分析、客群识别、会员签到。

## 5. 两周落地计划

### 第1周（字段治理 + 采集面）

1. 输出字段字典（A/B/C三层）并冻结版本 `v1.0.0`。
2. 小程序与技师端表单改造为A层字段；B层入口加“未开通”占位。
3. 新增字段级校验规则：枚举校验、范围校验、空值策略。
4. 采集埋点加 `purpose_code`、`consent_id`、`field_set_version`。

### 第2周（联调 + 稽核）

1. 接入授权校验：无 `consent_health_basic_id` 时禁止采健康字段。
2. 上线字段稽核报表：缺失率、冲突率、越权访问率。
3. 进行门店灰度（2-5店）并复盘采集时长与转化影响。
4. 验收标准：A层完整率 >= 85%，越权采集 = 0。

## 6. 与原方案的裁剪关系

1. 原 `aiuser.md` 中 IoT/图像/外部补充属于B或C层，不在首两周全量上线。
2. 高争议标签保留“定义”，但先不用于自动决策，详见 `用户数据-高争议标签治理规则-V1.md`。
3. 触达系统只允许使用A层低风险标签，B/C层默认禁用。
