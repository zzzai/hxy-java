# HXY 项目章程 V1（2026-02-22）

## 1. 项目定义

1. 项目名称：荷小悦（HXY）连锁 O2O 平台。
2. 项目目标：构建“总部统一治理 + 门店标准履约 + 用户稳定体验”的数字化底座。
3. 代码底座：`ruoyi-vue-pro-master`。
4. 知识库与长记忆目录：`/root/crmeb-java/hxy`。

## 2. 当前阶段目标

1. P0：支付与交易主链路稳定可用（下单、支付、回调、退款、对账、审计）。
2. P1：上线拦截规则、异常补偿、CI 验收闭环。
3. P2：非支付模块二开（商品、会员、门店运营、营销）。

## 3. 治理原则

1. 命名统一：新增命名一律使用 `hxy`，禁止新增 `crmeb` 命名资产。
2. 资金安全优先：任何上线动作不得牺牲支付一致性与审计可追溯性。
3. 文档即资产：关键决策、验收命令、回滚步骤必须文档化。
4. 单一知识入口：主线以本目录为准，历史资产进入归档区。

## 4. 权限边界

1. 总部：支付执行、退款执行、对账处理与发布决策。
2. 门店：仅履约与售后材料处理，不直接执行退款打款。
3. 系统：所有资金关键操作必须有审计日志。
