# HXY 项目事实基线（v1）

- 日期：2026-03-01
- 用途：作为开发与评审的唯一事实基线（Single Source of Truth）

## 1. 品牌与业务

1. 品牌：荷小悦（新中式·疗愈·社区）。
2. 业务形态：服务项目 + 实物商品 + 套餐组合混合 O2O。
3. 连锁目标：先跑通单店，再复制到 50 店，再区域化扩张。

## 2. 核心业务约束

1. 服务商品核心容量是技师时间，不以库存扣减为核心。
2. 实物商品核心容量是库存，必须具备幂等库存流水。
3. 套餐必须拆子项，按子项履约与退款。
4. 技师提成只针对服务子项，按履约完成计提。

## 3. 组织边界

1. 总部负责：主数据标准、价格边界、活动规则、结算治理。
2. 门店负责：上架、排班、门店价执行、现场履约。
3. 线上配置权限：仅保留总部与门店两级（不设区域配置层）。
4. 线下运行职能：训练、巡检、突发处置可存在，但通过总部工单/台账协同，不形成独立线上配置层。
5. 客服负责：统一售后工单受理、分级、升级与闭环。
6. 供应链负责：补货、调拨、盘点、损耗治理。
7. 财务与审计负责：月结、冲正、四账一致性与审计留痕。

## 4. 交易与支付

1. 支付架构：服务商模式 + 特约商户体系。
2. 资金原则：合规分账、可对账、可审计、可追溯。
3. 退款原则：与履约状态和子项状态绑定。

## 5. 当前工程并轨原则

1. 保留现有 `product_spu/product_sku/trade_order` 主链。
2. 通过增量改造补齐强约束，不做一次性推翻重写。
3. 所有新能力必须可回退。

## 6. 不可破坏项

1. 下单/支付主链稳定性。
2. 退款与对账可追溯性。
3. 多门店隔离与权限边界。
4. 价格与库存幂等一致性。
