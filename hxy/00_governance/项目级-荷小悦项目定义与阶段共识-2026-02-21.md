# 项目级：荷小悦项目定义与阶段共识

- 日期：2026-02-21
- 项目代号：`HXY / 荷小悦`
- 当前唯一代码底座：`/root/crmeb-java/ruoyi-vue-pro-master`

## 1. 荷小悦是什么（核心定义）

1. 荷小悦是一个“社区连锁轻养生”项目，不是单纯电商站点。
2. 业务形态是“线下门店履约 + 线上小程序交易”的 O2O 系统。
3. 目标是总部统一治理、门店标准履约、用户稳定体验，并逐步走向连锁规模化。

## 2. 当前系统要解决的问题

1. 先把交易与支付主链路跑稳：
   - 下单 -> 支付 -> 回调 -> 退款 -> 对账 -> 审计。
2. 先保证资金与订单一致性，再扩展非支付能力。
3. 把上线风险前置为“自动拦截规则”，避免带病发布。

## 3. 组织与权限共识

1. 总部是唯一资金执行入口（支付治理、退款执行、对账处理）。
2. 门店只负责履约与售后材料，不负责退款打款。
3. 高风险退款走总部审核，低风险退款尽量秒退。

## 4. 当前阶段共识（2026-02-21）

1. 技术主线已切到 RuoYi，CRMEB 仅保留兼容参考，不再作为主底座推进。
2. 支付系统已进入“阶段 A 内功收敛 + 阶段 B 等待特约商户参数”模式。
3. `sub_mchid` 到位前，持续完成可独立推进的自动化、补偿、门禁与验收能力。

## 5. 非目标（当前阶段不做）

1. 不做大规模前端重构与视觉重做。
2. 不做“先上线后补幂等/补偿”的高风险交付。
3. 不在 P0 未收敛前并行插入大量新需求。

## 6. 北极星指标（项目级）

1. 支付成功率与回调收敛时延（P95）。
2. 退款终态收敛率与平均收敛时长。
3. 对账差异发现率与工单闭环时长。
4. 发布后 P0 故障次数与回滚触发次数。
