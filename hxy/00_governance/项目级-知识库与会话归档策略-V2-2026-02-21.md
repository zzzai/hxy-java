# 项目级：知识库与会话归档策略 V2

- 日期：2026-02-21
- 目标：让“聊天窗口里的信息”自动沉淀为可检索、可验证、可恢复的本地资产

## 1. 策略目标

1. 自动化：不依赖手工复制，定时归档。
2. 可验证：每次归档输出索引与哈希校验。
3. 可追溯：保留 Git 元信息与删除文件恢复副本。
4. 可恢复：任何一次快照都能独立回看。

## 2. 采集范围

1. 会话记录：
   - Codex 日志
   - Cursor agent transcripts
   - Cursor plans
   - Cursor terminal logs
2. 项目知识库：
   - `hxy` 目录下文本型资产（md/sql/txt/json/csv/yml/yaml/sh）
   - 外部参考单文档（`/root/hxy-java-code-review.md`、`/root/hxyreview.md`）
3. 删除态恢复：
   - 当前工作区中 `git ls-files --deleted -- hxy` 的路径，从 `HEAD` 恢复到归档目录

## 3. 执行脚本

1. 归档主脚本：
   - `/_local_preservation/scripts/refresh_weekly_preservation.sh`
2. 自动任务安装：
   - `/_local_preservation/scripts/install_auto_archive_cron.sh`
3. 自动任务卸载：
   - `/_local_preservation/scripts/uninstall_auto_archive_cron.sh`

## 4. 自动频率（默认）

1. 每 30 分钟一次快照。
2. 每天 00:15 固化一次快照。
3. 每周默认保留最近 20 次快照（可用环境变量调整）。

## 5. 产物规范

1. 每次快照必须包含：
   - `indexes/summary.txt`
   - `indexes/file_inventory.txt`
   - `indexes/sha256sum.txt`（系统支持时）
2. Git 元信息：
   - `indexes/git-head.txt`
   - `indexes/git-status-short.txt`
   - `indexes/git-log-30.txt`
   - `indexes/deleted_hxy_paths.txt`

## 6. 落地规则

1. 所有“阶段总结、方案共识、执行记录”必须写成 `.md` 放入 `hxy`。
2. 所有脚本/SQL 验收证据必须有路径，可被归档脚本抓到。
3. 避免把临时对话结论只留在聊天窗口，必须落文档后再推进。
