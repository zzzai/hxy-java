# 项目级命名规范（HXY 私有化）V1

## 1. 目标与原则
- 目标：从开源底座迁移为私有化工程后，命名统一、可扩展、可审计。
- 原则：品牌统一（`hxy`）、技术中性、避免历史包袱词（如 `crmeb`）。
- 兼容原则：对外接口与线上域名先稳定，内部命名分阶段替换。

## 2. 统一命名根
- 品牌标识：`HXY`（展示层）/ `hxy`（代码与资源层）。
- 英文全称建议：`HexiaoYue`（仅品牌展示，不进入技术主命名）。
- 技术主命名：统一使用 `hxy`。

## 3. 仓库与目录命名
- 主仓库名：`hxy-platform`
- 若拆仓，推荐：
  - `hxy-server`
  - `hxy-admin-web`
  - `hxy-miniapp`
  - `hxy-ops`
  - `hxy-knowledge-base`
- 单仓目录建议：
  - `apps/`（前端与终端应用）
  - `services/`（后端服务）
  - `infra/`（部署与IaC）
  - `docs/`（文档）
  - `script/`（运维与工具脚本）

## 4. Java 命名规范
- GroupId：`com.hxy`
- 基础包：`com.hxy.platform`
- 模块包示例：
  - `com.hxy.platform.pay`
  - `com.hxy.platform.trade`
  - `com.hxy.platform.member`
  - `com.hxy.platform.system`
- Maven ArtifactId 建议：
  - `hxy-module-pay`
  - `hxy-module-trade`
  - `hxy-module-member`
- 类命名：`PayOrderService`、`TradeRefundController`（避免 `Crmeb*` 前缀）

## 5. API 与网关命名
- 域名建议：
  - `api.hexiaoyue.com`
  - `admin.hexiaoyue.com`
- API 路径前缀建议（新接口）：
  - `/api/v1/admin/...`
  - `/api/v1/app/...`
- 兼容接口路径（过渡期）：
  - 放在 `/compat/...`，禁止新增业务能力，仅做迁移兜底。

## 6. 数据库命名
- 库名：
  - `hxy_dev`
  - `hxy_stg`
  - `hxy_prod`
- 表命名：`领域前缀 + 语义`
  - `pay_order`、`pay_refund`
  - `trade_order`、`trade_after_sale`
  - `member_user`
- 索引命名：
  - 唯一索引：`uk_<table>_<field>`
  - 普通索引：`idx_<table>_<field>`
- 迁移脚本：`VYYYYMMDDHHMM__<desc>.sql`

## 7. 配置与环境变量
- Spring Profile：`application-dev.yaml`、`application-stg.yaml`、`application-prod.yaml`
- 业务扩展配置前缀：`hxy.*`
- 环境变量：全大写下划线
  - `HXY_DB_URL`
  - `HXY_WX_PAY_MCH_ID`
  - `HXY_WX_PAY_API_V3_KEY`

## 8. CI/CD 与制品命名
- 工作流名：`hxy-<domain>-<purpose>`
  - 例：`hxy-pay-release-gate`
- 制品目录：`artifacts/<pipeline>/<run_id>/`
- 镜像名：`registry/hxy/<service>:<version>`

## 9. 分支与版本命名
- 分支：
  - `main`
  - `release/<yyyy.mm>`
  - `feature/<domain>-<topic>`
  - `hotfix/<issue-id>`
- 版本：
  - `v<major>.<minor>.<patch>`
  - 例：`v1.3.0`

## 10. 迁移期落地策略（当前建议）
- 第一步：立即停止新增任何 `crmeb` 命名文件/类/配置。
- 第二步：保留运行所需兼容层，但归档到 `compat/` 并标记下线时间。
- 第三步：逐批重命名包、类、配置项到 `hxy` 体系。
- 第四步：删除所有 `crmeb` 历史目录与无效工作流。

## 11. 当前项目推荐映射
- 现目录：`/root/crmeb-java/ruoyi-vue-pro-master`
- 目标命名：`/root/hxy-platform`（代码层目标名）
- 知识库目录：`/root/crmeb-java/hxy`（短期保留）
- 后续可迁为：`/root/hxy-platform/docs` 或独立 `hxy-knowledge-base` 仓库
