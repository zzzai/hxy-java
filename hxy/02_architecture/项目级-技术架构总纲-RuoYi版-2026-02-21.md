# 项目级技术架构总纲（RuoYi版）

- 日期：2026-02-21
- 基座目录：`/root/crmeb-java/ruoyi-vue-pro-master`
- 目标：统一项目级架构约束，支付为首个优先收敛子域

## 1. 架构原则

1. 单体模块化优先，领域边界清晰，避免跨域直连。
2. 先保证主链路稳定，再扩展功能广度。
3. 全链路可观测、可回滚、可审计。
4. 配置与密钥外置，最小权限访问。

## 2. 逻辑架构分层

1. 接入层：`admin-api/app-api`、小程序/H5/后台前端。
2. 应用层：Controller + Application Service，负责编排和鉴权。
3. 领域层：交易、支付、商品、会员、营销等业务规则。
4. 基础设施层：MySQL、Redis、MQ、对象存储、日志监控。
5. 外部依赖层：微信支付、微信小程序平台、短信/消息平台。

## 3. 模块边界（项目级）

1. 交易模块只通过 `PayOrderApi/PayRefundApi` 调支付模块。
2. 支付模块统一对接渠道 SDK，不允许业务模块直连渠道。
3. 门店业务模块无资金写权限（退款执行权限在总部模块）。
4. 非支付模块（商品、会员、营销）按接口契约与交易模块协作。

## 4. 部署拓扑（当前环境）

1. 公网入口：Nginx + SSL（`api/admin/主站` 子域分流）。
2. 内网服务：RuoYi 后端与数据库运行在内网服务器。
3. 网络链路：FRP 提供公网到内网穿透。
4. 开发链路：Windows 本地运行 HBuilderX/微信开发者工具，调用公网域名接口。

## 5. 数据架构约束

1. 业务主库：MySQL（交易、支付、商品、会员等统一治理）。
2. 缓存：Redis（会话、限流、热点数据）。
3. 支付关键表：
   - `pay_order` / `pay_order_extension`
   - `pay_refund`
   - `pay_notify_task` / `pay_notify_log`
4. 数据变更策略：发布脚本采用 `strict/replay` 双模式（CI/运维分离）。

## 6. 安全与合规

1. 密钥与证书仅通过环境变量或安全挂载注入，不入仓库明文。
2. 支付回调必须验签，失败即拒绝并告警。
3. 敏感操作（退款执行、配置变更）必须记录审计日志。
4. 总部与门店权限边界通过角色与接口双重限制。

## 7. 可观测与可靠性

1. 指标：支付成功率、回调延迟、退款收敛时长、任务积压量。
2. 日志：按订单号/支付单号/退款单号可链路追踪。
3. 补偿任务：支付查单、超时关单、退款同步、通知重试。
4. 上线拦截规则：监控过期、任务未跑、验收缺失、回滚缺失即阻断。

## 8. 发布与回滚

1. 发布前：自动化测试 + 脚本验收 + 上线拦截规则判定。
2. 发布中：灰度放量（按比例/按门店/按租户）。
3. 发布后：观察窗口内关键指标守护，异常即回滚。
4. 回滚原则：分钟级可执行，优先保障资金与订单一致性。

## 9. 演进路径

1. 当前：RuoYi 单体模块化持续收敛。
2. 中期：支付与交易服务边界进一步清晰化，逐步服务化拆分准备。
3. 长期：面向万店规模的分层架构（治理中台 + 业务域服务化）。

## 11. 30项计划映射（架构视角）

| 30项任务组 | 架构机制 | 技术验收口径 |
|---|---|---|
| `#1-#2` 调用边界与兼容冻结 | 交易仅经 `PayOrderApi/PayRefundApi`，compat 只做迁移 | 无业务模块直连渠道 SDK，新能力不落 compat |
| `#3-#4,#25` 配置模型与服务商参数 | `pay_app/pay_channel` 统一模型 + 服务商参数校验 | 渠道配置可校验，缺项 fail-fast |
| `#5-#8` 回调与状态一致性 | 回调先应答后异步、幂等键、状态机单向流转 | 重复/乱序回调不破坏终态 |
| `#9-#12` 超时与退款补偿 | 查单关单任务、退款回调+查询双通道、通知重试 | 挂单可清理、退款可收敛、通知可重放 |
| `#13-#16` 权限与策略层 | 门店退款执行下线、总部策略引擎、规则分层、审计日志 | 门店入口无法打款，总部决策可追溯 |
| `#17-#18,#21-#22` 对账与观测 | 交易账/渠道账对比、差异工单、指标告警 | 差异自动发现，告警可用且降噪 |
| `#19,#23,#24` 工程化保障 | 异常回放脚本、CI 回归、DB strict/replay | 发布前自动验收可阻断高风险变更 |
| `#26-#30` 真单与灰度发布 | 服务商实单联调、故障演练、灰度放量与回滚 | 真单链路全绿，异常可分钟级回退 |
