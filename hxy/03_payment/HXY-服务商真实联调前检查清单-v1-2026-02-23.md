# HXY 服务商真实联调前检查清单 V1（2026-02-23）

## 1. 目标

1. 在 `sp_mchid + sub_mchid` 到位后，1 天内完成真实链路联调。
2. 联调前一次性排除证书、参数、回调、权限、网络类低级阻断问题。
3. 形成可复用的门店接入前置门禁模板。

## 2. 基础参数核对（必须 100%）

1. 服务商商户号：`sp_mchid`。
2. 特约商户号：`sub_mchid`（必须不是服务商自身商户号）。
3. 服务商 AppID：`sp_appid`（与发起支付应用一致）。
4. 商户 APIv3 Key：32 位字符串（非证书序列号）。
5. 商户证书序列号：`serial_no`（与当前私钥匹配）。
6. 商户私钥：`apiclient_key.pem` 可正常加载。
7. 微信平台证书：已拉取并缓存，支持自动轮换。
8. 支付回调 URL：公网可访问，HTTPS 有效证书，非自签。
9. 退款回调 URL：与支付回调隔离或可区分路由。
10. `notify` 鉴签配置：平台证书序列号与解密 key 已校验通过。

## 3. 账户与权限检查（避免 403/NO_AUTH）

1. 服务商号已开通服务商相关接口权限。
2. 特约商户已与服务商建立绑定关系且状态生效。
3. 特约商户可用于 JSAPI 支付（场景权限已放开）。
4. 特约商户可用于退款接口。
5. 若启用分账：已开通分账权限，接收方配置可用。
6. 操作账号具备 API 调用权限（非仅后台查看权限）。

## 4. 网络与安全检查（避免超时/握手失败）

1. 应用服务器出口 IP 稳定，防火墙允许访问微信支付域名。
2. DNS 可稳定解析 `api.mch.weixin.qq.com`。
3. 系统时间同步（NTP），避免签名过期。
4. TLS 版本满足微信要求（推荐 TLS1.2+）。
5. 回调域名无 WAF 误拦截（白名单已配置）。

## 5. 业务参数映射检查（避免金额/商户错单）

1. 下单请求中服务商号与特约商户号映射正确。
2. 订单金额单位统一为分，内部金额无浮点误差。
3. `out_trade_no` 全局唯一、可幂等重试。
4. `attach` / 扩展字段包含业务追踪信息（门店、订单类型）。
5. 回调落库与业务订单关联键明确（`out_trade_no` + `transaction_id`）。

## 6. 关键联调用例（最小集）

1. 正常下单并支付成功（JSAPI）。
2. 支付回调重复推送幂等。
3. 主动查单与被动回调结果一致。
4. 正常退款成功（全额）。
5. 退款回调重复推送幂等。
6. 退款查询与内部状态一致。
7. 异常金额/异常商户号请求被正确拒绝并记录审计日志。

## 7. 联调通过标准（Go 条件）

1. 支付成功率 >= 99.95%（联调窗口样本内）。
2. 回调幂等无重复扣减/重复入账。
3. 退款终态可收敛，无悬挂单。
4. 支付账、交易账、渠道账三账可对平。
5. 所有失败请求均可追溯到错误码和业务单。

## 8. 联调失败快速排障顺序（固定）

1. 先看参数映射：`sp_mchid/sub_mchid/appid`。
2. 再看证书：`serial_no` 与私钥匹配关系。
3. 再看权限：服务商/特约商户接口开通状态。
4. 再看网络：DNS、TLS、回调可达性。
5. 最后看业务：幂等键、状态机、补偿任务。
