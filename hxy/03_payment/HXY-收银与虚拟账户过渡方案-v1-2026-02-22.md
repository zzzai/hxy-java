# HXY 收银与虚拟账户过渡方案 V1

- 日期：2026-02-22
- 目标：在 0-10 家阶段以最低复杂度跑通“支付 -> 履约 -> 台账 -> 结算”闭环。

## 1. 范围与边界

1. 适用：小规模起盘、以直营同主体为主的门店阶段。
2. 不做：预收款跨店通用、自动分账到个人、复杂会计引擎。
3. 原则：支付做收款，台账做记账，结算做出款，三者解耦。

## 2. 收银模式定版

1. 推荐模式：`先建订单 -> 生成支付码 -> 用户支付 -> 回调落单`。
2. 不推荐模式：`固定码先收款 -> 事后人工匹配订单`（错配率高、对账重）。
3. 门店识别：每笔订单在支付前必须固化 `store_id`、`technician_id`、`item_snapshot`。

## 3. 资金与台账流程

1. 用户支付成功后：
   - 创建交易订单
   - 触发履约/核销状态流转
   - 写入门店应得台账
   - 写入技师应得台账
2. 周期结算（例如每周五）：
   - 生成结算批次
   - 财务复核
   - 人工打款
   - 回填出款流水号与凭证

## 4. 最小数据结构

1. `store_settlement_ledger`
   - `id`, `order_id`, `store_id`, `gross_amount`, `store_amount`, `settlement_status`, `settlement_batch_id`
2. `tech_income_ledger`
   - `id`, `order_id`, `store_id`, `technician_id`, `rule_snapshot`, `income_amount`, `settlement_status`, `settlement_batch_id`
3. `settlement_batch`
   - `id`, `biz_date`, `target_type`, `target_id`, `amount`, `status`, `paid_at`, `payment_ref_no`, `operator_id`, `reviewer_id`

## 5. 运营规则

1. 结算批次未复核不得出款。
2. 结算批次一旦出款，禁止修改金额，只允许冲正单。
3. 退款单需先冲减对应台账，再参与后续结算。
4. 台账口径与订单口径、渠道口径每日对齐，差异自动告警。

## 6. 预收款过渡处理

1. 次卡/储值卡在此阶段默认“购卡门店锁定”。
2. Web 端录入最小字段：`user_id/mobile`、`card_type`、`total_quota`、`used_quota`、`locked_store_id`、`purchase_time`、`expire_time`。
3. 每次核销必须记录 `store_id`、`technician_id`、`order_id`，形成可追溯流水。

## 7. 退出条件（切主路径）

1. 门店达到 10 家以上或出现独立法人加盟门店。
2. 周结人工处理时间持续超阈值（例如 > 2 小时/周）。
3. 退款与冲正对账差异频发。
4. 一旦满足任一条件，切换到“服务商 + 特约商户 + 分账自动化”主路径。
