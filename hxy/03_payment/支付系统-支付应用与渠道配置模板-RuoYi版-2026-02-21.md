# 支付应用与渠道配置模板（RuoYi-Vue-Pro）

- 日期：2026-02-21
- 适用基座：`/root/crmeb-java/ruoyi-vue-pro-master`
- 目标：统一 `pay_app` + `pay_channel` 配置口径，覆盖 `wx_lite / wx_pub / wx_wap / wallet`

## 1. pay_app 模板

建议按“一个业务域一个支付应用”管理，先落 1 个总部支付应用。

| 字段 | 示例值 | 说明 |
|---|---|---|
| `id` | `1001` | 主键（示例） |
| `name` | `总部支付应用` | 应用名称 |
| `status` | `1` | 启用状态 |
| `remark` | `门店订单统一收款` | 备注 |

## 2. pay_channel 模板

建议每个 `app_id` 固定 4 条渠道，统一编号与用途。

| code | 用途 | status | app_id | config 关键字段 |
|---|---|---|---|---|
| `wx_lite` | 小程序 JSAPI | `1` | `1001` | `partnerMode,mchId,subMchId,subAppId,apiV3Key,certSerialNo,privateKeyContent` |
| `wx_pub` | 公众号 JSAPI | `1` | `1001` | `partnerMode,mchId,subMchId,subAppId,apiV3Key,certSerialNo,privateKeyContent` |
| `wx_wap` | H5 支付 | `1` | `1001` | `partnerMode,mchId,subMchId,apiV3Key,certSerialNo,privateKeyContent` |
| `wallet` | 余额支付 | `1` | `1001` | `[]`（默认空配置） |

## 3. WxPayClientConfig 模板（服务商模式）

> 说明：以下为 `pay_channel.config` 的 JSON 模板。  
> `mchId=sp_mchid`，`subMchId=sub_mchid`，必须成对生效。

```json
{
  "apiVersion": "v3",
  "partnerMode": true,
  "appId": "${SP_APPID}",
  "subAppId": "${SUB_APPID}",
  "mchId": "${SP_MCHID}",
  "subMchId": "${SUB_MCHID}",
  "apiV3Key": "${API_V3_KEY_32}",
  "certSerialNo": "${CERT_SERIAL_NO}",
  "privateKeyContent": "-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----",
  "publicKeyId": "${PAY_PUBLIC_KEY_ID}",
  "publicKeyContent": "-----BEGIN PUBLIC KEY-----\n...\n-----END PUBLIC KEY-----"
}
```

字段说明：

| 字段 | 必填 | 说明 |
|---|---|---|
| `apiVersion` | 是 | 固定 `v3` |
| `partnerMode` | 是 | 服务商模式固定 `true` |
| `appId` | 是 | 服务商关联 appId（统一支付应用） |
| `subAppId` | 条件必填 | JSAPI/小程序场景必填 |
| `mchId` | 是 | 服务商号 `sp_mchid` |
| `subMchId` | 是 | 特约商户号 `sub_mchid` |
| `apiV3Key` | 是 | 32 位 APIv3 密钥 |
| `certSerialNo` | 是 | 商户 API 证书序列号 |
| `privateKeyContent` | 是 | 商户私钥 PEM 内容 |
| `publicKeyId` | 建议 | 微信支付公钥 ID |
| `publicKeyContent` | 可选 | 微信支付公钥内容（用于部分验签/加密场景） |

## 5. 上线前核对清单

1. `pay_app` 存在且启用。
2. `wx_lite/wx_pub/wx_wap/wallet` 四条渠道存在且启用。
3. 微信三条渠道 `partnerMode=true` 且 `mchId != subMchId`。
4. `apiV3Key` 长度为 32。
5. 证书序列号、私钥、公钥 ID 与商户平台一致。

## 6. 推荐执行顺序

1. 先跑 `script/dev/check_wx_partner_readiness.sh` 过配置校验。
2. 再执行渠道写入脚本（如 `script/dev/update_wx_partner_channel.sh`）。
3. 最后跑回归：`script/dev/check_trade_pay_boundary.sh` 与 `script/dev/check_crmeb_compat_freeze.sh`。

## 7. 门店退款执行开关（P0 #13）

为满足“门店仅售后、总部执行退款”的资金治理要求，建议在运行环境显式配置：

```yaml
yudao:
  trade:
    compat:
      crmeb:
        store-refund-execute-enabled: false
```

说明：

1. `false`：门店兼容退款执行接口返回失败，仅保留售后工单流程。
2. `true`：仅用于应急回退，不建议作为常态。
