# HXY 支付系统规划 V2（2026-02-22）

## 1. 支付系统目标

1. 建立总部统一支付中台，门店无资金执行权限。
2. 支持微信小程序支付与退款闭环。
3. 支持异常补偿、对账差异工单化和上线拦截规则。
4. 在 0-100 店阶段坚持“服务商 + 特约商户 + 预收款单店隔离”的可执行策略。

## 2. 当前状态

1. 阶段 A（内功阶段）已完成主干能力：
   - 幂等、状态机、回调策略、退款分流、对账脚本、异常回放、上线拦截。
2. 阶段 B 外部依赖：
   - 服务商模式真实联调依赖 `sub_mchid` 正式到位。

## 3. 模式定版（2026-02-22）

1. 主路径：服务商 + 特约商户 + 分账。
2. 过渡路径：总部统收 + 虚拟账户台账 + 周期结算（仅小规模直营同主体）。
3. 预收款：单店隔离，不跨店核销。
4. 总部不建设脱离渠道监管的资金池。
5. 详见：
   - `HXY-支付模式定版-服务商特约商户-v1-2026-02-22.md`
   - `HXY-收银与虚拟账户过渡方案-v1-2026-02-22.md`

## 4. 联调策略

1. 真实联调前：全部采用模拟与脚本回放做压力与异常覆盖。
2. 真实联调时：按 `0.01` 真单支付 -> 真单退款 -> 对账核对顺序推进。
3. 验收要素：渠道流水、支付单、交易单三方一致。

## 5. 风控与退款策略

1. 自动秒退：低金额、低风险、状态合法。
2. 人工复核：高金额、高频、黑名单、证据不足。
3. 所有退款执行都必须可审计、可追责。
4. 退款必须先过“分账回退”校验，再进入渠道退款。

## 6. 上线拦截规则

1. 验收证据缺失：阻断。
2. 回滚方案缺失：阻断。
3. 监控/补偿任务异常：阻断。
4. 对账数据不可得：阻断。
5. 分账超时率超阈值：阻断。
6. 分账回退失败积压：阻断。
