# 支付系统（服务商 + 子商户）接入执行手册

## 1. 当前代码状态（已落地）
- 已支持微信支付按 `渠道 + 门店` 解析配置（公众号/小程序/H5/App）。
- 已支持服务商模式字段透传：`sub_mch_id`、`sub_appid`、`sub_openid`。
- 已覆盖链路：统一下单、支付查询、退款、回调签名 key 选择（优先按 `mch_id` 匹配）。
- 已兼容纯服务商模式：不再强依赖旧直连商户配置。

## 2. 你需要提供的信息（以及时间点）

### T0（现在就要）
- 服务商信息：`sp_mchid`、`sp_appid`（小程序/公众号/App按实际提供）。
- 子商户信息：
  - 默认子商户号（兜底）：`sub_mchid`
  - 如按门店分账：每店 `storeId -> sub_mchid`
  - 如子商户 AppID 与服务商不同：`sub_appid`（默认或按门店）
- 回调域名：`api_url` 必须是可公网访问 `https` 域名。

### T1（联调前）
- APIv2 Key（按渠道）：
  - `pay_routine_sp_key`
  - `pay_weixin_sp_key`
  - `pay_weixin_app_sp_key`
- 退款证书路径（如要联调退款）：
  - `pay_routine_sp_certificate_path` / `pay_weixin_sp_certificate_path` / `pay_weixin_app_sp_certificate_path`

### T2（上线前）
- 生产环境全部真实 key/cert（不要在聊天里发明文）。
- 每门店子商户映射最终版（含新开店增量流程）。
- 微信商户平台侧：回调地址、API安全设置、域名白名单最终核验。

## 3. 账号绑定方式（系统内）
- 通过 `eb_system_config` 键值绑定。
- 可直接执行模板 SQL：
  - `crmeb_java/crmeb/sql/payment_service_provider_submerchant_template.sql`
- 推荐最小必填（小程序场景）：
  - `pay_routine_sp_appid`
  - `pay_routine_sp_mchid`
  - `pay_routine_sp_key`
  - `pay_routine_sub_mchid`（或 `pay_routine_sub_mchid_{storeId}`）
  - `pay_routine_sub_appid`（如与服务商 appid 不同）

## 4. 联调顺序（必须按顺序）
1. 统一下单（JSAPI/H5）
2. 支付回调
3. 订单查询
4. 退款 + 退款回调
5. 分门店子商户回归（至少 2 家门店）

## 4.1 明早一键脚本
- 脚本路径：`crmeb_java/crmeb/shell/payment_morning_quickstart.sh`
- 用法示例：
  - `SP_MCHID=xxx SP_KEY=xxx SUB_MCHID=xxx APPID=wx97fb30aed3983c2c ENABLE_PAY=1 ./shell/payment_morning_quickstart.sh`

## 4.1.1 服务商参数填充（新增）
- 脚本路径：`crmeb_java/crmeb/shell/payment_service_provider_fill.sh`
- 支持 `dry-run/apply`，并产出 SQL 与执行摘要。
- 推荐：
  - 先 dry-run 校验参数
  - 再 `--apply` 落库

## 4.2 门店映射批量导入（新增）
- 脚本路径：`crmeb_java/crmeb/shell/payment_store_mapping_import.sh`
- CSV 格式（两列）：
  - `storeId,sub_mchid`
  - `1,1900001001`
  - `2,1900001002`
- 先 dry-run（不写库）：
  - `./shell/payment_store_mapping_import.sh --csv /path/store_mapping.csv`
- 再 apply（写库）：
  - `./shell/payment_store_mapping_import.sh --csv /path/store_mapping.csv --apply`
- 严格模式（同一 `sub_mchid` 禁止多门店复用）：
  - `./shell/payment_store_mapping_import.sh --csv /path/store_mapping.csv --strict-submchid-unique --strict-store-exists 1 --reject-placeholder-submchid 1 --apply`

## 4.3 映射冲突处理策略
- 冲突类型 A：同一个 `storeId` 在 CSV 内对应多个 `sub_mchid`。
- 冲突类型 B：CSV 与库内现有 `pay_routine_sub_mchid_{storeId}` 不一致。
- 处理原则：
  - 出现冲突一律先停，不直接覆盖。
  - 先确认“真实商户台账”后再重跑脚本。
- 先做 dry-run，确认 0 冲突后再 `--apply`。

## 4.4 映射回滚（新增）
- 脚本路径：`crmeb_java/crmeb/shell/payment_store_mapping_rollback.sh`
- 用法：
  - 预览最近一次回滚脚本：`./shell/payment_store_mapping_rollback.sh --latest`
  - 执行回滚：`./shell/payment_store_mapping_rollback.sh --latest --apply`

## 4.5 上线前预检（新增）
- 脚本路径：`crmeb_java/crmeb/shell/payment_preflight_check.sh`
- 建议在参数填充后执行：
  - `MYSQL_DEFAULTS_FILE="$HOME/.my.cnf" ./shell/payment_preflight_check.sh`
- 预检覆盖：支付配置、回调域名、证书路径、cron 任务、安全凭据。
- 新增阻断规则：
  - 若检测到 `pay_routine_sub_mchid` 或 `pay_routine_sub_mchid_{storeId}` 使用占位号 `99xxxxxxxx`，预检直接 `FAIL`。

## 4.6 真实子商户号一键切换（新增）
- 脚本路径：`crmeb_java/crmeb/shell/payment_store_mapping_cutover.sh`
- 流程：
  1. 映射导入 dry-run
  2. 映射导入 apply
  3. 映射审计（必须 GREEN）
  4. 值守状态刷新（必须 GREEN）
  5. 切换上线拦截规则复核（必须 GREEN）
- 内置硬校验（默认开启）：
  - 阻断 `99xxxxxxxx` 占位号
  - 阻断不存在/已删除 `storeId`
- 用法：
  - `./shell/payment_store_mapping_cutover.sh --csv runtime/payment_store_mapping/store_mapping_real_template.csv --date 2026-02-17 --order-no <真实订单号> --no-alert`
- 产物：
  - `runtime/payment_store_mapping_cutover/run-*/summary.txt`
  - `runtime/payment_store_mapping_cutover/run-*/report.md`

## 5. 安全要求
- `APIv2 Key` 和证书路径只在服务器配置，不进代码库。
- 生产 key 禁止通过 IM 明文发送。
- 建议按环境隔离：`dev/staging/prod` 三套参数。
- 支付开关值建议统一用 `1/0`，系统现已兼容 `'1'/"1"/true/on/yes` 但不建议长期混用。

## 6. 结构化日志字段（新增）
- 统一下单日志位于：
  - `crmeb-service/src/main/java/com/zbkj/service/service/impl/OrderPayServiceImpl.java`
  - `crmeb-service/src/main/java/com/zbkj/service/service/impl/WeChatPayServiceImpl.java`
  - `crmeb-service/src/main/java/com/zbkj/service/service/impl/ServiceBookingServiceImpl.java`
- 核心字段说明：
  - `outTradeNo`：微信侧商户单号（排查回调、查单、退款主键）。
  - `channel`：支付渠道（0公众号/1小程序/2H5/4-5App）。
  - `storeId`：门店ID（用于追踪门店映射命中）。
  - `serviceProviderMode`：是否服务商模式。
  - `mchId` / `subMchId`：本次请求使用的服务商/子商户号。
- 安全边界：
  - 日志不输出 `API key`、证书、`openid/sub_openid`。

## 7. 明早执行流水线（建议顺序）
1. 执行最小支付配置：
   - `./shell/payment_morning_quickstart.sh`
2. 执行门店映射 dry-run：
   - `./shell/payment_store_mapping_import.sh --csv /path/store_mapping.csv`
3. 若 dry-run 无冲突，执行 apply：
   - `./shell/payment_store_mapping_import.sh --csv /path/store_mapping.csv --apply`
4. 重启 `crmeb-admin` / `crmeb-front`。
5. 执行冒烟：下单 -> 回调 -> 查单 -> 退款。
