# 支付系统-门店映射导入说明

## 1. 目标
- 批量写入小程序渠道门店子商户映射：`pay_routine_sub_mchid_{storeId}`。
- 在写入前自动做冲突检测，避免误覆盖。

## 2. 脚本位置
- `crmeb_java/crmeb/shell/payment_store_mapping_template_export.sh`
- `crmeb_java/crmeb/shell/payment_store_mapping_import.sh`
- `crmeb_java/crmeb/shell/payment_store_mapping_rollback.sh`
- `crmeb_java/crmeb/shell/payment_store_mapping_pipeline_smoke.sh`
- `crmeb_java/crmeb/shell/payment_store_mapping_csv_generate.sh`
- `crmeb_java/crmeb/shell/payment_store_mapping_placeholder_cleanup.sh`
- `crmeb_java/crmeb/shell/payment_store_mapping_cross_channel_audit.sh`

## 3. CSV 格式
```csv
storeId,sub_mchid
1,1900001001
2,1900001002
3,1900001003
```

## 4. 执行步骤
1. 先导出门店模板（推荐）
```bash
cd /root/crmeb-java/crmeb_java/crmeb
MYSQL_DEFAULTS_FILE="$HOME/.my.cnf" ./shell/payment_store_mapping_template_export.sh
```
- 导出后会得到：
  - `store_mapping_template.csv`（可直接导入）
  - `store_mapping_reference.tsv`（门店名称/电话/当前映射参考）

2. Dry-run（推荐先执行）
```bash
cd /root/crmeb-java/crmeb_java/crmeb
./shell/payment_store_mapping_import.sh --csv /path/store_mapping.csv
```

3. 严格校验（推荐）
```bash
cd /root/crmeb-java/crmeb_java/crmeb
./shell/payment_store_mapping_import.sh \
  --csv /path/store_mapping.csv \
  --strict-submchid-format \
  --strict-submchid-unique \
  --strict-store-exists 1 \
  --reject-placeholder-submchid 1
```

4. Apply（两步确认）
先执行预确认（不会写库）：
```bash
cd /root/crmeb-java/crmeb_java/crmeb
./shell/payment_store_mapping_import.sh --csv /path/store_mapping.csv --apply
```
再执行确认落库：
```bash
cd /root/crmeb-java/crmeb_java/crmeb
./shell/payment_store_mapping_import.sh --csv /path/store_mapping.csv --apply --confirm
```

5. 严格模式（同一个子商户号不能复用）
```bash
./shell/payment_store_mapping_import.sh --csv /path/store_mapping.csv --strict-submchid-unique --apply
```

6. 冲突策略（默认阻断）
```bash
# 默认：block（冲突即失败）
./shell/payment_store_mapping_import.sh --csv /path/store_mapping.csv --conflict-strategy block

# 允许覆盖：overwrite（用于批量纠偏）
./shell/payment_store_mapping_import.sh --csv /path/store_mapping.csv --conflict-strategy overwrite
```

7. 推荐使用凭据文件（避免明文密码）
```bash
MYSQL_DEFAULTS_FILE="$HOME/.my.cnf" ./shell/payment_store_mapping_import.sh --csv /path/store_mapping.csv
```

8. 导出空白模板（全部留空待运营填写）
```bash
cd /root/crmeb-java/crmeb_java/crmeb
MYSQL_DEFAULTS_FILE="$HOME/.my.cnf" ./shell/payment_store_mapping_template_export.sh --blank-sub-mchid 1 --out /path/store_mapping.csv
```

9. 仅导出“未配置子商户号”的门店（便于运营补齐）
```bash
cd /root/crmeb-java/crmeb_java/crmeb
MYSQL_DEFAULTS_FILE="$HOME/.my.cnf" ./shell/payment_store_mapping_template_export.sh --only-unmapped 1
```

10. 映射链路 smoke（每日回归）
```bash
cd /root/crmeb-java/crmeb_java/crmeb
MYSQL_DEFAULTS_FILE="$HOME/.my.cnf" ./shell/payment_store_mapping_pipeline_smoke.sh
```

11. 生成“真实映射CSV草稿”（按门店台账自动拉底）
```bash
cd /root/crmeb-java/crmeb_java/crmeb
MYSQL_DEFAULTS_FILE="$HOME/.my.cnf" ./shell/payment_store_mapping_csv_generate.sh --only-unmapped 1
```

12. 占位号清理（先 dry-run，再 apply）
```bash
cd /root/crmeb-java/crmeb_java/crmeb
MYSQL_DEFAULTS_FILE="$HOME/.my.cnf" ./shell/payment_store_mapping_placeholder_cleanup.sh
MYSQL_DEFAULTS_FILE="$HOME/.my.cnf" ./shell/payment_store_mapping_placeholder_cleanup.sh --apply
```

13. 跨渠道唯一性审计（上线前必跑）
```bash
cd /root/crmeb-java/crmeb_java/crmeb
MYSQL_DEFAULTS_FILE="$HOME/.my.cnf" ./shell/payment_store_mapping_cross_channel_audit.sh
```

## 5. 运行产物（新增）
- 每次执行会生成目录：
  - `crmeb_java/crmeb/runtime/payment_store_mapping/import-时间戳/`
- 关键文件：
  - `summary.txt`：本次导入汇总
  - `plan.tsv`：逐店执行计划（insert/unchanged/conflict）
  - `conflicts.tsv`：冲突清单
  - `normalized.tsv`：标准化后的输入映射
  - `warnings.txt` / `errors.txt`：校验告警与错误
  - `rollback.sql`：仅 `--apply` 且无冲突时生成
## 6. 冲突规则
- CSV 内冲突：同一 `storeId` 对应多个 `sub_mchid` -> 直接失败。
- DB 冲突：
  - `conflict-strategy=block`：直接失败；
  - `conflict-strategy=overwrite`：标记为覆盖更新（需 `--apply --confirm` 才会生效）。
- 严格模式冲突：同一 `sub_mchid` 同时映射多个 `storeId` -> 直接失败。
- 格式校验：
  - `--strict-submchid-format` 开启时，`sub_mchid` 必须是 10 位数字；
  - 未开启时，非标准格式给 warning 但不阻断。
- 占位号校验：
  - 默认阻断 `99xxxxxxxx`（`--reject-placeholder-submchid 1`）；
  - 仅在演练场景可显式放开：`--allow-placeholder-submchid`。
- 门店存在校验：
  - 默认阻断不存在/已删除门店（`--strict-store-exists 1`）；
  - 仅在特殊迁移场景可放开：`--strict-store-exists 0`（会记录 warning）。

## 7. 回滚方式（新增）
1. 预览最近一次回滚 SQL（dry-run）
```bash
cd /root/crmeb-java/crmeb_java/crmeb
./shell/payment_store_mapping_rollback.sh --latest
```
2. 执行回滚
```bash
cd /root/crmeb-java/crmeb_java/crmeb
MYSQL_DEFAULTS_FILE="$HOME/.my.cnf" ./shell/payment_store_mapping_rollback.sh --latest --apply
```

## 8. 常见问题
- Q: 可以先用虚拟 `sub_mchid` 吗？  
  A: 可以用于“联调流程打通”，但正式联调和上线前必须替换成真实子商户号并重跑导入。
- Q: 会覆盖没在 CSV 里的门店吗？  
  A: 不会。脚本仅处理 CSV 中出现的 `storeId`。
