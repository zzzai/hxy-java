## Constraints used in this implementation

- `wx.requestPayment` required params include `timeStamp`, `nonceStr`, `package(prepay_id=...)`, `paySign`; `signType` must match backend signing mode (v3 uses `RSA`).
- Frontend callback is not final truth; backend must use payment callback + query-order to converge final state.
- Callback missing/delay must be compensated by active query-order; do not rely on frontend return only.
- Close-order should be executed only after confirming order remains unpaid (`NOTPAY`), then treat as failure terminal.
- `prepay_id` validity is limited (docs indicate 2h); timeout requires re-order with original business parameters.
