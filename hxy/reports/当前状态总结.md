## ❌ 当前问题

### 问题：应用启动时仍然读取旧的数据库配置

**现象**：
```
java.sql.SQLException: Access denied for user 'single_open'@'localhost'
```

**已尝试的解决方案**：
1. ✅ 修改 application-dev.yml（已正确配置为crmeb_java/root）
2. ✅ 修改 application.yml（已正确配置）
3. ✅ mvn clean 清理缓存
4. ✅ 指定 -Dspring-boot.run.profiles=dev
5. ❌ 仍然读取 single_open 配置

**可能原因**：
- Maven可能从其他地方读取配置（环境变量、系统属性）
- 可能有其他配置文件覆盖了application.yml
- 可能需要使用jar包方式启动而非mvn spring-boot:run

---

## 🎯 推荐解决方案

### 方案1：使用jar包启动（推荐）

```bash
# 1. 编译打包
cd /root/crmeb-java/crmeb_java/crmeb
mvn clean package -DskipTests

# 2. 启动应用
cd crmeb-admin/target
java -jar crmeb-admin-0.0.1-SNAPSHOT.jar --spring.profiles.active=dev

# 3. 访问
http://localhost:8080
```

### 方案2：创建single_open数据库用户（临时方案）

```bash
# 如果急需启动，可以创建single_open用户
mysql -u root -p
CREATE USER 'single_open'@'localhost' IDENTIFIED BY '111111';
GRANT ALL PRIVILEGES ON crmeb_java.* TO 'single_open'@'localhost';
FLUSH PRIVILEGES;
```

### 方案3：检查环境变量

```bash
# 检查是否有环境变量覆盖
env | grep -i "spring\|mysql\|datasource"

# 检查Maven settings.xml
cat ~/.m2/settings.xml | grep -i "single_open"
```

```json

```
服务商品库存 = 每天的可预约时间槽数量

- 技师B：14:00-22:00，8个时间槽 = 8个库存
