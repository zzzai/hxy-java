# 支付系统-明早联调清单

## 1) 一键写入配置

在项目根目录执行：

```bash
SP_MCHID='你的服务商mchid' \
SP_KEY='你的服务商APIv2Key' \
SUB_MCHID='你的默认子商户号' \
APPID='wx97fb30aed3983c2c' \
ENABLE_PAY=1 \
./shell/payment_morning_quickstart.sh
```

## 2) 重启服务

- 重启 `crmeb-admin`
- 重启 `crmeb-front`

## 3) 导入门店映射（如有多门店）

```bash
# 先预检查（不写库）
./shell/payment_store_mapping_import.sh --csv /path/store_mapping.csv

# 0 冲突后再写库
./shell/payment_store_mapping_import.sh --csv /path/store_mapping.csv --apply
```

## 4) 冒烟顺序（严格按顺序）

1. 小程序下单（统一下单）
2. 微信支付回调入库
3. 订单查询（确认交易状态）
4. 发起退款
5. 退款回调验签与状态更新

## 5) 失败优先排查

- 服务商与子商户授权关系是否生效。
- 回调域名是否公网可达（HTTPS）。
- `pay_routine_sp_mchid / pay_routine_sp_key / pay_routine_sub_mchid` 是否误填。
- `pay_routine_sub_mchid_{storeId}` 是否与当前门店一致。
- 微信商户平台 APIv2 Key 是否与系统一致。
