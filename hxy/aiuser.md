# 一、用户数据战略价值

## 1.1 为什么数据是核心资产

【对用户运营的价值】\
\
用户全生命周期管理：\
├─ 从首次到店 → 成为会员 → 高频复购 → 推荐转介\
└─ 每个环节都需要数据支撑决策\
\
个性化服务提升：\
├─ 千人千面的服务推荐\
├─ 精准的健康方案定制\
└─ 技师与客户的最优匹配\
\
流失预警与挽回：\
├─ 提前30天预测用户流失\
├─ 精准挽回策略（优惠券/关怀电话）\
└─ 流失原因分析与改进\
\
【对企业资产的价值】\
\
数据资产估值：\
├─ 每个高质量用户画像价值 ¥500-1000\
├─ 100家店×5000活跃用户 = 2.5-5亿数据资产\
└─ 投融资时的核心竞争力证明\
\
运营效率提升：\
├─ 门店选址：基于用户地理分布\
├─ 库存管理：基于用户消费预测\
├─ 排班优化：基于客流量预测\
└─ 营销精准：ROI从1:3提升到1:8\
\
商业决策支撑：\
├─ 新服务项目开发（基于用户需求）\
├─ 定价策略优化（基于价格敏感度）\
└─ 竞争对手分析（用户流失去向）

## 1.2 健康养生行业的数据特殊性

-   【高价值】健康数据 \> 消费数据，一张舌诊照片价值远超订单记录

-   【高敏感】涉及个人健康隐私，需严格合规（《个人信息保护法》）

-   【高复杂】多模态数据（文本+图片+生理指标+行为轨迹）

-   【高连续】用户多次到店，数据可持续积累形成健康档案

-   【高专业】需要中医/健康专家参与数据标注和解读

## 2.2 线下门店数据采集方案

### 2.2.1 服务人员采集流程

【场景1：首次到店 - 建立健康档案】\
\
时机：前台登记时（5分钟）\
\
采集人：前台/店长\
工具：Pad端「首次到店问卷」\
\
必填项（快速采集）：\
├─ 姓名、手机号、性别、年龄\
├─ 职业类型：\[白领/蓝领/学生/自由职业/退休\]\
├─ 主要诉求：\[颈椎不适/腰椎问题/失眠/疲劳/养生保健\]\
└─ 是否吸烟：\[是/否\]，频率：\[偶尔/经常/重度\]\
\
可选项（诱导采集，送小礼品）：\
├─ 身高、体重（前台目测填写，或引导体测）\
├─ 过敏史：\[无/海鲜过敏/药物过敏/其他\]\
├─ 正在服药：\[是/否\]，药物名称\
└─ 家族病史：\[心脏病/糖尿病/高血压/无\]\
\
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━\
\
【场景2：服务前 - 技师问诊】\
\
时机：服务开始前（5-10分钟）\
\
采集人：技师\
工具：技师端「服务前问诊」\
\
问诊流程：\
1. 今日主要不适部位？\
\[颈部/肩部/腰部/腿部/全身疲劳\]\
→ 在人体穴位图上点击标记\
\
2. 不适持续时间？\
\[今天突发/近期（1周内）/长期（1个月以上）\]\
\
3. 疼痛等级？\
滑动条选择：1-10级（配图示）\
\
4. 日常工作姿势？\
\[久坐办公/久站/重体力/经常走动\]\
\
5. 近期压力状况？\
\[压力很大/一般/较轻松\]\
\
6. 睡眠质量？\
\[经常失眠/偶尔失眠/睡眠良好\]\
\
7. 力度偏好？\
\[轻柔/中等/较重/很重\]\
→ 技师根据首次服务反馈持续优化\
\
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━\
\
【场景3：服务中 - 持续观察记录】\
\
时机：服务过程中（技师观察）\
\
采集人：技师\
工具：技师端「服务记录」（语音输入）\
\
记录内容：\
├─ 体征观察：\
│ • 肩颈僵硬程度：\[轻度/中度/重度\]\
│ • 肌肉弹性：\[正常/偏硬/很硬\]\
│ • 皮肤状态：\[正常/偏干/偏油\]\
│ • 面色：\[红润/暗沉/苍白/发黄\]\
│\
├─ 服务反馈：\
│ • 力度反馈：\"稍微轻一点\" → 系统记录偏好\
│ • 敏感部位：技师标记怕痒/疼痛部位\
│ • 异常反应：\[无/头晕/恶心\]\
│\
└─ 效果评估（服务结束）：\
• 不适缓解程度：\[完全缓解/明显缓解/略有缓解/无效果\]\
• 客户主观评价：\[非常满意/满意/一般/不满意\]\
\
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━\
\
【场景4：服务后 - 健康建议】\
\
时机：服务结束后（2分钟）\
\
采集人：技师\
工具：技师端「健康建议」（模板化）\
\
系统根据本次服务记录，自动生成：\
├─ 问题诊断：\"您的颈椎劳损较明显，建议\...\"\
├─ 后续方案：\[继续观察/建议1周后复诊/建议配合艾灸\]\
├─ 生活建议：\[注意保暖/避免久坐/适度运动\]\
└─ 推荐服务：\[肩颈专项套餐/艾灸理疗\]\
\
技师可补充个性化建议（文字/语音）

# 五、隐私合规与数据安全

## 5.1 法律合规要求

【核心法律依据】\
\
《个人信息保护法》（2021年11月1日实施）\
├─ 健康数据属于\"敏感个人信息\"\
├─ 必须取得用户\"单独同意\"\
├─ 必须告知收集目的、方式、范围\
└─ 用户有权拒绝、删除、导出数据\
\
《数据安全法》（2021年9月1日实施）\
├─ 数据分级分类管理\
├─ 重要数据和核心数据需报备\
└─ 数据出境需安全评估\
\
《网络安全法》（2017年6月1日实施）\
├─ 采取技术措施保障数据安全\
├─ 定期备份，制定应急预案\
└─ 数据泄露需及时告知用户\
\
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━\
\
【健康数据敏感性分级】\
\
L4 极敏感（严格保护）：\
├─ 疾病诊断信息\
├─ 医疗记录\
├─ 基因数据\
└─ 精神健康状态\
\
L3 高敏感（加密存储）：\
├─ 舌诊照片\
├─ 面部照片\
├─ 身体局部照片\
├─ 体测数据（血压/心率）\
└─ 中医体质报告\
\
L2 中敏感（脱敏处理）：\
├─ 健康问卷（问题部位/疼痛等级）\
├─ 生活习惯（吸烟/饮酒/作息）\
├─ 服务记录（技师观察记录）\
└─ 用户反馈（服务评价）\
\
L1 低敏感（常规保护）：\
├─ 消费记录\
├─ 行为数据\
├─ 偏好标签\
└─ 到店记录\
\
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━\
\
【合规实施清单】\
\
✅ 必做项：\
\
1. 用户协议与授权\
├─ 注册时：《用户服务协议》+《隐私政策》\
├─ 首次采集健康数据：《健康数据授权书》（单独弹窗）\
├─ AI舌诊前：明确告知\"照片用于AI分析\"\
└─ 照片拍摄前：口头告知+屏幕提示\
\
2. 数据收集透明化\
├─ 在《隐私政策》中列明：\
│ • 收集哪些数据\
│ • 用于什么目的\
│ • 如何存储和使用\
│ • 第三方共享情况\
│ • 用户权利（查看/删除/导出）\
└─ 定期更新（每半年review一次）\
\
3. 最小化原则\
├─ 只采集业务必需的数据\
├─ 非必要不采集（如身份证号）\
├─ 匿名化处理：内部分析时去除姓名/手机号\
└─ 数据过期删除：用户注销后30天内删除\
\
4. 安全技术措施\
├─ 传输加密：HTTPS + TLS1.3\
├─ 存储加密：\
│ • L3/L4数据：AES256加密\
│ • 照片：加密存储+CDN鉴权访问\
├─ 访问控制：\
│ • 技师只能查看服务中客户数据\
│ • 店长只能查看本店客户数据\
│ • 数据分析师只能查看脱敏数据\
└─ 审计日志：记录所有数据访问行为\
\
5. 用户权利保障\
├─ 小程序「个人中心」提供：\
│ • \[查看我的数据\]\
│ • \[导出我的数据\]（JSON/PDF格式）\
│ • \[删除我的数据\]（7天后永久删除）\
│ • \[撤销授权\]（停止数据收集）\
├─ 客服渠道：400电话+在线客服\
└─ 响应时效：15个工作日内处理\
\
6. 数据泄露应急\
├─ 建立应急预案\
├─ 泄露后72小时内：\
│ • 上报监管部门\
│ • 告知受影响用户\
│ • 公开披露事件\
└─ 定期演练（每年2次）\
\
⚠️ 禁止项：\
\
❌ 未经授权采集：不能偷偷采集数据\
❌ 强制授权：不能\"不授权就不给用\"\
❌ 过度采集：不能要求提供无关数据\
❌ 非法买卖：严禁出售用户数据\
❌ 擅自共享：未经同意不得给第三方
