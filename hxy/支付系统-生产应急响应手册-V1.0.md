# 支付系统-生产应急响应手册 V1.0

- 版本：V1.0
- 日期：2026-02-18
- 适用范围：总部支付中台（下单/回调/查单/退款/对账/上线拦截规则）

## 1. 分级标准（P0-P2）

| 级别 | 触发条件 | 目标 |
|---|---|---|
| P0 | `cutover_gate=NO_GO`、支付大面积失败、回调不可用、退款大面积卡死 | 立即止损，30分钟内恢复主交易能力 |
| P1 | `GO_WITH_WARN`、单链路异常（如退款补偿失败率升高） | 2小时内收敛为GREEN |
| P2 | 观察类告警、非关键链路抖动 | 当日处理并复盘 |

## 2. 5分钟内动作（必须执行）

1. 冻结上线与高风险变更（暂停手工改配置、暂停非必要发布）。
2. 采集证据包（统一口径）：

```bash
cd /root/crmeb-java/crmeb_java/crmeb
./shell/payment_incident_bundle.sh --date 2026-02-18 --order-no <订单号或交易单号>
```

3. 复核值守与上线拦截规则：

```bash
DB_PORT=33306 MYSQL_DEFAULTS_FILE="$HOME/.my.cnf" ./shell/payment_ops_status.sh --date 2026-02-18 --refresh --no-alert
DB_PORT=33306 MYSQL_DEFAULTS_FILE="$HOME/.my.cnf" ./shell/payment_cutover_gate.sh --date 2026-02-18 --order-no <订单号> --allow-shared-submchid 1 --no-alert
```

4. 若为退款类故障，单独执行退款收敛巡检：

```bash
DB_PORT=33306 MYSQL_DEFAULTS_FILE="$HOME/.my.cnf" ./shell/payment_refund_convergence_check.sh --window-hours 72 --timeout-minutes 30 --no-alert
```

## 3. 职责边界（总部/门店）

1. 总部：支付执行权、退款执行权、配置变更权、故障处置与回滚。
2. 门店：仅提交售后事实与履约证据，不执行打款、不直连支付渠道。
3. 所有支付相关动作以总部系统日志与工单为准，不接受口头变更。

## 4. 处置流程

1. 发现：告警/监控/客服反馈触发。
2. 定位：以`payment_incident_bundle`证据包为唯一入口。
3. 隔离：必要时关闭自动退款/暂停营销活动/限制高风险入口。
4. 修复：先恢复交易主链路，再处理次要告警。
5. 验证：`ops_status=GREEN` 且 `cutover_gate=GO`。
6. 复盘：24小时内输出问题根因、修复项、预防项。

## 5. RTO / RPO 基线

1. 支付主链路（下单+回调+查单）：RTO <= 30分钟，RPO <= 5分钟。
2. 退款链路：RTO <= 2小时，RPO <= 15分钟。
3. 对账链路：当日闭环，T+1 09:00前完成差异工单化。

## 6. 发布前强制检查（上线拦截规则）

1. 必须满足：`cutover_gate=GO`。
2. 若出现`GO_WITH_WARN`，需由总部明确豁免并记录单号与时效。
3. `NO_GO`严禁上线。
