# 一、小店模型场景分析

## 1.1 小店业务特征

荷小悦典型小店模型特征：

-   【门店规模】营业面积80-150㎡，2-3个服务间

-   【技师团队】3-5名技师（含店长），店长通常身兼技师

-   【服务项目】聚焦核心服务，通常3-5个主力项目（足疗、推拿、艾灸、刮痧等）

-   【营业时间】早9:00-晚22:00，全年无休

-   【客流特征】工作日下午和晚上高峰，周末全天高峰

-   【排班模式】固定班+弹性班，部分技师轮休

## 1.2 排班核心诉求

  -----------------------------------------------------------------------
  **角色**                **核心诉求**            **痛点**
  ----------------------- ----------------------- -----------------------
  店长                    • 快速创建一周排班\     • 手工排班耗时久\
                          • 一眼看清技师忙闲\     • 纸质排班易出错\
                          • 灵活调整应对突发\     • 闲时空置浪费\
                          • 闲时吸引客流\         • 临时请假难调配
                          • 提升坪效和人效        

  技师                    • 清楚看到自己排班\     • 不知道今天有几单\
                          • 知道今日预约情况\     • 预约时间记混\
                          • 收到新预约提醒\       • 客户爽约空等
                          • 查看历史服务记录      

  用户                    • 看到技师实时空闲\     • 不知道技师是否有空\
                          • 选择心仪技师和时段\   • 到店才发现满员\
                          • 闲时优惠价格吸引\     • 想预约特定技师难\
                          • 预约成功即时确认      • 高峰时段价格贵
  -----------------------------------------------------------------------

# 二、典型用户故事（场景预演）

## 2.1 场景1：店长周一早上批量创建本周排班

📅 时间：周一 8:30 AM\
👤 角色：荷小悦朝阳门店店长 李姐\
📍 地点：店内办公区，打开管理后台\
\
【故事背景】\
李姐管理着一家小店，有4名技师（小王、小李、小张、小赵）。\
周一早上需要规划本周排班，既要保证服务覆盖，又要控制人力成本。\
\
【操作流程】\
\
1️⃣ 进入排班管理\
→ 点击「技师排班」tab\
→ 默认显示「本周视图」（周一至周日）\
→ 看到4名技师的排班日历（横轴=日期，纵轴=技师）\
\
2️⃣ 快速创建排班（智能推荐）\
→ 点击「智能排班」按钮\
→ 系统弹窗：\
• 选择服务项目：\[60分钟足疗\] ✓\
• 选择技师：\[小王✓ 小李✓ 小张✓ 小赵✓\]\
• 排班时段：\[全天 9:00-22:00\]\
• 休息日：\[周三小王休，周四小李休\]\
→ 点击「生成排班」\
→ 系统自动生成本周排班，显示预览\
\
3️⃣ 调整优化\
→ 看到周五晚上客流预测较高（系统智能提示）\
→ 点击周五19:00-21:00时段\
→ 增加技师：将休息的小赵调回（弹性加班）\
→ 设置加班费规则\
\
4️⃣ 设置闲时优惠\
→ 看到周二、周三下午14:00-16:00预约率低（历史数据）\
→ 点击该时段，标记为「闲时」\
→ 设置优惠价：原价128元 → 闲时98元\
→ 系统自动将时间槽标记为绿色\
\
5️⃣ 确认发布\
→ 点击「发布排班」\
→ 系统自动：\
• 生成所有技师的时间槽（每60分钟一个槽）\
• 发送通知给4名技师：\"本周排班已发布，请查看\"\
• 在用户端开放预约\
\
⏱️ 总耗时：5分钟（对比纸质排班：30分钟+）\
\
💡 李姐感受：\
\"以前手工排班，要在Excel表上画表格，改来改去容易出错。\
现在5分钟搞定，系统还会提醒我哪天客流高，智能多了！\"

## 4.1 智能排班推荐算法

【推荐逻辑】\
\
1️⃣ 分析历史预约热度\
统计每天每时段的预约率：\
• 周一-周五：\
- 早上9-12点：预约率 40%（低）\
- 下午14-17点：预约率 30%（低）→ 建议设为闲时\
- 晚上18-22点：预约率 85%（高）→ 需要满编\
• 周末：\
- 全天预约率 90%（高）→ 需要满编\
\
2️⃣ 计算所需技师数量\
基于预约率预测每时段所需技师：\
\
时段 预测需求 推荐技师数\
周一 9-12点 2.4人 3人（小王、小李、小张）\
周一 14-17点 1.8人 2人（小王、小李）→ 闲时\
周一 18-22点 4.1人 4人（全员）→ 高峰\

• 工作日优先安排休息（周一至周五）\
• 错峰休息，保证服务覆盖\
• 考虑技师偏好（小王倾向周三休）\

• 小王：周三休\
• 小李：周二休\
• 小张：周四休\
• 小赵：弹性班，工作日下午上班\

# 七、产品设计总结

## 7.1 设计亮点

-   【智能化】系统智能推荐排班、动态调价、冲突检测，减少人工决策

-   【可视化】一屏总览、时间轴展示、状态颜色区分，信息一目了然

-   【自动化】预约锁定、到期释放、异常处理，减少人工干预

-   【人性化】友好提示、智能建议、补偿机制，提升体验

-   【数据驱动】基于历史数据推荐、实时监控、持续优化

## 7.2 实施建议

-   【MVP阶段】先实现基础排班和预约功能，验证核心流程

-   【V1.0版本】增加闲时优惠、智能推荐、异常处理

-   【V2.0版本】增加动态定价、数据大屏、智能分析

-   【运营配合】培训店长使用系统，收集反馈持续优化

-   【灰度发布】先在1-2家门店试点，稳定后全面推广

**💡 核心价值：\
**通过技师排班产品化，将传统手工排班提效80%，闲时利用率提升40%，用户预约体验提升，技师收入增长，门店坪效和人效双提升，最终实现「用户满意、技师增收、门店增效」的三赢局面。
