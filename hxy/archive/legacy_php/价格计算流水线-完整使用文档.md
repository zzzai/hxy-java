## 🎯 功能概述

价格计算流水线是荷小悦社区小店版的**核心定价引擎**，负责根据不同的营销策略动态计算服务的最终价格。

### 设计目标

解决社区小店的**营销维度盲区**：
- ❌ **痛点**: 闲时优惠、会员价、优惠券叠加规则混乱，导致价格计算错误
- ✅ **解决方案**: 可配置的价格计算流水线，支持灵活的策略组合

### 核心算法

采用**责任链模式**，按照配置的流水线顺序依次计算价格：

```
base（基础价格）→ time（闲时优惠）→ member（会员价）→ coupon（优惠券）
```

每个步骤独立封装，支持灵活配置和扩展。

## ⭐ 核心特性

### 1. 闲时优惠自动判断

**功能**: 根据用户预约时间自动判断是否在闲时时段内

**配置示例**:
```php
// 在 eb_store_service_sku 表中配置
[
    'is_time_discount' => 1,                      // 参与闲时优惠
    'time_discount_price' => 88.00,               // 闲时价
    'time_discount_period' => '14:00-16:00,22:00-24:00'  // 闲时时段
]
```

**判断逻辑**:
```php
// 示例：用户预约时间为 2026-02-08 15:30:00
// 闲时时段：14:00-16:00,22:00-24:00
// 结果：在闲时时段内 ✅

// 示例：用户预约时间为 2026-02-08 18:00:00
// 结果：不在闲时时段内 ❌
```

**支持特性**:
- ✅ 支持多个时段配置（用逗号分隔）
- ✅ 支持跨天时段（如 22:00-02:00）
- ✅ 精确到分钟级别

---

### 2. 会员价自动应用

**功能**: 根据用户会员等级自动应用会员价

**配置示例**:
```php
// 在 eb_store_service_sku 表中配置
[
    'price' => 128.00,          // 指导价
    'store_price' => 118.00,    // 门店价
    'member_price' => 98.00     // 会员价
]
```

**应用逻辑**:
```php
// 非会员用户
最终价格 = 门店价 (118元)

// 会员用户
最终价格 = 会员价 (98元)
优惠金额 = 门店价 - 会员价 = 20元
```

**支持特性**:
- ✅ 自动识别用户会员等级
- ✅ 支持会员价与闲时价叠加（可配置）
- ✅ 会员价不可用时自动降级到门店价

---

### 3. 优惠券抵扣逻辑

**功能**: 支持满减券和折扣券，可配置抵扣基数

**支持的优惠券类型**:

#### 类型1: 满减券
```php
[
    'type' => 1,                // 满减券
    'coupon_price' => 10.00,    // 优惠金额
    'use_min_price' => 50.00    // 使用门槛
]

// 示例：当前价格 98元，使用满50减10券
// 结果：98 - 10 = 88元 ✅
```

#### 类型2: 折扣券
```php
[
    'type' => 2,        // 折扣券
    'discount' => 85    // 8.5折
]

// 示例：当前价格 98元，使用8.5折券
// 结果：98 × 0.85 = 83.3元 ✅
```

**抵扣基数配置**:
```php
// 在 eb_pricing_strategy 表中配置
[
    'coupon_deduct_base' => 1   // 1=基于原价 2=基于最终价
]

// 示例1: 基于原价（coupon_deduct_base=1）
// 原价128元，闲时价88元，使用8.5折券
// 折扣计算基数 = 128元
// 优惠金额 = 128 × (1 - 0.85) = 19.2元
// 最终价格 = 88 - 19.2 = 68.8元

// 示例2: 基于最终价（coupon_deduct_base=2）
// 原价128元，闲时价88元，使用8.5折券
// 折扣计算基数 = 88元
// 优惠金额 = 88 × (1 - 0.85) = 13.2元
// 最终价格 = 88 - 13.2 = 74.8元
```

**支持特性**:
- ✅ 自动验证优惠券有效期
- ✅ 自动验证优惠券使用条件
- ✅ 支持门店和服务限制
- ✅ 确保最终价格不低于0

### 4. 价格计算流水线

**功能**: 可配置的价格计算顺序，支持灵活的策略组合

**流水线配置**:
```php
// 在 eb_pricing_strategy 表中配置
[
    'price_calc_pipeline' => 'base->time->member->coupon'
]
```

**支持的流水线组合**:

| 流水线 | 说明 | 适用场景 |
|--------|------|----------|
| `base->time->member->coupon` | 标准流水线 | 大部分门店 |
| `base->member->time->coupon` | 会员价优先 | 会员优惠力度大的门店 |
| `base->time->coupon` | 无会员价 | 不区分会员的门店 |
| `base->coupon` | 仅优惠券 | 简化版门店 |

**叠加策略配置**:
```php
// 在 eb_pricing_strategy 表中配置
[
    'is_time_member_overlay' => 0   // 0=不叠加 1=叠加
]

// 示例1: 不叠加（is_time_member_overlay=0）
// 原价128元，闲时价88元，会员价98元
// 流水线: base->time->member
// 步骤1: base = 128元
// 步骤2: time = 88元（应用闲时价）
// 步骤3: member = 98元（应用会员价，覆盖闲时价）
// 最终价格 = 98元

// 示例2: 叠加（is_time_member_overlay=1）
// 原价128元，闲时价88元，会员价98元
// 流水线: base->time->member
// 步骤1: base = 128元
// 步骤2: time = 88元（应用闲时价，折扣率 = 88/128 = 0.6875）
// 步骤3: member = 88 × (98/128) = 67.38元（叠加会员折扣率）
// 最终价格 = 67.38元
```

## ⚙️ 配置说明

### 1. 配置总部全局策略

```php
use app\dao\pricing\PricingStrategyDao;

$pricingStrategyDao = app()->make(PricingStrategyDao::class);

// 配置总部全局策略
$pricingStrategyDao->saveStrategy(1, 0, [
    'price_calc_pipeline' => 'base->time->member->coupon',
    'is_time_member_overlay' => 0,  // 闲时价不与会员价叠加
    'coupon_deduct_base' => 2,      // 优惠券基于最终价抵扣

```

### 2. 配置门店自定义策略

```php
// 配置某门店特殊策略（会覆盖总部策略）
$pricingStrategyDao->saveStrategy(2, 10, [
    'price_calc_pipeline' => 'base->member->time->coupon',
    'is_time_member_overlay' => 1,  // 闲时价可与会员价叠加
    'coupon_deduct_base' => 1,      // 优惠券基于原价抵扣
    'status' => 1
]);
```

$storeServiceSkuDao = app()->make(StoreServiceSkuDao::class);

## ❓ 常见问题

### Q1: 闲时优惠和会员价能叠加吗？

**A**: 可以，通过配置 `is_time_member_overlay` 控制：

- `is_time_member_overlay = 0`: 不叠加（后面的价格覆盖前面的）
- `is_time_member_overlay = 1`: 叠加（折扣率相乘）

**示例**:
```
原价128元，闲时价88元，会员价98元

不叠加模式（is_time_member_overlay=0）:
- 流水线: base->time->member
- 最终价格 = 98元（会员价覆盖闲时价）

叠加模式（is_time_member_overlay=1）:
- 流水线: base->time->member
- 闲时折扣率 = 88/128 = 0.6875
- 会员折扣率 = 98/128 = 0.765625
- 最终价格 = 128 × 0.6875 × 0.765625 = 67.38元
```

---

### Q2: 优惠券基于原价还是最终价抵扣？

**A**: 通过配置 `coupon_deduct_base` 控制：

- `coupon_deduct_base = 1`: 基于原价
- `coupon_deduct_base = 2`: 基于最终价（推荐）

**示例**:
```
原价128元，闲时价88元，使用8.5折券

基于原价（coupon_deduct_base=1）:
- 优惠金额 = 128 × (1 - 0.85) = 19.2元
- 最终价格 = 88 - 19.2 = 68.8元

基于最终价（coupon_deduct_base=2）:
- 优惠金额 = 88 × (1 - 0.85) = 13.2元
- 最终价格 = 88 - 13.2 = 74.8元
```

---

### Q3: 如何自定义价格计算流水线？

**A**: 修改 `eb_pricing_strategy` 表的 `price_calc_pipeline` 字段：

```php
// 标准流水线
'price_calc_pipeline' => 'base->time->member->coupon'

// 会员价优先
'price_calc_pipeline' => 'base->member->time->coupon'

// 仅闲时优惠和优惠券
'price_calc_pipeline' => 'base->time->coupon'

// 仅优惠券
'price_calc_pipeline' => 'base->coupon'
```

---

### Q4: 如何处理跨天的闲时时段？

**A**: 系统自动支持跨天时段，例如：

```php
'time_discount_period' => '22:00-02:00'

// 判断逻辑会自动处理跨天情况
// 22:00-24:00 → 在闲时时段内 ✅
// 00:00-02:00 → 在闲时时段内 ✅
// 02:00-22:00 → 不在闲时时段内 ❌
```

---

### Q5: 价格计算失败怎么办？

**A**: 系统会抛出 `ValidateException` 异常，包含详细的错误信息：

```php
try {
    $result = $priceCalculator->calculatePrice(...);
} catch (ValidateException $e) {
    // 错误信息示例：
    // - "服务不存在或已下架"
    // - "定价策略未配置"
    // - "用户不存在"
    // - "优惠券已过期"
    // - "优惠券不适用于当前门店"
    
    return app('json')->fail($e->getMessage());
}
```
