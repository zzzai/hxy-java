# Domain Priority Matrix

## P0
- Active release path without complete product or recovery definition
- Release-gated domains with unresolved freeze blockers
- Domains with known route/API drift still affecting active scope

## P1
- Planned-near-release domains that need closure next batch
- Cross-domain experience gaps that affect KPI or support burden

## P2
- Longer-tail or reserved domains whose truth is not yet in active release scope
